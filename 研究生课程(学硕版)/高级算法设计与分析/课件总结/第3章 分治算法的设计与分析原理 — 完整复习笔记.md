# 3.1 分治算法原理

------

## 📌 Divide-and-Conquer 算法的设计

设计过程分为**三个阶段**：

| 阶段     | 英文    | 说明                             |
| -------- | ------- | -------------------------------- |
| **分解** | Divide  | 将整个问题划分为多个子问题       |
| **求解** | Conquer | 求解各子问题（递归调用算法）     |
| **合并** | Combine | 合并子问题的解，形成原始问题的解 |

### 结构示意（以归并排序为例）

```
M-Sort(A, p, r)
    q = p + (r-p+1)/2;
    If p < r Then
        B = M-sort(A, p, q-1);    // 求解子问题
        C = M-sort(A, q, r);      // 求解子问题
        Merge(C, B);              // 合并子解
原始问题 ──→ 问题分解(Divide)
              ├── 子问题 → 求解(Conquer) → 子问题解 ─┐
              ├── 子问题 → 求解(Conquer) → 子问题解 ─┤
              └── ...                                 ─┤
                                                        ↓
                                              合并子解(Combine)
                                                        ↓
                                              原始问题的解 → 返回
```

------

## 📌 Divide-and-Conquer 算法的分析

### 分析过程

1. **建立递归方程**
2. **求解递归方程**

### 递归方程的建立方法

设输入大小为 n，T(n) 为时间复杂性：

- **边界条件**：当 n < c 时，T(n) = \Theta(1)
- **Divide 阶段**：划分问题为 a 个子问题，每个子问题大小为 n/b，划分时间 = D(n)
- **Conquer 阶段**：递归调用，时间 = aT(n/b)
- **Combine 阶段**：合并时间 = C(n)

### 最终递归方程

\boxed{T(n) = \begin{cases} \Theta(1) & \text{if } n \leq c \ aT(n/b) + D(n) + C(n) & \text{if } n > c \end{cases}}

> 🔑 **优化方向**：不同算法通过优化不同阶段来降低总体复杂度（降低 a，或降低 D(n)+C(n)）。

# 3.2 整数乘法

> **优化思路**：优化**划分阶段**，降低 T(n) = aT(n/b) + f(n) 中的 **a**

------

## 📌 问题定义

- **输入**：n 位二进制整数 X 和 Y
- **输出**：X 和 Y 的乘积
- 通常计算 X \times Y 的时间复杂性为 O(n^2)
- **目标**：给出复杂性为 O(n^{1.59}) 的算法

------

## 📌 算法思想

将 X 和 Y 各拆为高低两半（各 n/2 位）：

X = \underbrace{A}_{n/2\text{位}} \underbrace{B}_{n/2\text{位}} \quad \Rightarrow \quad X = A \cdot 2^{n/2} + B

Y = \underbrace{C}_{n/2\text{位}} \underbrace{D}_{n/2\text{位}} \quad \Rightarrow \quad Y = C \cdot 2^{n/2} + D

### 朴素展开

XY = (A \cdot 2^{n/2} + B)(C \cdot 2^{n/2} + D) = AC \cdot 2^n + AD \cdot 2^{n/2} + BC \cdot 2^{n/2} + BD

需要 **4 次** n/2 位乘法 → T(n) = 4T(n/2) + O(n) = O(n^2)，**没有改进！**

### 关键变换（Karatsuba 技巧）

XY = AC \cdot 2^n + \underbrace{\left[(A-B)(D-C) + AC + BD\right]}_{替代, AD+BC} \cdot 2^{n/2} + BD

只需计算 **3 次**乘法：AC、BD、(A-B)(D-C)

------

## 📌 算法步骤

```
1. 计算 A-B 和 D-C
2. 计算 n/2 位乘法：AC、BD、(A-B)(D-C)     ← 3次递归
3. 计算 M = (A-B)(D-C) + AC + BD
4. N = AC 左移 n 位，M = M 左移 n/2 位
5. 计算 XY = N + M + BD
```

------

## 📌 时间复杂度

T(n) = \begin{cases} \Theta(1) & n = 1 \ 3T(n/2) + O(n) & n > 1 \end{cases}

使用 **Master 定理**（a=3, b=2，\log_b a = \log_2 3 \approx 1.59）：

\boxed{T(n) = O(n^{\log_2 3}) = O(n^{1.59})}

### 对比总结

| 方法          | 乘法次数 | 递推式       | 复杂度      |
| ------------- | -------- | ------------ | ----------- |
| 小学竖式      | —        | —            | O(n^2)      |
| 朴素分治      | 4 次     | 4T(n/2)+O(n) | O(n^2)      |
| **Karatsuba** | **3 次** | 3T(n/2)+O(n) | O(n^{1.59}) |

# 3.3 最近点对发现算法

> **优化思路**：优化 **Combine 阶段**，降低 T(n) = aT(n/b) + f(n) 中的 **f(n)**

------

## 📌 问题定义

- **输入**：Euclidean 空间上的 n 个点的集合 Q
- **输出**：A, B \in Q，使得

\text{Dis}(A, B) = \min{\text{Dis}(P_i, P_j) \mid P_i, P_j \in Q}

其中 Euclidean 距离：

\text{Dis}(P_i, P_j) = \sqrt{(x_i - x_j)^2 + (y_i - y_j)^2}

------

## 📌 一维空间算法

### 利用排序的算法

```
1. 把 Q 中的点排序
2. 通过排序集合的线性扫描找出最近点对
```

**时间复杂性**：T(n) = O(n \log n)

### Divide-and-Conquer 算法

**边界条件**：若 Q 中仅包含 2 个点，则返回这个点对

**Divide**：

1. 求 Q 中点的中位数 m
2. 用中位数 m 把 Q 划分为两个大小相等的子集合：
   - Q_1 = {x \in Q \mid x \leq m}
   - Q_2 = {x \in Q \mid x > m}

**Conquer**：递归地在 Q_1 和 Q_2 中找出最接近点对 (p_1, p_2) 和 (q_1, q_2)

**Merge**：在 (p_1, p_2)、(q_1, q_2) 和某个 (p_3, q_3) 之间选择最接近点对 (x, y)

- 其中 p_3 是 Q_1 中最大点，q_3 是 Q_2 中最小点

```
Q1         |         Q2
  p1  p2   p3 | q3   q2  q1
               m
```

### 一维时间复杂性

| 阶段    | 时间    |
| ------- | ------- |
| Divide  | O(n)    |
| Conquer | 2T(n/2) |
| Merge   | O(1)    |

T(n) = 2T(n/2) + O(n) \xrightarrow{\text{Master定理}} \boxed{T(n) = O(n\log n)}

------

## 📌 二维空间算法

### 前提假设

Q 中点已经分别按 x 坐标和 y 坐标排序后存储在 X 和 Y 中。

### 边界条件

若 Q 中仅包含 \leq 3 个点，则直接返回最近点对。

### Divide

1. 计算 Q 中各点 x 坐标的中位数 m
2. 用垂线 L: x = m 把 Q 划分成两个大小相等的子集合 Q_L 和 Q_R
   - Q_L 中点在 L 左边
   - Q_R 中点在 L 右边
3. 把 X 划分为 X_L 和 X_R；把 Y 划分为 Y_L 和 Y_R

```
      L: x=m
  QL    |    QR
  XL    |    XR
  YL    |    YR
```

### Conquer

递归地在 Q_L、Q_R 中找出最近点对：

(p_1, p_2) \in Q_L, \quad (q_1, q_2) \in Q_R

d = \min{\text{Dis}(p_1, p_2), , \text{Dis}(q_1, q_2)}

### Merge（核心步骤）

1. 在**临界区**（L 两侧距离 d 以内的带状区域）查找距离小于 d 的最近点对 (p_l, q_r)
2. 若找到，则 (p_l, q_r) 是临界区中最近点对；否则 (p_1, p_2) 和 (q_1, q_2) 中距离最小者为 Q 中最近点对

```
     m-d    L: x=m   m+d
      |       |       |
      |  QL   |  QR   |
      |←─ 临界区 ─→|
```

### (p_l, q_r) 搜索算法

```
1. Q'_L = Q_L - {非临界区点}
   Q'_R = Q_R - {非临界区点}

2. For p(xp, yp) ∈ Q'_L Do
3.     For q(xq, yq) ∈ Q'_R (yp - d ≤ yq ≤ yp + d) Do
            /* 这样的点至多6个 */
4.         If Dis(p, q) < d
5.         Then d = Dis(p, q), 记录(p, q)

6. 如果 d 发生过变化, 与最后的 d 对应的点对即为 (pl, qr),
   否则不存在 (pl, qr)
```

**搜索时间**：O(6n) = O(n)

------

### 📌 为什么每个点最多检查 6 个点？（定理1）

**定理1**：对于左临界区中的每个点 p，p 的右邻域中至多包含 **6 个点**。

**p-右邻域**：以 p 为左边界的 d \times 2d 矩形。

```
    L
    |
  d |  2d
    |←──→|
  ──┼─────┤
    |     |  ↑
    | p   |  d
    |     |  ↓
    |     |  ↑
    |     |  d
    |     |  ↓
  ──┼─────┤
```

**证明**：

1. 把 p-右邻域划分为 **6 个** (d/2) \times (2d/3) 的矩形

```
  ┌──────┬──────┐
  │      │      │  2d/3
  ├──────┼──────┤
  │      │      │  2d/3
  ├──────┼──────┤
  │      │      │  2d/3
  └──────┴──────┘
   d/2     d/2
```

1. 若 p-右邻域中点数 > 6，由**鸽巢原理**，至少有一个矩形中有两个点 u, v
2. 则 (x_u - x_v)^2 + (y_u - y_v)^2 \leq (d/2)^2 + (2d/3)^2 = 25d^2/36
3. 即 \text{Dis}(u, v) \leq 5d/6 < d，与 d 的定义矛盾 ⚡

------

### 📌 二维算法总体流程

```
1. X = 按 x 排序 Q 中点
2. Y = 按 y 排序 Q 中点
3. FindCPP(X, Y)
```

**时间复杂性** = O(n \log n) + T(\text{FindCPP}) = O(n \log n)

### 📌 二维时间复杂性

| 阶段    | 时间    |
| ------- | ------- |
| Divide  | O(n)    |
| Conquer | 2T(n/2) |
| Merge   | O(n)    |

T(n) = 2T(n/2) + O(n) \xrightarrow{\text{Master定理}} \boxed{T(n) = O(n\log n)}

# 3.4 凸包（Convex Hull）构建算法

------

## 📌 问题定义

- **输入**：平面上的 n 个点的集合 Q
- **输出**：CH(Q)：Q 的 convex hull

**凸包**是一个**最小凸多边形** P，Q 的点或者在 P 上或者在 P 内。

**凸多边形** P：连接 P 内任意两点的边都在 P 内。

------

## 📌 基本思想

- 当沿着 Convex hull **逆时针漫游**时，总是**向左转**
- 在极坐标系下按照极角大小排列，然后逆时针方向漫游点集，去除非 Convex hull 顶点（非左转点）

------

## 📌 Graham-Scan 算法

### 算法步骤

```
Graham-Scan(Q):
  /* 栈 S 从底到顶存储按逆时针方向排列的 CH(Q) 顶点 */

  1. 求 Q 中 y 坐标值最小的点 p₀
  2. 按照与 p₀ 极角（逆时针方向）大小排序 Q 中其余点，
     结果为 <p₁, p₂, …, pₙ>
  3. Push p₀, p₁, p₂ into S
  4. FOR i = 3 TO n DO
  5.     While Next-to-top(S)、Top(S) 和 pᵢ 形成非左移动 Do
  6.         Pop(S)
  7.     Push(pᵢ, S)
  8. Return S
```

### 时间复杂性

| 步骤               | 时间                                             |
| ------------------ | ------------------------------------------------ |
| 第1步：求 y 最小点 | O(n)                                             |
| 第2步：极角排序    | O(n \log n)                                      |
| 第3步：初始化栈    | O(1)                                             |
| 第4-7步：扫描      | O(n)（每个点至多进栈一次出栈一次，每次常数时间） |

\boxed{T(n) = O(n \log n)}

------

## 📌 正确性分析（循环不变量方法）

### 循环不变量方法简介

适用于**主要结构为循环的算法**的正确性证明的通用方法。

**主要步骤**：

1. **确定循环不变量 P**：定义在算法所操作的数据上的一个谓词（关键性质）
2. **Initialization**：循环开始前，P 成立
3. **Maintenance**：循环体每执行一次之后 P 仍然成立
4. **Termination**：循环结束后，P 成立

### 循环不变量

> **在处理第 i 个顶点之前，栈 S 中自底到顶存储 CH(Q_{i-1}) 的顶点。**

### Initialization（i=3）

处理 i=3 之前，栈 S 中包含了 Q_{i-1} = Q_2 = {p_0, p_1, p_2} 中的顶点，这三个点形成了一个 CH。**循环不变量为真** ✅

### Maintenance

设在处理第 i（i \geq 3）个顶点之前，循环不变量为真。

**往证**：算法执行第 5~7 步之后，栈 S 中自底到顶存储 CH(Q_i) 的顶点。

**证明要点**：

1. 第 5~6 步 while 循环结束后、第 7 步将 p_i 压入栈之前，设栈顶元素为 p_j，次栈顶为 p_k。此时栈中包含了与 for 循环第 j 轮迭代后相同的顶点，即 CH(Q_j)，循环不变量为真
2. 执行第 7 步后，p_i 入栈，栈 S 中包含了 CH(Q_j \cup {p_i}) 中的顶点，且按逆时针顺序自底向上排列
3. 对于第 i 轮中被弹出的栈顶点 p_t，设 p_r 为紧靠 p_t 的次栈顶点，p_t 被弹出**当且仅当** p_r、p_t、p_i 构成非左移动。因此 p_t 不是 CH(Q_i) 的顶点，即 CH(Q_i \setminus {p_t}) = CH(Q_i)
4. 设 P_i 为 for 循环第 i 轮迭代中被弹出的所有点的集合，则 CH(Q_i \setminus P_i) = CH(Q_i)
5. 又 Q_i \setminus P_i = Q_j \cup {p_i}，故 CH(Q_j \cup {p_i}) = CH(Q_i \setminus P_i) = CH(Q_i) ✅

### Termination

i = n+1，栈 S 中自底到顶存储 CH(Q_n) 的顶点，**算法正确** ✅

------

## 📌 分治法构建凸包

### 边界条件（O(1)）

1. 如果 |Q| < 3，算法停止
2. 如果 |Q| = 3，按照逆时针方向输出 CH(Q) 的顶点

### Divide（使用 O(n) 算法求中值）

选择一个垂直于 x 轴的直线，把 Q 划分为基本相等的两个集合 Q_L 和 Q_R，Q_L 在 Q_R 的左边。

### Conquer（2T(n/2)）

递归地为 Q_L 和 Q_R 构造 CH(Q_L) 和 CH(Q_R)。

### Merge（O(n) — 核心步骤）

```
Merge 实例:
              L
     g        |        a
   h          |      b
              |    c
   i          |    d
     j        |    e
       k      |  f
              p

3个序列：<g, h, i, j, k>（CH_L逆时针）, 
         <a, b, c, d>（CH_R从u到v逆时针）, 
         <f, e>（CH_R从u到v顺时针）

合并后：<g, h, a, b, f, c, e, d, i, j, k>

对合并序列应用 Graham-Scan
```

**Merge 的具体步骤**：

```
1. 找一个 Q_L 的内点 p
2. 在 CH(Q_R) 中找与 p 的极角最大和最小顶点 u 和 v
3. 构造如下三个点序列：
   (1) 按逆时针方向排列的 CH(Q_L) 的所有顶点
   (2) 按逆时针方向排列的 CH(Q_R) 从 u 到 v 的顶点
   (3) 按顺时针方向排列的 CH(Q_R) 从 u 到 v 的顶点
4. 合并上述三个序列
5. 在合并的序列上应用 Graham-Scan
```

### 时间复杂性

| 阶段                  | 时间    |
| --------------------- | ------- |
| Preprocessing         | O(1)    |
| Divide（O(n) 求中值） | O(n)    |
| Conquer               | 2T(n/2) |
| Merge                 | O(n)    |

T(n) = 2T(n/2) + O(n) \xrightarrow{\text{Master定理}} \boxed{T(n) = O(n\log n)}

# 3.5 分位数选择算法

------

## 📌 问题定义

- **Input**：含 n 个（不同）元素的集合 A，和一个数 k
- **Output**：A 中恰好大于 k-1 个元素的元素 x，即**第 k 小**元素

**中位数问题**：找第 \lceil n/2 \rceil 小的元素

**直接算法**：先排序，再定位第 k 个 → O(n \log n)

------

## 📌 主要思想

设 S = {a_1, a_2, \ldots, a_n}

1. 取 p \in S，用 p 将 S 划分为 3 个子集：
   - S_1 = {a_i \mid a_i < p}
   - S_2 = {a_i \mid a_i = p}
   - S_3 = {a_i \mid a_i > p}
2. **三种情况**：

| 条件                     | 操作                                               |
| ------------------------ | -------------------------------------------------- |
| \|S_1\| \geq k           | 第 k 小在 S_1 中，**丢弃** S_2 和 S_3              |
| \|S_1\| + \|S_2\| \geq k | **p 就是第 k 小元素**                              |
| 否则                     | 令 k' = k - \|S_1\| - \|S_2\|，在 S_3 中找第 k' 小 |

------

## 📌 如何选择 p？（Median of Medians）

将 n 个元素分为 \lceil n/5 \rceil 个子集（每个子集 5 个元素）：

```
每组5个元素，组内排序（列为一组，从小到大）：

  ●   ●   ●   ●   ●   ...   ●     ← 最大
  ●   ●   ●   ●   ●   ...   ●
 [M] [M] [M] [M] [M]  ... [M]     ← 中位数行
  ●   ●   ●   ●   ●   ...   ●
  ●   ●   ●   ●   ●   ...   ●     ← 最小
```

- 取所有组的中位数 M，再找中位数的中位数 → 即为 P
- **至少 1/4 的 S 已知 \leq P**
- **至少 1/4 的 S 已知 \geq P**
- 因此递归子问题的规模**最多为 3n/4**

------

## 📌 算法步骤

```
Step 1: 将 S 划分为 ⌈n/5⌉ 个子集，每个包含 5 个元素。
        若 n 不是 5 的倍数，在最后一个子集中添加哑元素。   → O(n)

Step 2: 对每个 5 元素子集排序。                             → O(n)

Step 3: 找元素 p，即 ⌈n/5⌉ 个子集的中位数的中位数。        → T(n/5)
        （递归调用本算法）

Step 4: 用 p 将 S 划分为 S₁、S₂、S₃                       → O(n)
        （分别包含小于、等于、大于 p 的元素）

Step 5: 判断并递归：                                        → T(3n/4)
        - 若 |S₁| ≥ k，丢弃 S₂ 和 S₃，在 S₁ 中找第 k 小
        - 若 |S₁| + |S₂| ≥ k，则 p 就是答案
        - 否则令 k' = k - |S₁| - |S₂|，在 S₃ 中找第 k' 小
```

------

## 📌 算法复杂性分析

### 递推方程

T(n) = T(3n/4) + T(n/5) + O(n)

### 求解过程

设 T(n) = a_0 + a_1 n + a_2 n^2 + \cdots（a_1 \neq 0）

T(3n/4) = a_0 + \frac{3}{4}a_1 n + \frac{9}{16}a_2 n^2 + \cdots

T(n/5) = a_0 + \frac{1}{5}a_1 n + \frac{1}{25}a_2 n^2 + \cdots

T(19n/20) = a_0 + \frac{19}{20}a_1 n + \frac{361}{400}a_2 n^2 + \cdots

注意到 \frac{3}{4} + \frac{1}{5} = \frac{19}{20}，而对高次项 \frac{9}{16} + \frac{1}{25} < \frac{361}{400}，因此：

T(3n/4) + T(n/5) \leq a_0 + T(19n/20)

\Rightarrow T(n) \leq cn + T(19n/20)

### 展开递推

T(n) \leq cn + \frac{19}{20}cn + T\left(\left(\frac{19}{20}\right)^2 n\right)

\leq cn + \frac{19}{20}cn + \left(\frac{19}{20}\right)^2 cn + \cdots + \left(\frac{19}{20}\right)^p cn + T\left(\left(\frac{19}{20}\right)^{p+1} n\right)

其中 \left(\frac{19}{20}\right)^{p+1} n \leq 1，即 \left(\frac{19}{20}\right)^p \leq n

= cn \cdot \frac{1}{1 - \frac{19}{20}} + b = 20cn + b

\boxed{T(n) = O(n)}

# 3.6 快速排序（Quick Sort）

------

## 📌 算法思想

选取一个**枢轴（pivot）**，将数组划分为两部分：左边 \leq pivot，右边 \geq pivot，然后递归排序两部分。

------

## 📌 算法步骤


 QuickSort(A, lo, hi):
 if lo < hi:
 // Step 1: 选择 pivot
 p = 选择枢轴（可以是A[lo], A[hi], 随机, 三数取中等）
 ​

// Step 2: Partition — 划分
 mid = Partition(A, lo, hi, p)
 // 划分后：A[lo..mid-1] ≤ p, A[mid] = p, A[mid+1..hi] ≥ p
 ​
 // Step 3: 递归排序两侧
 QuickSort(A, lo, mid - 1)
 QuickSort(A, mid + 1, hi)



------

## 📌 Partition（划分过程）

### Lomuto 划分方案


 Partition(A, lo, hi):
 pivot = A[hi]      // 取最后一个元素为枢轴
 i = lo - 1
 for j = lo to hi - 1:
 if A[j] ≤ pivot:
 i = i + 1
 swap(A[i], A[j])
 swap(A[i+1], A[hi])
 return i + 1
 ​

### Hoare 划分方案（双指针）


 Partition(A, lo, hi):
 pivot = A[lo]
 i = lo - 1
 j = hi + 1
 while true:
 do: i = i + 1  while A[i] < pivot
 do: j = j - 1  while A[j] > pivot
 if i ≥ j: return j
 swap(A[i], A[j])
 ​

**Partition 的时间复杂度**：O(n)（线性扫描一遍）

------

## 📌 时间复杂度分析

### 最好情况

每次 pivot 恰好选到中位数，划分均匀：

T(n) = 2T(n/2) + O(n) \Rightarrow \boxed{T(n) = O(n \log n)}

### 最坏情况

每次 pivot 选到最大或最小元素，极度不均匀（如已排序数组 + 选首/尾元素）：

T(n) = T(n-1) + O(n) \Rightarrow \boxed{T(n) = O(n^2)}

### 平均情况

假设输入随机排列，或使用随机化选 pivot：

\boxed{T(n) = O(n \log n)}

#### 平均情况推导（简要）

设 T(n) 为平均比较次数，pivot 等概率落在任何位置：

T(n) = \frac{1}{n}\sum_{i=0}^{n-1}\left[T(i) + T(n-1-i)\right] + (n-1)

= \frac{2}{n}\sum_{i=0}^{n-1}T(i) + (n-1)

解此递推得 T(n) = O(n \log n)（具体约 2n \ln n \approx 1.39 n \log_2 n）。

------

## 📌 Pivot 选择策略

| 策略              | 说明                    | 最坏情况                  |
| ----------------- | ----------------------- | ------------------------- |
| 取首/尾元素       | 最简单                  | 已排序数据退化为 O(n^2)   |
| **随机选取**      | 随机选一个元素          | 期望 O(n\log n)，极难退化 |
| **三数取中**      | 取首、中、尾的中位数    | 实践中效果好              |
| Median of Medians | 用3.5的算法找真正中位数 | 保证 O(n\log n)，但常数大 |

------

## 📌 快速排序的性质

| 性质           | 说明                                  |
| -------------- | ------------------------------------- |
| **原地排序**   | 只需 O(\log n) 额外栈空间（递归深度） |
| **不稳定排序** | 相等元素的相对顺序可能改变            |
| **缓存友好**   | 顺序访问，实践中比归并排序快          |
| **平均最快**   | 常数因子小，是实践中最常用的排序算法  |

------

## 📌 优化技巧

| 优化             | 说明                            |
| ---------------- | ------------------------------- |
| 随机化           | 避免最坏情况                    |
| 三数取中         | 更好的 pivot 选择               |
| 小数组用插入排序 | 当 n \leq 10 时切换到插入排序   |
| 尾递归优化       | 只递归较小的一半，较大的用循环  |
| 三路划分         | 处理大量重复元素：< p、= p、> p |

------

## 📌 与归并排序的对比

|          | 快速排序               | 归并排序           |
| -------- | ---------------------- | ------------------ |
| 最好     | O(n\log n)             | O(n\log n)         |
| 平均     | O(n\log n)             | O(n\log n)         |
| 最坏     | O(n^2)                 | O(n\log n)         |
| 空间     | O(\log n)（原地）      | O(n)（需辅助数组） |
| 稳定性   | 不稳定                 | 稳定               |
| 实际速度 | **通常更快**（常数小） | 稍慢               |

> **分治法的统一模式**：
>
> 1. **分解**大问题为小问题
> 2. **递归**解决小问题
> 3. **合并**小问题的解
>
> 不同算法的区别主要在于**如何分**和**如何合并**，合并步骤的效率往往决定了总体复杂度。
