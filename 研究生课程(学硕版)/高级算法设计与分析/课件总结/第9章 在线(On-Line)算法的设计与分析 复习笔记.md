# 第九章 在线(On-Line)算法的设计与分析 复习笔记

------

## 9.1 基本概念

### 一、离线算法 vs 在线算法

|          | 离线算法 (Off-line) | 在线算法 (On-line)         |
| -------- | ------------------- | -------------------------- |
| 输入信息 | 执行前全部已知      | 输入数据实时逐个到达       |
| 解的质量 | 可求最优解          | 通常只能给出近似解         |
| 应用场景 | 传统算法问题        | 数据流、磁盘调度、页面调度 |

### 二、竞争分析 (Competitive Analysis)

**定义：** 在线算法性能与**最优离线算法**性能的比较方法（类似近似比分析）。

**竞争比 (Competitive Ratio) 定义：**

设算法 A 是求解问题 R 的一个在线算法，对于 R 的实例 I：

- $C_{on\text{-}line}(I)$：算法 A 在 I 上产生的近似解代价
- $C_{off\text{-}line}(I)$：最优离线算法在 I 上产生的最优解代价

若对任意实例 I，存在常数 b，使得：
$$C_{on\text{-}line}(I) \leq c(n) \cdot C_{off\text{-}line}(I) + b$$

则称算法 A 的**竞争比为 $c(n)$**，或 A 是 **$c(n)$-competitive** 的。

## 9.2 在线最小生成树问题（贪心确定性算法）

### 一、问题定义

- **输入：** $S = {v_1, v_2, \ldots, v_n}$ 是平面上 n 个点的集合，这 n 个点**逐个出现**（非一次给定）
- **输出：** 一个"小"的生成树（总距离尽量短）
- **注意：** 在线情况下只能给出近似解，无法保证"最小"

### 二、贪心在线生成树算法

```
Greedy-On-line-ST(S)
1. n = |S|;
2. T = ∅;
3. While n ≠ 0 Do
4.     Input(v); n = n-1;
5.     把 v 与 V(T) 之间的最短边加入 T;  // V(T)是T的节点集合
6. Endwhile
```

**算法思想：** 每到达一个新点，将其连接到已有树中距离最近的节点。

### 三、时间复杂度

- 第3-6步需要的比较次数 = $1 + 2 + \cdots + (n-1)$
- **$T(n) = O(n^2)$**

### 四、竞争比分析：$O(\log n)$

**记号：**

- $T_{onl}$：算法产生的生成树
- $l$：最优生成树的代价（长度）

------

**引理1：** $T_{onl}$ 中第 $k$ 最长边的长度最多为 $\frac{2l}{k}$，$1 \leq k \leq n-1$。

**证明：**

令 $S_k = {v \in S \mid v$ 加入 $T_{onl}$ 后，$T_{onl}$ 中出现长度大于 $\frac{2l}{k}$ 的边$}$

**目标：** 证明 $|S_k| \leq k-1$（则 $T_{onl}$ 中至多 $k-1$ 条边长度大于 $\frac{2l}{k}$，即第 $k$ 最长边 $\leq \frac{2l}{k}$）

**关键步骤：**

1. $S_k$ 中任意两点之间的距离必大于 $\frac{2l}{k}$
   - 若不然，设 $u, v$ 之间距离 $\leq \frac{2l}{k}$，则后加入的点不会产生大于 $\frac{2l}{k}$ 的边（矛盾）
2. $S_k$ 上 TSP 优化解的长度 $> |S_k| \cdot \frac{2l}{k}$
3. 利用性质：任意点集上 TSP 优化解长度 $\leq$ 2 × 最小生成树长度
   - 所以：$S_k$ 的最小生成树长度 $> |S_k| \cdot \frac{l}{k}$
4. 由于 $S_k \subseteq S$，$S_k$ 上最小生成树长度 $\leq$ $S$ 上最小生成树长度 $= l$
   - 所以：$l > |S_k| \cdot \frac{l}{k}$，即 $|S_k| < k$

------

**定理1：** Greedy-On-line-ST 算法的竞争比是 $O(\log n)$。

**证明：**

由引理1，$T_{onl}$ 的总长度：
$$L \leq \sum_{k=1}^{n-1} \frac{2l}{k} = 2l \sum_{k=1}^{n-1} \frac{1}{k} = 2l \cdot H(n-1) = l \cdot O(\log n)$$

其中 $H(n-1)$ 是调和级数。

竞争比 $= \frac{L}{l} = O(\log n)$。

## 9.3 随机在线生成树算法

### 一、问题定义

- **输入：** Euclidean 空间上的 n 个点 $v_1, v_2, \ldots, v_n$，逐个出现
- **输出：** 由这 n 个点构成的生成树

### 二、随机在线算法 R(m)

```
Algorithm R(m)
1. T = ∅; input(v₁); input(v₂);
2. 把边(v₁, v₂)添加到 T;
3. For k = 3 To n Do
4.     input(vₖ);
5.     If k ≤ m+1
6.         Then 从(vₖ, v₁), ..., (vₖ, vₖ₋₁)中选择最小边加入T;
7.     Else 从(vₖ, v₁), ..., (vₖ, vₖ₋₁)中随机选择m条边,
8.          把这m条边中最小边加入T;
9. Return T.
```

**说明：**

- 当 $m = n-1$ 时，R(n-1) 退化为确定性贪心算法（每次检查所有已有点）
- 当 $m$ 为常数时，每次只随机抽样 $m$ 条边来做决策

### 三、时间复杂度

- 每次考察需要 $O(m)$ 时间
- 需要 $O(n)$ 次考察
- **$T(n) = O(nm)$**
- 当 $m$ 为常数时，$T(n) = O(n)$

### 四、随机在线算法的竞争比定义

**定义1：** 随机在线算法 A 称为 $c(n)$-competitive，如果存在常数 $b$，使得：
$$E(C_A(\sigma)) \leq c(n) \cdot C_{opt}(\sigma) + b$$

其中 $E(C_A(\sigma))$ 是算法 A 在实例 $\sigma$ 上产生的近似解的**期望代价**，$C_{opt}(\sigma)$ 是最优解代价。

### 五、性能分析

**记号：**

- $\sigma = {1, 2, \ldots, n}$ 是输入点集合，输入顺序为 $1, 2, \ldots, n$
- $D(x, y)$ = 点 $x$ 和 $y$ 之间的距离
- $N(x, k)$ = 在已出现的点中，点 $x$ 的第 $k$ 最近点

------

**定理2（期望长度公式）：**

$$E(L_{R(m)}(\sigma)) = \sum_{i=2}^{m+1} D(i, N(i,1)) + \sum_{i=m+2}^{n} \sum_{j=1}^{m} \frac{\binom{i-1-j}{m-1}}{\binom{i-1}{m}} D(i, N(i,j))$$

**解释：**

- 前 $m+1$ 个点：确定性地选最近点
- 第 $i$ 个点（$i > m+1$）：与前 $i-1$ 个点中第 $j$ 最近点连接的概率为 $\frac{\binom{i-1-j}{m-1}}{\binom{i-1}{m}}$

------

**定理3（竞争比上界）：** 如果 $m$ 是固定常数，则算法 R(m) 的竞争比上界是 $n-1$。

**证明：**

- 令 $D$ 是 n 个输入点的最长边长度
- 最小生成树长度 $L(\sigma) \geq D$
- 在线算法产生的树长度 $\leq (n-1)D$
- 竞争比 $\leq \frac{(n-1)D}{D} = n-1$

------

**定理4：** 在线生成树算法 R(m) 是 $\Theta(n)$-competitive。

即竞争比的下界也为 $\Omega(n)$（当 $m$ 为常数时）。

------

## 9.4 在线凸包问题

### 一、问题定义

- **输入：** 平面上 n 个点的集合 Q，逐个到达
- **输出：** $CH(Q)$：Q 的凸包（Convex Hull）
  - 凸包是一个凸多边形 P，Q 的所有点在 P 上或 P 内
  - **凸多边形性质：** 连接 P 内任意两点的线段都在 P 内

### 二、在线算法基本思想

当已有 $k$ 个节点到达后，得到部分凸包 CH。当第 $k+1$ 个节点 $v$ 到达时：

- 若 $v$ 在 CH **内部** → 不做任何操作
- 若 $v$ 在 CH **外部** → 调整 CH 的边界

**核心方法：平行线近似法** — 用一组平行线记录、调整、近似凸包。

### 三、平行线近似法

**平行线设置：**

- 取 $m$ 对平行线，斜率分别为：
  $$0,\ \tan(\pi/m),\ \tan(2\pi/m),\ \ldots,\ \tan((m-1)\pi/m)$$

**平行线约束条件：**

1. 每个输入节点必须在所有平行线对之间（内部）
2. 每对平行线必须尽可能靠近

### 四、算法 A 详细步骤

```
算法 A：
输入：节点序列 p₁, p₂, ..., 满足条件的 m 对平行线
输出：近似凸包序列 a₁, a₂, ...（aᵢ 是覆盖 p₁,...,pᵢ 的近似凸包）

初始化：
  接收第一个点 p₁;
  构造 m 对平行线，斜率为 0, tan(π/m), ..., tan((m-1)π/m);
  所有平行线初始地相交在 p₁。

第一步：对于新到达点 pᵢ：
  若 pᵢ 落在所有平行线对之间 → 不执行任何操作;
  否则 → 平移最接近 pᵢ 的线到 pᵢ，扩大多边形使 pᵢ 与该线相交。

第二步：沿逆时针方向连接每条平行线上的输入点，形成近似凸包 Aᵢ；
        所有平行线交点形成近似凸包 Eᵢ。

第三步：若不再有点输入 → 停止，输出 Eᵢ;
        否则 i = i+1，接受下一个点，goto 第一步。
```

### 五、时间复杂度

- 每个点到达时需检查 $m$ 对平行线
- 共 $n$ 个点
- **$T(n) = O(mn) = O(n)$**（因为 $m$ 是常数）

### 六、近似解精度分析

**三种多边形：**

- **E**：由 $2m$ 条平行线交点形成的多边形（外接）
- **C**：输入点集合的真实凸包（精确解）
- **A**：算法给出的近似凸包

**关系：** $L(E) \geq L(C) \geq L(A)$（$L(P)$ 表示多边形 $P$ 的周长）

**误差定义：**
$$ERR(A) = \frac{L(E) - L(C)}{L(C)} \leq \frac{L(E) - L(A)}{L(A)}$$

------

**定理1：** $ERR(A) \leq \sec(\pi/2m) - 1$

**证明要点：**

1. **关键不等式：** 对三角形 ABC，角 B 对应角度 $\theta$：
   $$AB + AC \geq BC \cdot \sec(\theta/2)$$

   证明利用余弦定理和函数 $f(x) = \frac{x+1}{x+\cos\theta}$ 的单调递减性（当 $x \geq 1$ 时取到最大值）。

2. **应用到多边形：**

   $$L(E) = P_{2m}A_1 + A_1P_1 + P_1A_2 + \cdots + A_{2m}P_{2m}$$
   $$\leq \sec(\pi/2m) \cdot (P_{2m}P_1 + P_1P_2 + \cdots + P_{2m-1}P_{2m})$$
   $$= \sec(\pi/2m) \cdot L(A)$$

3. **结论：**
   $$ERR(A) \leq \frac{L(E) - L(A)}{L(A)} \leq \sec(\pi/2m) - 1$$

**物理意义：** $m$ 越大（平行线对越多），$\sec(\pi/2m) \to 1$，误差趋于 0。
