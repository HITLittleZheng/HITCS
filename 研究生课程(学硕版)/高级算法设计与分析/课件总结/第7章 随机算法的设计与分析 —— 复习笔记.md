# 第七章 随机算法的设计与分析 —— 复习笔记

------

## 7.1 随机算法简介

### 7.1.1 随机算法基本概念

**定义：**
随机算法是一种在执行过程中，利用概率和统计方法对下一计算步骤作出**随机选择**的算法。

**优越性：**

- 对某些问题：算法简单
- 对某些问题：时间复杂性低
- 对某些问题：两者兼备

**随机性体现：**

- 对同一实例多次执行，效果可能完全不同
- 时间复杂性是一个**随机变量**
- 解的正确性和准确性也是随机的

------

### 7.1.2 随机算法的分类

#### ① Las Vegas 算法

| 特点       | 说明                                   |
| ---------- | -------------------------------------- |
| 解的正确性 | 一旦找到解，**一定正确**               |
| 缺陷       | 可能找不到解                           |
| 改进方式   | 增加反复求解次数，使求解失败概率任意小 |

**平均时间公式：**

设 $p$ 为算法获得解的概率，$S(n)$ 为成功所需平均时间，$E(n)$ 为失败所需平均时间，则：

$$T(n) = \frac{1-p}{p} E(n) + S(n)$$

**推导：** $T(n) = p \cdot S(n) + (1-p)(E(n) + T(n))$，解方程得上式。

------

#### ② Monte Carlo 算法

| 特点           | 说明               |
| -------------- | ------------------ |
| 解的正确性     | **可能给出错误解** |
| 获得正确解概率 | 与执行时间成正比   |

**p-正确算法：**
若 Monte Carlo 算法对任意实例得到正确解的概率 $\geq p$（$0.5 < p < 1$），则称算法是 **p-正确的**，$p - 0.5$ 称为该算法的**优势**。

**一致性：**
若对同一实例，Monte Carlo 算法不会给出两个不同的正确答案，则称算法是**一致的**。

> 对于一致的 p-正确 Monte Carlo 算法，可通过多次执行并选择出现频次最高的解来提高正确率。

**偏真算法：**

- 返回 `true` 时**一定正确**
- 返回 `false` 时**可能错误**
- 多次调用中，只要有一次返回 `true`，即可断定答案为 `true`

------

#### ③ Sherwood 算法

- **一定能**求得正确解
- 适用场景：确定性算法最坏复杂度与平均复杂度相差很大时
- 做法：在确定算法中加入随机性，**消除最坏行为与特定实例的联系**

#### ④ 随机数值算法

- 主要用于数值问题求解
- 输出往往是**近似解**
- 近似解精确度与算法执行时间成正比

------

### 7.1.3 随机算法性能分析

**与确定算法对比：**

|      | 随机算法       | 确定算法平均复杂性分析 |
| ---- | -------------- | ---------------------- |
| 依赖 | 仅依赖随机选择 | 依赖输入的分布         |

**分析目标：**

1. 平均时间复杂性（时间复杂性随机变量的期望）
2. 获得正确/最优解的概率
3. 解的精确度分析

## 7.2 随机数值分析算法

### 7.2.1 数学基础：伯努利大数定律

设 $n_A$ 是 $n$ 次独立试验中事件 $A$ 发生的次数，$A$ 在每次试验中发生的概率为 $p$，则对任意正数 $\varepsilon$：

$$\lim_{n \to \infty} P\left(\left|\frac{n_A}{n} - p\right| < \varepsilon\right) = 1$$

> 用频率近似概率的理论依据。

------

### 7.2.2 随机投点法（计算 π）

**思想：**

- 设半径为 $r$ 的圆及其外切正方形
- 向正方形内随机投点，落入圆内的概率为 $\dfrac{\pi r^2}{4r^2} = \dfrac{\pi}{4}$
- 投掷 $n$ 个点，$k$ 个落入圆内，则 $\dfrac{k}{n} \approx \dfrac{\pi}{4}$，即 $\pi \approx \dfrac{4k}{n}$

**算法：**

```
k = 0
For i = 1 To n:
    随机产生正方形中一点 (x, y)
    If x² + y² ≤ 1 Then k = k + 1
Return (4k) / n
```

**时间复杂度：** $O(n)$（$n$ 为随机样本大小，而非输入规模）

**精度：** 随样本大小 $n$ 增加而提高

------

### 7.2.3 平均值法（计算定积分）

**目标：** 计算 $I = \int_a^b g(x),dx$

**数学基础：**

令 $f(x) = \dfrac{1}{b-a}$（均匀分布密度），则：

$$I = \int_a^b g(x),dx = (b-a) \cdot E[g(\xi)]$$

其中 $\xi$ 是 $[a,b]$ 上均匀分布的随机变量。

由强大数定律，可用下式近似：

$$I \approx \frac{b-a}{n} \sum_{i=1}^{n} g(x_i)$$

**算法：**

```
I = 0
For i = 1 To n:
    随机产生 [a, b] 中的点 x
    I = I + g(x)
Return I × (b - a) / n
```

**时间复杂度：** $O(n)$（$n$ 为随机样本大小）

**精度：** 随样本大小 $n$ 增加而提高

------

## 7.3 随机选择算法（LazySelect）

### 7.3.1 问题定义

- **输入：** 集合 $S = {x_1, x_2, \ldots, x_n}$，整数 $k$（$1 \leq k \leq n$）
- **输出：** $S$ 中第 $k$ 小元素
- **记号：**
  - $\text{Rank}(Q, t)$：元素 $t$ 在集合 $Q$ 中的排名
  - $\min(Q, i)$：集合 $Q$ 中第 $i$ 小元素

------

### 7.3.2 算法思想（Las Vegas 算法）

1. 从 $S$ 中有放回地随机抽取 $n^{3/4}$ 个样本，存入 $R$ 并排序
2. 估计目标元素在 $R$ 中的位置：$x = k \cdot n^{-1/4}$
3. 取区间 $[l, h] = [x - n^{1/2},; x + n^{1/2}]$（对应 $R$ 中的位置）
4. 令 $L = \min(R, l)$，$H = \min(R, h)$
5. 将 $S$ 中属于 $[L, H]$ 的元素存入集合 $P$
6. 在 $P$ 中寻找 $\min(S, k)$

------

### 7.3.3 算法详细步骤

```
LAZYSELECT(S, k):
1.  R = 从S中独立、均匀、有放回地随机选取 n^(3/4) 个元素
2.  在 O(n^(3/4) log n^(3/4)) 时间内对 R 排序
3.  x = (k/n) * n^(3/4) = k * n^(-1/4)
4.  l = max(⌊x - √n⌋, 1);  h = min(⌈x + √n⌉, n^(3/4))
5.  L = min(R, l);  H = min(R, h)
6.  计算 Lp = Rank(S, L),  Hp = Rank(S, H)   // 与S中所有元素比较，O(n)
7.  P = {y ∈ S | L ≤ y ≤ H}
8.  If min(S, k) ∈ P  AND  |P| ≤ 4n^(3/4) + 1
        // 条件 min(S,k) ∈ P 等价于 Lp ≤ k ≤ Hp
9.  Then 排序P，min(S,k) = min(P, k - Lp + 1)，算法结束
10. Else goto 1
```

------

### 7.3.4 时间复杂度分析

| 步骤                | 时间                             |
| ------------------- | -------------------------------- |
| 第6、7步（与S比较） | $O(n)$                           |
| 第2、9步（排序）    | $O(n^{3/4} \log n^{3/4}) = o(n)$ |
| 总计（一遍成功时）  | $O(n)$                           |

------

### 7.3.5 数学工具

**Markov 不等式：**
$$P(Y \geq t) \leq \frac{E[Y]}{t} \quad (Y \geq 0,; t > 0)$$

**Chebyshev 不等式：**
$$P(|X - \mu_X| > t\sigma_X) \leq \frac{1}{t^2}$$

**方差性质：**

- 若 $P(X=1)=p$，$P(X=0)=1-p$，则 $\sigma_X^2 = p(1-p)$
- 若 $X = \sum_{i=1}^n X_i$，$X_i$ 独立同分布，$P(X_i=1)=p$，则 $\sigma_X^2 = np(1-p)$

------

### 7.3.6 性能分析定理

**定理：** 算法执行第1-9步一遍求出 $\min(S,k)$ 的概率为 $1 - O(n^{-1/4})$，即算法需要 $O(n)$ 次比较的概率是 $1 - O(n^{-1/4})$。

**证明要点：**

执行一遍成功的条件：

- **条件 A：** $\min(S,k) \in P$（即 $L \leq \min(S,k) \leq H$）
- **条件 B：** $|P| \leq 4n^{3/4} + 1$

需证 $\Pr(\overline{A \cap B}) = O(n^{-1/4})$。

**① 分析条件A不成立的概率：**

令 $X_i = 1$ 若第 $i$ 个样本 $\leq \min(S,k)$，则：

- $P(X_i=1) = k/n$，$E(X_i) = k/n$
- $X = \sum X_i$：$R$ 中 $\leq \min(S,k)$ 的样本数
- $\mu_X = n^{3/4} \cdot k/n = kn^{-1/4} = x$
- $\sigma_X^2 \leq n^{3/4}/4$，$\sigma_X \leq n^{3/8}/2$

$A$ 不成立 $\Leftrightarrow$ $L > \min(S,k)$ 或 $H < \min(S,k)$：

$$P(L > \min(S,k)) = P(X < l) \leq P(|X - x| > n^{1/2}) \leq P\left(|X-\mu_X| > 2n^{1/8}\sigma_X\right) \leq \frac{1}{(2n^{1/8})^2} = O(n^{-1/4})$$

类似地，$P(H < \min(S,k)) = O(n^{-1/4})$。

**② 分析 A 成立但 B 不成立的概率：**

若 $|P| > 4n^{3/4}+1$，则 $[L, H]$ 中 $S$ 的元素超过 $4n^{3/4}$ 个。取区间 $[\min(S, k - 4n^{3/4}), \min(S, k)]$ 中的样本数 $Y$：

- $E(Y) = 4n^{1/2}$，$\sigma_Y^2 \leq 4n^{3/4}$

由 Chebyshev：
$$P(Y \leq 2n^{1/2}) \leq P(|Y - 4n^{1/2}| \geq 2n^{1/2}) \leq \frac{4n^{3/4}}{4n} = n^{-1/4} = O(n^{-1/4})$$

**结论：** 两个坏事件的概率均为 $O(n^{-1/4})$，故算法一遍成功概率为 $1 - O(n^{-1/4})$。

## 7.4 随机最小割算法（Karger's Algorithm）

### 7.4.1 问题定义

- **输入：** 无向多重连通图 $G = (V, E)$
- **输出：** 最小割（Min-Cut）

**基本概念：**

- **Cut（割）：** 一组边，删除后图变为两个或多个连通分量
- **Cut的大小：** 边数（多重边重复计算）
- **Min-Cut：** 边数最少的割

------

### 7.4.2 边收缩操作（Contraction）

对图 $G$ 中边 $(x, y)$ 做收缩：

1. 用新节点 $z$ 替代节点 $x$ 和 $y$
2. 原来连接到 $x$ 或 $y$ 的边，改为连接到 $z$
3. 删除 $x$ 与 $y$ 之间的边（自环）
4. 图其余部分不变

------

### 7.4.3 算法步骤

```
Karger-MinCut(G):
1.  H = G
2.  While |V(H)| > 2 Do:
3.      随机从 E(H) 中选择一条边 (x, y)
4.      H = H / (x, y)   // 收缩边 (x, y)
5.  Cut = 连接 H 中剩余两个节点的 G 中所有边
6.  Return Cut
```

> 算法输出的 Cut 是连接最终两个超级节点的、所有未被收缩的原始边。

------

### 7.4.4 时间复杂度

**定理1：** 若输入图有 $n$ 个节点，则算法时间复杂度为 $O(n^2)$。

**证明：**

- 每次边收缩需 $O(n)$ 时间
- 共进行至多 $O(n)$ 次收缩（每次减少一个节点，从 $n$ 减到 2）
- 故总时间为 $O(n^2)$

------

### 7.4.5 正确性概率分析

**引理1：** 若 min-cut 大小为 $k$，则图 $G$ 至少有 $\dfrac{kn}{2}$ 条边。

**证明：** 若 $|E| < kn/2$，则存在度 $< k$ 的节点 $p$，删去与 $p$ 相关的 $<k$ 条边即构成比 $k$ 更小的割，矛盾。

------

**定理2：** 设 $C$ 是大小为 $k$ 的 min-cut，则算法结束时 $C$ 中的边未被收缩的概率 $> \dfrac{2}{n(n-1)}$。

**证明：**

设 $A_i$ = 第 $i$ 步未选中 $C$ 中的边（$1 \leq i \leq n-2$）：

| 步骤 | 剩余节点数 | 最少边数 | 选中C中边的概率 | $P(A_i | \text{前面均成功})$ |
|------|-----------|---------|----------------|--------------------------|
| 第1步 | $n$ | $kn/2$ | $\leq k/(kn/2) = 2/n$ | $\geq (n-2)/n$ |
| 第2步 | $n-1$ | $k(n-1)/2$ | $\leq 2/(n-1)$ | $\geq (n-3)/(n-1)$ |
| 第 $i$ 步 | $n-i+1$ | $k(n-i+1)/2$ | $\leq 2/(n-i+1)$ | $\geq (n-i-1)/(n-i+1)$ |
| 第 $n-2$ 步 | $3$ | — | — | $\geq 1/3$ |

$$P\left(\bigcap_{i=1}^{n-2} A_i\right) \geq \frac{n-2}{n} \cdot \frac{n-3}{n-1} \cdot \frac{n-4}{n-2} \cdots \frac{1}{3} = \frac{2}{n(n-1)} \geq \frac{2}{n^2}$$

------

**推论1：** 若独立重复运行算法 $n^2/2$ 次，未发现 min-cut 的概率为：

$$\left(1 - \frac{2}{n^2}\right)^{n^2/2} \leq e^{-1} \approx 0.368$$

> 重复次数越多，发现 min-cut 的概率越高。

## 7.5 随机快速排序算法

### 7.5.1 算法步骤

**随机划分：**

```
Randomized-Partition(A, p, r):
1.  i = Random(p, r)        // 随机选一个下标
2.  交换 A[r] 与 A[i]
3.  Return Partition(A, p, r)
```

**随机快速排序：**

```
Randomized-Quicksort(A, p, r):
1.  If p < r Then
2.      q = Randomized-Partition(A, p, r)
3.      Randomized-Quicksort(A, p, q-1)
4.      Randomized-Quicksort(A, q+1, r)
```

------

### 7.5.2 性能分析

**关键观察：**

- 递归树中，一个子树的根必须与该子树所有节点比较
- 不同子树中的节点不会相互比较
- 任意两个节点**最多比较一次**

**定义随机变量：**

- $x_{(i)}$ 表示 $A$ 中 Rank 为 $i$ 的元素
- $X_{ij} = 1$ 若 $x_{(i)}$ 与 $x_{(j)}$ 发生比较，否则为 $0$
- 算法总比较次数：$T(n) = \sum_{i=1}^{n}\sum_{j>i} X_{ij}$

**期望比较次数：**

$$E[T(n)] = \sum_{i=1}^{n}\sum_{j>i} E[X_{ij}] = \sum_{i=1}^{n}\sum_{j>i} p_{ij}$$

------

### 7.5.3 计算 $p_{ij}$

**分析：**

令 $Z_{ij} = {x_{(i)}, x_{(i+1)}, \ldots, x_{(j)}}$。

$x_{(i)}$ 与 $x_{(j)}$ 被比较，当且仅当：

- $Z_{ij}$ 中，$x_{(i)}$ 或 $x_{(j)}$ **先于其他元素**被选为划分点

在 $Z_{ij}$（共 $j-i+1$ 个元素）中，每个元素等可能被选为划分点，因此：

$$p_{ij} = \frac{2}{j - i + 1}$$

------

### 7.5.4 期望时间复杂度推导

$$E[T(n)] = \sum_{i=1}^{n}\sum_{j>i} \frac{2}{j-i+1} = \sum_{i=1}^{n}\sum_{k=1}^{n-i} \frac{2}{k+1} \leq \sum_{i=1}^{n}\sum_{k=1}^{n} \frac{2}{k} = 2nH_n$$

其中 $H_n = \sum_{k=1}^{n} \dfrac{1}{k} = \Theta(\log n)$，故：

$$E[T(n)] = O(n \log n)$$

**定理：** 随机快速排序算法的**期望时间复杂性**为 $O(n \log n)$。

------

## 7.6 随机 Max-3-CNF 算法

### 7.6.1 问题定义

**输入：**

- CNF 合取范式，每个析取子句恰好包含**三个变量**
- 无任何变量和其非同时出现在同一子句中

**输出：**

- 一组变量赋值，使得值为 $1$（True）的子句个数最多

**例：** $(\neg x_1) \wedge (x_2 \vee x_3) \wedge (x_3) \wedge (\neg x_2)$

------

### 7.6.2 随机算法

```
Random-Max-3-CNF(CNF):
1.  For 对于 CNF 中每个变量 x Do:
2.      以 1/2 的概率令 x = 0
        以 1/2 的概率令 x = 1
3.  Return
```

------

### 7.6.3 性能分析

**定理：** Random-Max-3-CNF 是一个 $\dfrac{8}{7}$-近似算法。

**证明：**

设 CNF 有 $n$ 个变量，$m$ 个子句，第 $i$ 个子句形如 $x_{i1} \vee x_{i2} \vee x_{i3}$。

定义随机变量 $Y_i = 1$ 若第 $i$ 个子句为真，否则 $Y_i = 0$：

$$P(\text{第}i\text{个子句为0}) = P(x_{i1}=0) \cdot P(x_{i2}=0) \cdot P(x_{i3}=0) = \left(\frac{1}{2}\right)^3 = \frac{1}{8}$$

$$P(\text{第}i\text{个子句为1}) = 1 - \frac{1}{8} = \frac{7}{8}$$

$$E[Y_i] = \frac{7}{8}$$

令 $Y = \sum_{i=1}^{m} Y_i$（值为1的子句总数）：

$$E[Y] = \sum_{i=1}^{m} E[Y_i] = \frac{7m}{8}$$

最优解的目标函数值 $\leq m$，故近似比为：

$$\frac{m}{E[Y]} = \frac{m}{7m/8} = \frac{8}{7}$$

> 即算法期望找到最优解的 $7/8$ 以上，近似比为 $8/7$。

## 关键概率工具速查

| 工具               | 公式                                      | 用途             |
| ------------------ | ----------------------------------------- | ---------------- |
| Markov 不等式      | $P(Y \geq t) \leq E[Y]/t$                 | 非负变量上界     |
| Chebyshev 不等式   | $P(|X-\mu| > t\sigma) \leq 1/t^2$         | 偏离期望的概率界 |
| 期望线性性         | $E[\sum X_i] = \sum E[X_i]$               | 复杂期望计算     |
| 独立变量方差可加性 | $\sigma^2(\sum X_i) = \sum \sigma^2(X_i)$ | 方差计算         |
| Bernoulli 方差     | $P(X=1)=p \Rightarrow \sigma^2 = p(1-p)$  | 0-1变量方差      |
