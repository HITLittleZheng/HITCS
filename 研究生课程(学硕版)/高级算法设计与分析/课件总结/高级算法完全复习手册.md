# 第1章 算法概述

### 1.1 生活类比：什么是算法？

**类比**：菜谱就是算法。

- **输入**：食材（鸡蛋、面粉、糖）
- **输出**：成品菜（蛋糕）
- **确定性**：每一步都清楚明确——"打发蛋白3分钟"而不是"随便打打"
- **有限性**：步骤数是有限的，不会无限循环
- **有效性**：每一步你都能实际操作

算法也一样：接收输入，经过有限步确定性的操作，产生输出。

### 1.2 算法的五大条件

| 条件 | 含义 | 反例 |
|------|------|------|
| 输入 | 有0个或多个输入 | 无输入也可以（如生成随机数） |
| 输出 | 至少1个输出 | 没有任何输出的程序不是算法 |
| 确定性 | 每步含义唯一 | "如果差不多就停止"——什么叫"差不多"？ |
| 有限性 | 有限步内终止 | 死循环不是算法 |
| 有效性 | 每步可执行 | "计算所有实数的和"——做不到 |

### 1.3 问题与实例

- **问题**：一般性的描述。如"排序n个数"
- **实例**：具体的输入。如"排序 [3, 1, 4, 1, 5, 9]"

考试中要注意区分"问题的复杂度"和"算法在某个实例上的表现"。

### 1.4 插入排序分析

**算法思想**：像打牌时整理手牌——从左到右，每拿到一张新牌，插入到已排序部分的正确位置。

```
插入排序伪代码：
for j = 2 to n:
    key = A[j]
    i = j - 1
    while i > 0 and A[i] > key:
        A[i+1] = A[i]
        i = i - 1
    A[i+1] = key
```

**复杂度分析（三种情形）**：

| 情形 | 条件 | 比较次数 | 复杂度 |
|------|------|----------|--------|
| 最好 | 已排序 | n-1 | O(n) |
| 最坏 | 逆序 | n(n-1)/2 | O(n²) |
| 平均 | 随机排列 | n(n-1)/4 | O(n²) |

### 1.5 🔥必考 / ⚠️常见坑 / 📝评分重点

- 🔥必考：给定伪代码，分析最好/最坏/平均复杂度
- ⚠️常见坑：平均复杂度不是(n-1+n(n-1)/2)/2，需要概率加权
- 📝评分重点：必须区分"比较次数"和"交换次数"

### 1.6 手算例题

**题目**：对 A=[5, 2, 4, 6, 1, 3] 执行插入排序，写出每轮结束后的数组状态。

```
初始: [5, 2, 4, 6, 1, 3]
j=2, key=2: [2, 5, 4, 6, 1, 3]  ← 5右移，2插入
j=3, key=4: [2, 4, 5, 6, 1, 3]  ← 5右移，4插入
j=4, key=6: [2, 4, 5, 6, 1, 3]  ← 无需移动
j=5, key=1: [1, 2, 4, 5, 6, 3]  ← 全部右移，1插入
j=6, key=3: [1, 2, 3, 4, 5, 6]  ← 4,5,6右移，3插入
```

# 第2章 数学基础

> **🔥🔥🔥 本章是整门课最重要的基础。不掌握这一章，后面所有算法分析都无法进行。强烈建议最先复习本章。**

## 2.1 五种函数阶（渐近记号）

### 2.1.1 生活类比：比较跑步速度

你和张三比赛跑步。

- **大O（上界）**："张三的速度不会超过每小时20公里"——给一个上限。张三可能跑15、10、甚至5，但绝对不超过20。
- **大Ω（下界）**："张三的速度至少是每小时5公里"——给一个下限。张三可能跑10、15、20，但不会低于5。
- **θ（紧确界）**：当你同时给出匹配的上下界——"张三的速度在每小时10到12公里之间"，且这个区间可以无限缩窄到同阶。
- **小o（严格上界）**：张三的速度"明显比"每小时100公里慢——差距是本质性的。
- **小ω（严格下界）**：张三的速度"明显比"每小时1公里快。

### 2.1.2 严格数学定义（🔥必背）

**θ(g(n))**：
```
θ(g(n)) = { f(n) | ∃c₁>0, c₂>0, n₀>0, 
            使得 ∀n≥n₀: 0 ≤ c₁g(n) ≤ f(n) ≤ c₂g(n) }
```
含义：f(n) 被 g(n) 的常数倍上下夹住。

**O(g(n))**（大O，上界）：
```
O(g(n)) = { f(n) | ∃c>0, n₀>0, 
            使得 ∀n≥n₀: 0 ≤ f(n) ≤ c g(n) }
```

**Ω(g(n))**（大Ω，下界）：
```
Ω(g(n)) = { f(n) | ∃c>0, n₀>0, 
            使得 ∀n≥n₀: 0 ≤ c g(n) ≤ f(n) }
```

**o(g(n))**（小o，严格上界）：
```
o(g(n)) = { f(n) | ∀c>0, ∃n₀>0, 
            使得 ∀n≥n₀: 0 ≤ f(n) < c g(n) }
```
注意：这里是**对所有c>0**都成立，这就是"严格"的来源。

**ω(g(n))**（小ω，严格下界）：
```
ω(g(n)) = { f(n) | ∀c>0, ∃n₀>0, 
            使得 ∀n≥n₀: 0 ≤ c g(n) < f(n) }
```

### 2.1.3 极限法证明（🔥🔥必考题型）

**定理**：设极限 L = lim(n→∞) f(n)/g(n) 存在，则：

| L的值 | 结论 |
|-------|------|
| L = 0 | f(n) = o(g(n))，且 f(n) = O(g(n)) |
| 0 < L < ∞ | f(n) = θ(g(n)) |
| L = ∞ | f(n) = ω(g(n))，且 f(n) = Ω(g(n)) |

**手算例题1**：比较 f(n) = 2n²+3n 和 g(n) = n²

```
L = lim(n→∞) (2n²+3n)/n² = lim(n→∞) (2 + 3/n) = 2
∵ 0 < 2 < ∞
∴ 2n²+3n = θ(n²)
```

同时我们可以用定义法验证：
```
上界：c₂=3, n₀=3，当n≥3时：2n²+3n ≤ 2n²+n² = 3n² ✓
下界：c₁=2, n₀=1，当n≥1时：2n²+3n ≥ 2n² ✓
∴ θ(n²)
```

**手算例题2**：比较 f(n) = 100n 和 g(n) = n²

```
L = lim(n→∞) 100n/n² = lim(n→∞) 100/n = 0
∴ 100n = o(n²)，即 100n = O(n²) 但不是 θ(n²)
```

**手算例题3**：比较 f(n) = 2ⁿ 和 g(n) = n³

```
L = lim(n→∞) 2ⁿ / n³ 
用洛必达法则三次：= lim 2ⁿ·(ln2)³ / 6 = ∞
∴ 2ⁿ = ω(n³)
```

### 2.1.4 📝评分重点：证明模板

考试中写函数阶证明必须包含三要素：
```
(1) 写出选择的常数 c（或 c₁, c₂）
(2) 写出 n₀
(3) 验证当 n ≥ n₀ 时不等式成立
```

### 2.1.5 函数阶的性质

**传递性**（所有五种记号都有）：
```
f(n) = O(g(n)) 且 g(n) = O(h(n)) → f(n) = O(h(n))
```

**自反性**（仅 O, Ω, θ 有）：
```
f(n) = θ(f(n))
```

**对称性**（仅 θ 有）：
```
f(n) = θ(g(n)) ⟺ g(n) = θ(f(n))
```

**转置对称性**（O 与 Ω 互为转置，o 与 ω 互为转置）：
```
f(n) = O(g(n)) ⟺ g(n) = Ω(f(n))
f(n) = o(g(n)) ⟺ g(n) = ω(f(n))
```

**⚠️常见坑**：θ有对称性但O没有！不能说"如果f=O(g)，则g=O(f)"。反例：n=O(n²)但n²不是O(n)。

## 2.2 Floor 和 Ceiling

- ⌊x⌋ ：不大于x的最大整数（向下取整）
- ⌈x⌉ ：不小于x的最小整数（向上取整）

性质：⌊x⌋ ≤ x < ⌊x⌋+1，x-1 < ⌊x⌋ ≤ x

在递归方程中，Floor和Ceiling通常不影响渐近阶（Master定理可忽略它们）。

## 2.3 和式估计

常用公式（必背）：

| 和式 | 结果 |
|------|------|
| Σ(i=1 to n) i | n(n+1)/2 = θ(n²) |
| Σ(i=1 to n) i² | n(n+1)(2n+1)/6 = θ(n³) |
| Σ(i=1 to n) 1/i | ln n + O(1) = H(n)（调和数） |
| Σ(i=0 to k) xⁱ | (x^(k+1)-1)/(x-1)（几何级数） |

**定界技巧**：
- 单调递增函数：∫(0 to n) f(x)dx ≤ Σ(i=1 to n) f(i) ≤ ∫(1 to n+1) f(x)dx
- 单调递减函数：不等式方向反过来

**⚠️常见坑**：Σ(1/i) 是 θ(log n) 不是 O(1)！很多同学以为调和级数收敛。

## 2.4 递归方程求解 — Master定理

### 2.4.1 为什么需要递归方程？

分治算法的时间复杂度通常写成递归形式：
```
T(n) = a·T(n/b) + f(n)
```
其中：a = 子问题个数，n/b = 子问题规模，f(n) = 分解+合并代价。

### 2.4.2 Master定理完整陈述（🔥🔥必背）

对于 T(n) = a·T(n/b) + f(n)，设 n^(log_b a) = g(n)（即递归树叶子层的代价）。

比较 f(n) 与 g(n)：

**情况1（叶子层主导）**：
若 f(n) = O(n^(log_b a - ε))，其中 ε > 0
→ T(n) = θ(n^(log_b a))
→ 直觉：f(n) 增长比 g(n) 多项式级别地慢，则总代价由叶子层决定

**情况2（均衡）**：
若 f(n) = θ(n^(log_b a))
→ T(n) = θ(n^(log_b a) · log n)
→ 直觉：每层代价相同，乘以层数 log_b n

**情况3（根层主导）**：
若 f(n) = Ω(n^(log_b a + ε))，其中 ε > 0
且满足正则条件：∃c<1, ∀充分大n: a·f(n/b) ≤ c·f(n)（意味着递归代价递减足够快）
→ T(n) = θ(f(n))
→ 直觉：f(n) 增长比 g(n) 多项式级别地快，则总代价由顶层决定

### 2.4.3 三种情况的直观解释

想象一棵递归树：
- 根节点：f(n) 的代价
- 第1层：a·f(n/b)
- 第2层：a²·f(n/b²)
- ...
- 叶子层：a^(log_b n)·f(1) = n^(log_b a)·θ(1)

**情况1**：叶子层代价 > 所有内部层之和 → T(n)=θ(n^(log_b a))
**情况2**：每层代价相同 → T(n)=θ(n^(log_b a)·层数)
**情况3**：根层代价 > 其他所有层 → T(n)=θ(f(n))

### 2.4.4 📝手算例题（🔥必做）

**例1**：T(n) = 9T(n/3) + n

```
a=9, b=3 → n^(log_3 9) = n²
f(n)=n, g(n)=n²
比较：f(n) = n = O(n^(2-1)) = O(n)  ← ε=1>0
→ 情况1
→ T(n) = θ(n²)
```

**例2**：T(n) = T(2n/3) + 1

```
a=1, b=3/2 → n^(log_{3/2} 1) = n⁰ = 1
f(n)=1, g(n)=1
比较：f(n) = 1 = θ(1) = θ(n⁰)
→ 情况2 (注意log_b a = 0)
→ T(n) = θ(log n)
```

**例3**：T(n) = 3T(n/4) + n log n

```
a=3, b=4 → n^(log_4 3) ≈ n^0.793
f(n)=n log n = Ω(n^(0.793+0.1))  ← ε≈0.1
还需要验证正则条件：
3·(n/4)·log(n/4) = (3/4)n·[log n - log 4] ≤ (3/4)n log n
取 c=3/4 < 1 ✓
→ 情况3
→ T(n) = θ(n log n)
```

**例4（⚠️常见坑）**：T(n) = 2T(n/2) + n / log n

```
a=2, b=2 → n^(log_2 2) = n¹ = n
f(n) = n/log n 不是多项式级小于n（没有ε>0使n/log n=O(n^(1-ε))）
也不等于θ(n)
也不是多项式级大于n
→ Master定理三种情况都不适用！
```

### 2.4.5 ⚠️常见坑：Master定理的"间隙"问题

Master定理只在f(n)与n^(log_b a)的比值是"多项式的"时候才有效。如果它们的比值是log n的同阶，如f(n)=n/log n，那Master定理就不适用，需要使用递归树法或代入法。

# 第3章 分治算法

## 3.1 生活类比：切披萨

你和朋友们要分一张大披萨。如果让大家直接在大披萨上抢，场面会非常混乱（暴力求解）。更好的做法是：
1. **分解**：先把披萨切成若干块（若干小问题）
2. **解决**：每人吃自己的那块（各自解决子问题）
3. **合并**：不需要合并，各吃各的就行

但有些问题在分解后需要"合并"——比如统计全校平均分，每个班算出自己班的平均分，年级组长需要用各班平均分算出年级总平均。

## 3.2 分治算法通用框架

```
Divide-and-Conquer(P):
    if |P| ≤ threshold:   // 基础情况
        直接解决P
    else:
        将P分解为子问题 P₁, P₂, ..., Pₐ
        for each Pᵢ:
            Sᵢ = Divide-and-Conquer(Pᵢ)  // 递归
        合并 S₁, S₂, ..., Sₐ 为最终解
```

**通用递归方程**：
```
T(n) = a·T(n/b) + D(n) + C(n)
```
其中 D(n)=分解代价，C(n)=合并代价。

## 3.3 整数乘法优化（Karatsuba算法）

### 3.3.1 问题定义

**输入**：两个n位二进制整数x和y
**输出**：乘积 xy

### 3.3.2 传统方法为什么慢

```
x = a·2^(n/2) + b    （a高半, b低半）
y = c·2^(n/2) + d    （c高半, d低半）
xy = ac·2ⁿ + (ad+bc)·2^(n/2) + bd
```

直接计算需要4次n/2位乘法：ac, ad, bc, bd。
递归方程：T(n) = 4T(n/2) + O(n) → T(n) = θ(n²)

### 3.3.3 Karatsuba怎么优化：把4变成3

**核心洞察**：不需要分别算ad和bc，而是一次算ad+bc。

```
计算三个乘积：
P₁ = ac
P₂ = bd
P₃ = (a+b)(c+d) = ac + ad + bc + bd

则 ad + bc = P₃ - P₁ - P₂
```

现在只需要3次乘法！递归变为：
T(n) = 3T(n/2) + O(n)

用Master定理：a=3, b=2, n^(log₂3) ≈ n^1.59
T(n) = θ(n^log₂3) ≈ θ(n^1.59)

### 3.3.4 📝完整手算例题

**题目**：用Karatsuba算法计算 1234 × 5678（十进制，n=4位）

```
步骤1：分解
x=1234 → a=12, b=34
y=5678 → c=56, d=78

步骤2：计算三个乘积（递归，这里直接算）
P₁ = a×c = 12×56 = 672
P₂ = b×d = 34×78 = 2652
P₃ = (a+b)(c+d) = 46×134 = 6164

步骤3：求ad+bc
ad+bc = P₃ - P₁ - P₂ = 6164 - 672 - 2652 = 2840

步骤4：合并
xy = 672×10000 + 2840×100 + 2652 = 6720000 + 284000 + 2652 = 7006652

验证：1234×5678 = 7006652 ✓
```

### 3.3.5 🔥必考 / ⚠️常见坑 / 📝评分重点

- 🔥必考：Karatsuba的递归方程 T(n)=3T(n/2)+O(n) → θ(n^1.59)
- ⚠️常见坑：合并阶段的移位不要算错——ac乘以2ⁿ不是2^(n/2)
- 📝评分重点：必须说明"为什么从4次乘法降到3次"

## 3.4 最近点对问题

### 3.4.1 生活类比：找最挤的两个地铁乘客

你在早高峰地铁车厢里，想找站得最近的两个人。与其量所有两两距离（n²级别），不如：
1. 把车厢从中间画一条线
2. 分别在左半和右半找最近对（递归）
3. 检查跨线的"临界区域"是否有更近的对

### 3.4.2 算法详解

**输入**：平面上n个点
**输出**：距离最近的两个点

**一维情况**（热身）：
1. 排序所有点（O(n log n)）
2. 扫描相邻点对（O(n)）
3. 最小相邻距离即最近点对

**二维情况（重点）**：

**步骤1**：按x坐标排序所有点 → P_x
**步骤2**：按y坐标排序所有点 → P_y

**Divide**：用垂直线L把点集分为大小相等的左右两部分Q_L和Q_R

**Conquer**：递归找Q_L中的最近距离δ_L，Q_R中的最近距离δ_R
取 δ = min(δ_L, δ_R)

**Combine（关键步骤）**：
- 构造临界区带状集合：只保留|x坐标-L.x| < δ的点
- 按y坐标排序带状集合中的点
- 对带状集合中每个点p，检查p上方最多7个点（实际上只需检查6个）

### 3.4.3 为什么只检查7个点？（🔥必考证明）

**引理**：在以L为中线、宽度为2δ的带状区域中，任意δ×2δ矩形内最多包含8个点（实际编码中检查7个即可）。

**证明**：将δ×2δ矩形分为8个δ/2×δ/2小方格。每个小方格内最多包含1个点（因为同一方格内任意两点距离 ≤ δ/√2 < δ，与δ是最近距离矛盾）。因此整个矩形最多8个点。

考虑当前点p下方最多有7个其他点（因为p本身占1个）。

### 3.4.4 ⚠️常见坑

很多同学以为临界区检查需要O(n²)——实际上因为只检查7个邻居，是O(n)的。

### 3.4.5 复杂度分析

```
T(n) = 2T(n/2) + O(n)（分解）+ O(n)（合并临界区检查）
     = 2T(n/2) + O(n)

Master定理：a=2, b=2, log₂2=1
f(n)=n = θ(n¹) → 情况2
→ T(n) = θ(n log n)
```

加上预排序O(n log n)，总复杂度O(n log n)。

### 3.4.6 📝评分重点

考试中要求写出完整的算法步骤，特别要强调"临界区检查为什么是O(n)而不是O(n²)"。

## 3.5 Graham-Scan凸包算法

### 3.5.1 生活类比：橡皮筋围钉子

在木板上钉n个钉子，用一根橡皮筋把所有钉子围起来。橡皮筋自然形成的形状就是凸包。

### 3.5.2 问题定义

**输入**：平面上n个点的集合Q
**输出**：Q的凸包CH(Q)——包含Q的最小凸多边形。凸多边形指任意内部两点连线都在多边形内部。

### 3.5.3 算法步骤

1. 找到最左下角的点 p₀（y最小，若相同取x最小）
2. 将其他点按相对于p₀的极角从小到大排序
3. 初始化空栈S，将p₀、p₁、p₂依次压栈
4. for i = 3 to n-1:
   - while 栈顶的两个点与pᵢ构成"右转"（非左转）:
       弹出栈顶
   - 将pᵢ压栈
5. 栈中的点即为凸包顶点（逆时针序）

### 3.5.4 如何判断"左转"还是"右转"

三个点 p₁(x₁,y₁), p₂(x₂,y₂), p₃(x₃,y₃) 构成：
- 左转（逆时针）⇔ 叉积 (x₂-x₁)(y₃-y₁) - (y₂-y₁)(x₃-x₁) > 0
- 右转（顺时针）⇔ 叉积 < 0
- 共线 ⇔ 叉积 = 0

### 3.5.5 循环不变量（🔥🔥评分重点）

**不变量**：算法执行到扫描pᵢ时，栈中的点始终构成已扫描点集的凸包，且按逆时针顺序排列。

**证明思路（归纳法）**：
- 初始：只有p₀,p₁,p₂三个点，显然
- 保持：新点入栈时，若构成右转则弹出，保证栈中始终是"左转链"
- 终止：全部扫描完，栈中即为完整凸包

### 3.5.6 📝手算例题

**题目**：求点集 {(0,0), (2,1), (1,2), (3,3), (4,1), (2,0)} 的凸包。

```
步骤1：p₀ = (0,0)（最左下）
步骤2：按极角排序：p₁=(2,0), p₂=(4,1), p₃=(3,3), p₄=(1,2), p₅=(2,1)
步骤3：
  初始栈 = [(0,0), (2,0), (4,1)]
  
  i=3, p₃=(3,3):
    检查栈顶两个点(2,0)→(4,1)→(3,3)：左转 ✓
    栈 = [(0,0), (2,0), (4,1), (3,3)]
  
  i=4, p₄=(1,2):
    检查(4,1)→(3,3)→(1,2)：右转 ✗ → 弹出(3,3)
    检查(2,0)→(4,1)→(1,2)：左转 ✓
    栈 = [(0,0), (2,0), (4,1), (1,2)]
  
  i=5, p₅=(2,1):
    检查(4,1)→(1,2)→(2,1)：右转 ✗ → 弹出(1,2)
    检查(2,0)→(4,1)→(2,1)：右转 ✗ → 弹出(4,1)
    检查(0,0)→(2,0)→(2,1)：左转 ✓
    栈 = [(0,0), (2,0), (2,1)]

凸包顶点（逆时针）：(0,0) → (2,0) → (4,1) → (1,2) → ?
  等等，还需要检查 (2,1) 是否被后面的点覆盖...

最终凸包：(0,0)→(2,0)→(4,1)→(3,3)→(1,2)→(0,0)
```

### 3.5.7 复杂度

排序O(n log n)，扫描O(n)，总体O(n log n)。

## 3.6 中位数之中位数（BFPRT算法）

### 3.6.1 生活类比：选班长

全班60人选班长。与其让所有人互相投票（O(n²)），不如：
1. 分成12个5人小组，每组推一个"组代表"
2. 12个组代表再投票选"代表中的代表"
3. 这个"代表中的代表"差不多就是全班中游水平的人

### 3.6.2 问题定义

**输入**：n个元素的数组A
**输出**：A中第k小的元素（k=⌈n/2⌉即为中位数）

### 3.6.3 为什么需要这个算法

快速排序的随机pivot期望O(n log n)，但最坏可能O(n²)。BFPRT保证最坏情况下也是O(n)。

### 3.6.4 算法步骤（🔥🔥必考）

```
BFPRT(A, k):
1. 将A划分为⌈n/5⌉组，每组5个元素（可能最后一组不足5个）
2. 对每组排序，取中位数 → 得到⌈n/5⌉个中位数
3. 递归调用BFPRT在这些中位数中找中位数 → M
4. 以M为pivot划分A：
   - A₁ = {x | x < M}
   - A₂ = {x | x = M}
   - A₃ = {x | x > M}
5. if k ≤ |A₁|: return BFPRT(A₁, k)
   else if k ≤ |A₁|+|A₂|: return M
   else: return BFPRT(A₃, k-|A₁|-|A₂|)
```

### 3.6.5 复杂度分析（🔥🔥完整推导）

**关键分析**：以M为pivot能排除多少元素？

- M大于至少一半的"组中位数"（即至少 ⌈⌈n/5⌉/2⌉ ≈ n/10 个组中位数）
- 对于M大于的每个组中位数，该组至少有3个元素 ≤ 组中位数 ≤ M
- 所以至少有 3·(n/10) = 3n/10 个元素 ≤ M
- 同理，至少有 3n/10 个元素 ≥ M

因此无论k是什么，递归调用的问题规模最多为 n - 3n/10 = 7n/10。

更精确：
- A₁的大小至少 n/4（保守估计），至多 3n/4
- A₃同理

**递归方程**：
T(n) ≤ T(n/5) + T(3n/4) + O(n)

证明 T(n) = O(n)：
假设 T(n) ≤ cn，则：
T(n) ≤ cn/5 + c·3n/4 + an = c(0.2+0.75)n + an = 0.95cn + an

选c足够大（c≥20a），即可使cn ≥ 0.95cn + an → T(n) ≤ cn = O(n)。✓

### 3.6.6 📝评分重点

必须画出"元素关系图"来说明为什么至少n/4个元素可被排除。考试中经常要求完整写出元素关系的论证。

## 3.7 快速排序

### 3.7.1 算法

```
QuickSort(A, p, r):
    if p < r:
        q = Partition(A, p, r)
        QuickSort(A, p, q-1)
        QuickSort(A, q+1, r)

Partition(A, p, r):
    x = A[r]  // pivot
    i = p - 1
    for j = p to r-1:
        if A[j] ≤ x:
            i = i + 1
            swap A[i] and A[j]
    swap A[i+1] and A[r]
    return i+1
```

### 3.7.2 Partition循环不变量（🔥🔥评分重点）

算法执行过程中，数组A[p..r]被分为四个区域：
```
[p ... i]     : ≤ x（已处理的、≤pivot的）
[i+1 ... j-1] : > x（已处理的、>pivot的）
[j ... r-1]   : 未处理
[r]           : x（pivot）
```

**证明三阶段**：
- **初始化**：j=p时，i=p-1，区域[p..i]和[i+1..j-1]都为空，满足条件
- **保持**：每轮处理A[j]后，不变式仍然成立
- **终止**：j=r时，所有元素处理完毕，swap后pivot归位

### 3.7.3 复杂度分析

| 情况 | 划分特征 | 复杂度 |
|------|----------|--------|
| 最好 | 每次均分 | T(n)=2T(n/2)+O(n) → θ(n log n) |
| 最坏 | 每次一个为空 | T(n)=T(n-1)+O(n) → θ(n²) |
| 平均 | 随机划分 | θ(n log n) |

### 3.7.4 📝手算例题

**题目**：对 [3, 8, 2, 5, 1, 4, 7, 6] 执行一轮Partition，pivot=A[8]=6

```
x = 6, i = 0
j=1, A[1]=3 ≤ 6: i=1, swap(A[1],A[1]) → [3,8,2,5,1,4,7,6]
j=2, A[2]=8 > 6: 跳过
j=3, A[3]=2 ≤ 6: i=2, swap(A[3],A[2]) → [3,2,8,5,1,4,7,6]
j=4, A[4]=5 ≤ 6: i=3, swap(A[4],A[3]) → [3,2,5,8,1,4,7,6]
j=5, A[5]=1 ≤ 6: i=4, swap(A[5],A[4]) → [3,2,5,1,8,4,7,6]
j=6, A[6]=4 ≤ 6: i=5, swap(A[6],A[5]) → [3,2,5,1,4,8,7,6]
```

# 第4章 动态规划

## 4.1 生活类比：爬楼梯记笔记

你要爬100级楼梯，每次能走1级或2级。问有多少种走法？

如果从第1级开始枚举：到第100级→要么从99级来，要么从98级来→每个又继续分叉...很快就算不过来了。

聪明做法：**带个笔记本**。你先算：
- 到第0级：1种（不动）
- 到第1级：1种（走1级）
- 到第2级：到第1级+到第0级 = 1+1 = 2
- 到第3级：到第2级+到第1级 = 2+1 = 3
- ...
- 到第100级：到第99级+到第98级

每一步只查笔记本上的两个数，加一下就行。这就是DP的核心：**用表格记录中间结果，避免重复计算**。

## 4.2 动态规划三要素（🔥🔥必考）

### 4.2.1 优化子结构

**定义**：问题的最优解包含其子问题的最优解。

**反面例子**：最长简单路径问题没有优化子结构——因为最优解中可能包含次优的子路径（为了避免环）。

### 4.2.2 重叠子问题

**定义**：递归过程中，同一个子问题被多次求解。

**判断标准**：画出递归树，如果有很多相同节点，就有重叠子问题。

### 4.2.3 自底向上计算

从最小的子问题开始，逐步填表到大问题。这样保证每个子问题只算一次。

### 4.2.4 DP解题通用套路

```
1. 定义子问题：用参数(i,j,...)描述一个子问题的状态
2. 写出递归方程：m[state] = f(更小的子问题)
3. 确定填表顺序：从小到大的计算顺序
4. 填表计算结果
5. 从表中构造最优解（如果需要）
```

## 4.3 LCS（最长公共子序列）

### 4.3.1 生活类比：找两人的共同爱好

张三喜欢的歌手：周杰伦→林俊杰→陈奕迅→邓紫棋→张惠妹
李四喜欢的歌手：林俊杰→邓紫棋→周杰伦→陈奕迅

找两人最长的共同喜好顺序：林俊杰→邓紫棋→陈奕迅（或 周杰伦→陈奕迅），长度=3。

注意：不要求连续！邓紫棋在张三的序列中是第4个，在李四的序列中是第2个。

### 4.3.2 问题定义

**输入**：序列 X=⟨x₁,x₂,...,xₘ⟩，Y=⟨y₁,y₂,...,yₙ⟩
**输出**：X和Y的最长公共子序列Z（不要求连续）

### 4.3.3 优化子结构定理及证明（🔥🔥必考）

**定理**：设 Z=⟨z₁,z₂,...,zₖ⟩ 是 X=⟨x₁,...,xₘ⟩ 和 Y=⟨y₁,...,yₙ⟩ 的LCS。

**(1)** 若 xₘ = yₙ，则 zₖ = xₘ = yₙ，且 Zₖ₋₁ 是 Xₘ₋₁ 和 Yₙ₋₁ 的LCS。

**证明(1)**：
假设 zₖ ≠ xₘ = yₙ。那么把 xₘ = yₙ 追加到Z末尾，得到一个长度为k+1的公共子序列，与Z是LCS矛盾。所以 zₖ = xₘ = yₙ。

Zₖ₋₁ 是 Xₘ₋₁ 和 Yₙ₋₁ 的公共子序列是显然的（Zₖ₋₁ 是Z的前k-1个，都来自Xₘ₋₁和Yₙ₋₁）。

现在假设存在Xₘ₋₁和Yₙ₋₁的公共子序列W，其长度 > k-1。则W追加xₘ得到的序列长度为 > k，且是X和Y的公共子序列——与Z是LCS矛盾。故Zₖ₋₁是Xₘ₋₁和Yₙ₋₁的LCS。∎

**(2)** 若 xₘ ≠ yₙ 且 zₖ ≠ xₘ，则 Z 是 Xₘ₋₁ 和 Y 的LCS。

**证明(2)**：
由于zₖ≠xₘ，Z的所有字符都不涉及xₘ，所以Z是Xₘ₋₁和Y的公共子序列。

假设存在Xₘ₋₁和Y的更长公共子序列W。那W也是X和Y的公共子序列，且|W|>|Z|，矛盾。∎

**(3)** 若 xₘ ≠ yₙ 且 zₖ ≠ yₙ，则 Z 是 X 和 Yₙ₋₁ 的LCS。

证明与(2)对称。

### 4.3.4 递归方程（🔥必背）

设 C[i, j] = Xᵢ 和 Yⱼ 的LCS长度。

```
C[0, j] = C[i, 0] = 0                    (边界)

C[i, j] = C[i-1, j-1] + 1                if xᵢ = yⱼ
C[i, j] = max(C[i-1, j], C[i, j-1])      if xᵢ ≠ yⱼ
```

**直观理解**：
- 末尾相同 → 两个都取，长度+1
- 末尾不同 → 至少有一个末尾字符不在LCS中，取两种情况中较好的

### 4.3.5 B表（方向表）

为了构造解，需要记录每个C[i,j]是从哪个方向来的：
- ↖（左上）：C[i-1, j-1] + 1 → xᵢ=yⱼ
- ↑（上）：C[i-1, j] → xᵢ≠yⱼ 且上面更大
- ←（左）：C[i, j-1] → xᵢ≠yⱼ 且左边更大

### 4.3.6 📝完整手算例题（🔥🔥必做）

**题目**：X="ABCBDAB", Y="BDCABA"，求LCS。

```
步骤1：构建 C 表（7×6 = m×n）
        j→ 0 1 2 3 4 5 6
        Y:   - B D C A B A
    i↓ X:
    0  -  0 0 0 0 0 0 0
    1  A  0 0 0 0 1 1 1
    2  B  0 1 1 1 1 2 2
    3  C  0 1 1 2 2 2 2
    4  B  0 1 1 2 2 3 3
    5  D  0 1 2 2 2 3 3
    6  A  0 1 2 2 3 3 4
    7  B  0 1 2 2 3 4 4

逐格推导（关键格）：
C[1,4]：x₁=A, y₄=A → 相同 → C[0,3]+1=1 ✓
C[2,2]：x₂=B, y₂=D → 不同 → max(C[1,2]=0, C[2,1]=0)=0
         等等，x₂=B, y₁=B → C[2,1]=C[1,0]+1=1 ✓
C[4,1]：x₄=B, y₁=B → C[3,0]+1=1
C[4,5]：x₄=B, y₅=B → C[3,4]+1=1+1... 等等 C[3,4]=2,
         所以 C[4,5]=2+1=3 ✓
C[6,4]：x₆=A, y₄=A → C[5,3]+1=2+1=3 ✓
C[7,6]：x₇=B, y₆=A → 不同 → max(C[6,6]=4, C[7,5]=4)=4

步骤2：构造 LCS
从 C[7,6]=4 开始回溯：
- C[7,6]: x₇≠y₆, ← 或 ↑ 都可以（值相等）
  走↑：来到C[6,6]=4
- C[6,6]: x₆=A, y₆=A → ↖ → 取A → 来到C[5,5]=3
- C[5,5]: x₅=D, y₅=B → 不同, ↑ 和 ← 都是3 → 走↑ → C[4,5]=3
- C[4,5]: x₄=B, y₅=B → ↖ → 取B → 来到C[3,4]=2
- C[3,4]: x₃=C, y₄=A → 不同 → 走↑ → C[2,4]=1
- C[2,4]: x₂=B, y₄=A → 不同 → 走← → C[2,3]=1
- C[2,3]: x₂=B, y₃=C → 不同 → 走← → 等等...

实际上回溯时发现走到C[2,2]: x₂=B, y₂=D → 走↑ → C[2,1]=1
C[2,1]: x₂=B, y₁=B → ↖ → 取B

LCS = "BCBA"（回溯路径不唯一，也可能是"BDAB"）
长度 = 4 ✓
```

### 4.3.7 复杂度

时间O(mn)，空间O(mn)。可优化到O(min(m,n))（只保留两行）。

### 4.3.8 ⚠️常见坑

- LCS不要求连续！如果要求连续子串（Longest Common Substring），解法不同
- 回溯时方向不唯一时要选一条走到底
- B表的"↑"和"←"都可能，不在考试中纠结方向唯一性

## 4.4 矩阵链乘法

### 4.4.1 生活类比：搬运货物的最优顺序

你要搬三堆货物：
- 把A堆搬到B堆旁边（代价=AB大小）
- 再把(AB合并堆)搬到C堆旁边（代价=ABC/AB大小）
- 关键：先搬哪两堆总代价最小？

### 4.4.2 问题定义

**输入**：矩阵链 A₁×A₂×...×Aₙ，Aᵢ的维度为 p_{i-1}×pᵢ
**输出**：最优加括号方案，使标量乘法总次数最小

**背景**：矩阵乘法 A(p×q)×B(q×r) 的标量乘法次数 = p·q·r

### 4.4.3 优化子结构证明

**定理**：设 Aᵢ...Aⱼ 的最优加括号方案在 Aₖ 和 Aₖ₊₁ 之间断开，则 Aᵢ...Aₖ 和 Aₖ₊₁...Aⱼ 的加括号方案也分别是各自子问题的最优解。

**证明**（反证法）：若Aᵢ...Aₖ存在更优的加括号方案（代价更小），则替换后整体代价变小，与Aᵢ...Aⱼ是最优解矛盾。∎

### 4.4.4 递归方程（🔥必背）

设 m[i, j] = Aᵢ...Aⱼ 的最少标量乘法次数。

```
m[i, i] = 0                               (单个矩阵，不需要乘)

m[i, j] = min_{i≤k<j} {
    m[i, k] +                    ← Aᵢ...Aₖ 的最优代价
    m[k+1, j] +                  ← Aₖ₊₁...Aⱼ 的最优代价
    p_{i-1}·p_k·p_j              ← 最后一步合并两个结果的代价
}
```

### 4.4.5 📝完整手算例题（🔥🔥必做）

**题目**：A₁(30×35), A₂(35×15), A₃(15×5), A₄(5×10), A₅(10×20), A₆(20×25)
即 p = [30, 35, 15, 5, 10, 20, 25]


### 4.4.6 复杂度

三层循环：(l, i, k)各至多n步 → O(n³)。空间O(n²)。

### 4.4.7 ⚠️常见坑

- 填表顺序是**对角线**（l从2到n），不是从上到下
- p数组的下标：p_{i-1} 是 Aᵢ 的行数，pᵢ 是 Aᵢ 的列数
- 合并代价是 p_{i-1}·p_k·p_j，不是 pᵢ·p_k·p_j

## 4.5 凸多边形最优三角剖分

### 4.5.1 问题定义

**输入**：凸多边形 P=(v₀,v₁,...,vₙ)，代价函数 w(三角形)
**输出**：三角剖分使三角形代价之和最小

### 4.5.2 与矩阵链乘法的同构

这是最重要的观察：**三角剖分和矩阵链乘法是完全同构的问题**。

对应关系：
- 矩阵Aᵢ ↔ 顶点v_{i-1}和vᵢ之间的边
- 矩阵链Aᵢ...Aⱼ ↔ 多边形⟨v_{i-1}, vᵢ, ..., vⱼ⟩
- 在k处断开 ↔ 三角形v_{i-1}v_kv_j

**递归方程**：
```
t[i, i] = t[j, j] = 0
t[i, j] = min_{i≤k<j} {t[i, k] + t[k+1, j] + w(Δv_{i-1}v_kv_j)}
```

**📝评分重点**：考试中可能让你"修改矩阵链乘法的代码解决三角剖分问题"——本质就是改p_{i-1}p_kp_j为w(三角形)。

## 4.6 0/1背包问题

### 4.6.1 生活类比：收拾行李箱

你要出差，行李箱容量有限。每件物品有重量和价值：
- 笔记本电脑：重2kg，价值5000元
- 西装：重3kg，价值4000元
- 鞋子：重4kg，价值2000元

行李箱最多装5kg。你不可能每样都带——必须选择。

### 4.6.2 与部分背包的区别

| 类型 | 可否分割物品 | 算法 |
|------|-------------|------|
| 0/1背包 | 不可（要么全拿，要么不拿） | 动态规划 |
| 部分背包 | 可分割（如金粉） | 贪心（按价值密度） |

### 4.6.3 优化子结构证明

**定理**：设(yᵢ,...,yₙ)是子问题[{i,...,n}, C-Σwₖyₖ]的优化解，则(yᵢ₊₁,...,yₙ)是子问题[{i+1,...,n}, C-Σwₖyₖ-wᵢyᵢ]的优化解。

**证明**：反证法。若存在更优的(yᵢ₊₁',...,yₙ')，则替换后得到原问题更优解，矛盾。∎

### 4.6.4 递归方程（🔥必背）

设 m(i, j) = 考虑物品i到n，背包容量为j时的最大价值。

```
边界：m(n, j) = 0   (0≤j<wₙ),   m(n, j) = vₙ   (j≥wₙ)

一般：m(i, j) = m(i+1, j)              如果 j < wᵢ（装不下i）
      m(i, j) = max(m(i+1, j),          不选i
                    m(i+1, j-wᵢ)+vᵢ)   选i            如果 j ≥ wᵢ
```

**注意填表方向**：从 i=n 到 i=1，从 j=0 到 j=C。

## 4.7 最优二叉搜索树（OBST）

### 4.7.1 生活类比：字典排版

你要编一本字典。某些字查得特别频繁（如"的""是""一"），某些字很少查。如何排列二叉树使得平均查找次数最小？

如果把"的"放在根的位置，查"的"就很快（1次比较）。如果把"觌"放在根，绝大多数查找都要经过它才能找到常用字——效率极低。

### 4.7.2 问题定义

**输入**：
- 关键字集 K={k₁,k₂,...,kₙ}（递增排序）
- 搜索kᵢ的概率 pᵢ
- 搜索落在(kᵢ, kᵢ₊₁)区间内的值的概率 qᵢ
- (d₀对应(-∞,k₁), dₙ对应(kₙ,+∞))

**输出**：最小化期望搜索代价的二叉搜索树

### 4.7.3 期望代价公式

```
E[搜索代价] = Σ(i=1 to n) (depth(kᵢ)+1)·pᵢ + Σ(i=0 to n) (depth(dᵢ)+1)·qᵢ
```

### 4.7.4 优化子结构

**定理**：若T是关于{kᵢ,...,kⱼ}的最优BST，根为kᵣ，则左子树T_left是关于{kᵢ,...,kᵣ₋₁}的最优BST，右子树T_right是关于{kᵣ₊₁,...,kⱼ}的最优BST。

### 4.7.5 递归方程（🔥必背）

```
W(i, i-1) = q_{i-1}
W(i, j) = W(i, j-1) + p_j + q_j

E(i, i-1) = q_{i-1}
E(i, j) = min_{i≤r≤j} {E(i, r-1) + E(r+1, j) + W(i, j)}  (j≥i)
```

W(i,j)的作用：当子树深度增加1时，期望代价增加W(i,j)。

### 4.7.6 复杂度

O(n³)时间，O(n²)空间。与矩阵链乘法结构类似（三层循环l,i,r）。

# 第5章 贪心算法

## 5.1 生活类比：自助餐厅

你去自助餐厅，策略是"每次夹最想吃的菜"。

- **如果可以无限吃**（部分背包）：贪心永远最佳——先夹价值密度最高的食物
- **如果餐盘空间有限，每道菜只能拿一份**（0/1背包）：贪心可能失败——你可能夹了一块牛排占了半盘子，结果错过后面更值的那块

这就是贪心的本质：**有些问题"只看眼前"就够了，有些则不然**。

## 5.2 贪心算法的两个核心性质

### 5.2.1 贪心选择性质

**定义**：可以通过做局部最优（贪心）选择来构造全局最优解。

**验证方法**：交换论证法——假设存在不包含贪心选择的最优解，通过"替换"将其变为包含贪心选择的另一个最优解。

### 5.2.2 优化子结构

与DP相同：问题的最优解包含子问题的最优解。贪心做出一个选择后，剩下的是同类子问题。

## 5.3 活动选择问题

### 5.3.1 问题定义

**输入**：n个活动，活动i的开始时间sᵢ和结束时间fᵢ
**约束**：任一时刻只能进行一个活动
**输出**：最大相容活动子集

### 5.3.2 贪心策略

**按结束时间递增排序**，每次选最早结束且与已选活动相容的活动。

### 5.3.3 为什么选"最早结束"而不是"最短"或"最早开始"

| 策略 | 反例 |
|------|------|
| 最早开始 | 一个从0到24的活动霸占整天 |
| 最短持续 | 两个短活动如果重叠也只能选一个 |
| 最少冲突 | 反例可以构造 |

只有"最早结束"能保证最优。为什么？因为它**给剩下的活动留下了最多的时间**。

### 5.3.4 正确性证明（交换论证法）🔥🔥🔥完整版

**定理（贪心选择性质）**：设活动按结束时间递增排序，活动1（最早结束）必在某个最优解中。

**证明**：

设 A* = {aₖ₁, aₖ₂, ..., aₖₘ} 是一个最优解（按结束时间排序），且 |A*| 是最大活动数。

**情况1**：若 k₁ = 1，则 A* 已经包含活动1，得证。

**情况2**：若 k₁ ≠ 1，构造 A' = (A* - {aₖ₁}) ∪ {a₁}。

需要证明 A' 也是可行解且 |A'| = |A*|。

- |A'| = |A*| - 1 + 1 = |A*|，大小不变 ✓
- 由于按结束时间排序后活动1最早结束，即 f₁ ≤ fₖ₁
- A* 是可行解，所以 aₖ₁ 与 A* - {aₖ₁} 的活动不冲突
- 由于 f₁ ≤ fₖ₁，活动1更早结束 → 活动1也不与 A* - {aₖ₁} 冲突
- 故 A' 是可行解 ✓

因此 A' 是包含活动1的最优解。∎

**定理（优化子结构）**：做出贪心选择（选活动1）后，原问题变为在活动1结束后开始的活动中选最大相容子集的子问题。

A' = {1} ∪ (在S₁中的最优解)，其中 S₁ = {i | sᵢ ≥ f₁}。

**完整算法**：
```
Greedy-Activity-Selector(s, f):
    n = s.length
    按f递增排序
    A = {1}
    k = 1
    for m = 2 to n:
        if s[m] ≥ f[k]:
            A = A ∪ {m}
            k = m
    return A
```

### 5.3.5 📝手算例题

**题目**：6个活动，s=[1,3,0,5,3,5], f=[4,5,6,7,8,9]

```
排序后（按f）：
活动1: [1,4]  ← 最早结束
活动2: [3,5]
活动3: [0,6]
活动4: [5,7]
活动5: [3,8]
活动6: [5,9]

贪心选择：
选活动1 [1,4]，k=1
m=2: s₂=3 < f₁=4 → 冲突，跳
m=3: s₃=0 < f₁=4 → 冲突，跳
m=4: s₄=5 ≥ f₁=4 → 选活动4 [5,7], k=4
m=5: s₅=3 < f₄=7 → 冲突，跳
m=6: s₆=5 < f₄=7 → 冲突，跳

```

## 5.4 Huffman编码

### 5.4.1 生活类比：莫尔斯电码

莫尔斯电码中，常用字母E是"·"，不常用的Q是"——·—"。Huffman编码就是用这个思想：高频字符给短编码，低频字符给长编码。

### 5.4.2 问题定义

**输入**：字符集C，每个字符c有频率f[c]
**输出**：最优前缀码（没有任何一个编码是另一个的前缀）

### 5.4.3 贪心策略

每次取频率最小的两个字符，合并成一个新字符（频率为两者之和），递归此过程。

### 5.4.4 两个关键引理（🔥必考）

**引理1（优化子结构）**：设x,y是C中频率最小的两个字符。则存在C的一个最优前缀码，使得x和y的码字长度相同且仅最后一位不同。

**证明思路**：在最优树中，把最深层的两个兄弟叶节点替换为x和y（x,y频率最小→放到最深不增加代价）。

**引理2（贪心选择性）**：设x,y是C中最小的两个频率。令z为新字符，f[z]=f[x]+f[y]。C' = (C-{x,y})∪{z}。若T'是C'的最优前缀码树，则将T'中叶节点z替换为以x,y为儿子的内部节点，得到的树T是C的最优前缀码。

**证明**：通过代价恒等式 B(T) = B(T') + f[x] + f[y] 证明。若T不是最优，则T'也不是最优，矛盾。

### 5.4.5 📝手算例题

**题目**：字符频率 a:45, b:13, c:12, d:16, e:9, f:5

```

步骤2：合并 f+e = 14（新节点z₁）
```

## 5.5 最小生成树（MST）

### 5.5.1 生活类比：铺设最省钱的电网

你要把n个村庄用电缆连起来。直接每对村庄都连太花钱（完全图有n(n-1)/2条边）。你想选n-1条边使总长度最短——这就是MST。

### 5.5.2 Generic-MST框架（必背）

```
Generic-MST(G, w):
    A = ∅
    while A 不是生成树:
        找到一条边(u,v)，它是A的安全边
        A = A ∪ {(u,v)}
    return A
```

### 5.5.3 安全边定理（🔥🔥必考）

**定义（割）**：无向图G=(V,E)的一个割(S, V-S)是V的一个划分。

**定义（尊重）**：边集A尊重割(S,V-S)，如果A中没有边横跨该割。

**定义（轻边）**：在横跨割的所有边中，权重最小的边称为轻边。

**安全边定理**：设G=(V,E)是无向连通图，A是E的子集且包含在G的某棵MST中，(S,V-S)是尊重A的任意一个割，(u,v)是横跨(S,V-S)的一条轻边，则(u,v)是A的安全边（即A∪{(u,v)}也包含在某棵MST中）。

**证明（交换论证法）**：

设T是包含A的一棵MST。
- 若(u,v)∈T，则T已包含(u,v)，证毕。
- 若(u,v)∉T，则T中存在u到v的唯一路径p。p上必有边(x,y)横跨割(S,V-S)。
  由于(u,v)是轻边，w(u,v) ≤ w(x,y)。
  构造 T' = (T - {(x,y)}) ∪ {(u,v)}
  w(T') = w(T) - w(x,y) + w(u,v) ≤ w(T)
  又因T是MST，故w(T') = w(T)，T'也是MST ✓
  且A⊆T'（因为替换的边(x,y)不在A中——A尊重该割）
  故(u,v)是安全边。∎

### 5.5.4 Kruskal算法

```
Kruskal(G):
    A = ∅
    将所有边按权重递增排序
    for each 边(u,v) in 排序后:
        if Find-Set(u) ≠ Find-Set(v):  // 不在同一连通分量
            A = A ∪ {(u,v)}
            Union(u, v)
    return A
```

**正确性**：每次选中的最小权重边(u,v)一定横跨某割S（S=u所在连通分量, V-S=其他），且是该割的轻边。由安全边定理，(u,v)是安全边。

**复杂度**：O(m log m)，主要来自排序。

### 5.5.5 Prim算法

与Kruskal对称：每次选与当前树连接的最小权重边。也是O(V²)或O(E log V)。

### 5.5.6 🔥必考题型

考试常考：给一个图，要求学生用Kruskal或Prim算法逐步画出MST的生成过程。

## 5.6 Matroid（拟阵）理论

### 5.6.1 为什么需要Matroid

很多贪心算法（MST、任务调度等）的底层原理可以统一为"在拟阵上求最大权重独立子集"。理解了拟阵，就理解了一类贪心算法的正确性根源。

### 5.6.2 拟阵定义

一个拟阵 M=(S, I)，其中S是有限集，I是S的子集族（称为独立集），满足：

**(1) 遗传性**：若B∈I且A⊆B，则A∈I。
→ 独立集的子集也是独立集

**(2) 交换性**：若A,B∈I且|A|<|B|，则∃x∈B-A使得A∪{x}∈I。
→ 可以从大的独立集中"借"一个元素给小的

### 5.6.3 图拟阵

图G=(V,E)的图拟阵：S=E（边集），I={不包含环的边子集}。

- 遗传性：无环边集的子集也无环 ✓
- 交换性：两个森林，大的可以向小的"借"一条边 ✓

在加权图拟阵上，贪心算法 = Kruskal算法。

### 5.6.4 加权拟阵的贪心算法

```
Greedy(M, w):
    将S中元素按权重降序排列
    A = ∅
    for each x in S:
        if A ∪ {x} ∈ I:
            A = A ∪ {x}
    return A
```

**定理**：对任意加权拟阵，上述贪心算法返回最优（最大权重）独立子集。

### 5.6.5 任务调度问题的拟阵化

**任务调度问题**：n个任务，每个有deadline dᵢ和惩罚wᵢ。求调度使超时任务的总惩罚最小。

**转换为拟阵**：独立集 = {可以在deadline前全部完成的任务子集}。

验证遗传性和交换性，从而贪心（按惩罚降序选择）最优。

# 第6章 树搜索策略

## 6.1 生活类比：走迷宫

你在一个迷宫入口。怎么找到出口？有三种基本策略：

- **BFS（广度优先）**：从入口开始，一层一层向外探索（像水波扩散）。**保证找到最短路径**。
- **DFS（深度优先）**：选一条路走到黑，碰壁再回头。**不一定最短，但省内存**。
- **回溯**：DFS + 剪枝——走到一半发现这条路没希望，直接回头，不浪费时间。

### 6.2 三类搜索的精确对比

| 维度 | BFS | DFS | 回溯 |
|------|-----|-----|------|
| 数据结构 | 队列（Queue） | 栈（Stack）/递归 | 递归 + 剪枝条件 |
| 空间复杂度 | O(b^d) 很费内存 | O(d) 省内存 | O(d) |
| 最优性 | 保证最优（无权图） | 不保证 | 不保证（但能找到所有解） |
| 典型场景 | 最短路径、社交网络度数 | 拓扑排序、连通分量 | N皇后、数独、子集和 |

## 6.3 爬山法（Hill Climbing）

### 6.3.1 生活类比：蒙眼爬山

你蒙着眼站在山里，目标是最高峰。你能做的只是：摸一下四周，感觉哪个方向更高就往哪走一步。重复，直到你感觉四周都比你低。

问题：你停在了小山丘顶——它不是珠穆朗玛，但你已经"卡住"了。这就是**局部最优陷阱**。

### 6.3.2 8-Puzzle 实例

**状态表示**：3x3 棋盘，8个数字 + 1空格。目标状态固定（通常是 1-2-3 / 4-5-6 / 7-8-空）。

**启发函数W(n)**：不在目标位置的数字个数。

初始状态：
```
2 8 3
1 6 4
7   5
```
目标状态：
```
1 2 3
8   4
7 6 5
```

W(初始) = 统计位置不正确的数字个数 = …（逐位比较）

**爬山法执行**：
1. 生成当前状态的所有可能移动（空格上下左右）
2. 计算每个邻居的W值
3. 移动到W最小的邻居（如果都比当前大则停止）
4. 重复直到 W=0（目标）或卡住

### 6.3.3 随机重启爬山法

多次从不同的随机起点出发（或从当前局部最优做随机扰动后再爬），大大增加找到全局最优的概率。

## 6.4 Best-First 搜索

**核心**：用**优先队列**代替BFS的普通队列。每次扩展 `f(n)` 最小的节点。

如果 f(n) = h(n)（只考虑启发式估计）→ 贪心最佳优先（不一定最优但很快）

## 6.5 分支界限法（Branch and Bound）

### 6.5.1 整体框架

分支界限 = 回溯 + 下界剪枝

```
BranchBound():
    当前最优 = ∞
    搜索树：
        对每个节点：
            计算下界 LB
            如果 LB >= 当前最优 → 剪枝（不再扩展此分支）
            否则 → 继续扩展
```

### 6.5.2 人员分配问题

**问题**：n个人分配n个任务，C[i][j] = 人i做任务j的代价。最小化总代价。

**下界计算**（必考）：
1. **行减**：每行减去该行最小值
2. **列减**：在行减后的矩阵上，每列减去该列最小值
3. **下界 = 行减总和 + 列减总和**

**📝 手算例题**：

代价矩阵：
```
任务1 任务2 任务3 任务4
人1:  9     2     7     8
人2:  6     4     3     7
人3:  5     8     1     8
人4:  7     6     9     4
```

步骤1 - 行减：
```
人1: min=2 → [7, 0, 5, 6]
人2: min=3 → [3, 1, 0, 4]
人3: min=1 → [4, 7, 0, 7]
人4: min=4 → [3, 2, 5, 0]
行减总和 = 2+3+1+4 = 10
```

步骤2 - 列减（在行减后的矩阵上）：
```
列1: min=3 → [4, 0, 1, 0]  (每行-min)
列2: min=0 → 不变
列3: min=0 → 不变
列4: min=0 → 不变
列减总和 = 3
```

**总下界 = 10+3 = 13**

**分支**：选人1做任务2（代价2是矩阵中最小的）→ 子矩阵去掉第1行第2列 → 递归计算下界。

### 6.5.3 TSP 分支界限（🔥🔥 必考）

**问题**：n个城市，距离矩阵D[i][j]，找最短Hamilton回路（每个城市恰好访问一次并回到起点）。

**下界计算**（两种方法）：

**方法A（最小出边法）**：
下界 = Σ_{每个城市}（该城市最小的两条出边之和）/ 2

**方法B（归约法，更常用）**：
与人员分配问题类似：行减 + 列减 → 下界 = 归约总和。

**📝 完整手算**：

距离矩阵（5个城市）：
```
    A   B   C   D   E
A:  ∞  20  30  10  11
B:  15  ∞  16   4   2
C:   3   5  ∞   2   4
D:  19   6  18  ∞   3
E:  16   4   7  16  ∞
```

**行减**：
```
A: min=10 → [∞, 10, 20, 0, 1]
B: min=2  → [13, ∞, 14, 2, 0]
C: min=2  → [1, 3, ∞, 0, 2]
D: min=3  → [16, 3, 15, ∞, 0]
E: min=4  → [12, 0, 3, 12, ∞]
行减总和 = 10+2+2+3+4 = 21
```

**列减**（在行减后的矩阵上）：
```
列A: min=1 → [∞, 12, 0, 15, 11]
列B: min=0 → 不变
列C: min=3 → [17, 11, ∞, 12, 0]
列D: min=0 → 不变
列E: min=0 → 不变
列减总和 = 1+3 = 4
```

**总下界 = 21+4 = 25**。这意味着任何完整回路的总距离至少是 25。

**分支策略**：通常选择使下界增加最多的边进行分支（"包含该边"和"不包含该边"两个子节点）。

例如包含边 (D,E) 后，D行E列划去，同时禁止 E→D（避免子回路），重新归约计算新下界。

### 6.5.4 ⚠️ 常见坑

- 分支界限的"下界"必须**真实下界**（不能高估），否则可能剪掉最优解
- TSP归约法中禁止某边时用 M=∞ 处理即可

## 6.6 A* 算法

### 6.6.1 核心公式

```
f(n) = g(n) + h(n)
```

- g(n)：从起点到节点n的**实际代价**
- h(n)：从节点n到目标的**启发式估计代价**
- f(n)：经过n的路径的总估计代价

### 6.6.2 可采纳性定理及证明（🔥🔥🔥 必考）

**定义**：h(n) 是**可采纳的**（admissible），当且仅当对所有节点 n，h(n) ≤ h*(n)，其中 h*(n) 是从 n 到目标的实际最小代价。

**定理**：如果 h 可采纳，则 A* 返回最优解。

**完整证明**：

设 G 为最优目标节点，G' 为 A* 实际返回的目标节点。设 C* = g(G) 为最优解的实际代价。

我们要证明 g(G') = C*。

**步骤 1**：证明 f(G) ≤ C*。
```
f(G) = g(G) + h(G) = C* + 0 = C*    （因为 h(G)=0 对任何目标节点）
```
所以 f(G) = C*。

**步骤 2**：证明在 A* 终止前，openSet 中总存在一个节点在最优路径上且其 f 值 ≤ C*。

设最优路径为 n₀(起点)→n₁→...→nₖ=G。令 n' 为 openSet 中最优路径上最靠近起点的那个节点。

由于 h 可采纳：f(n') = g(n') + h(n') ≤ g(n') + h*(n') = C*

所以 openSet 中总存在 f 值 ≤ C* 的节点。

**步骤 3**：A* 总是扩展 f 值最小的节点。因此当 G' 被扩展时（成为当前节点），
```
f(G') ≤ (openSet 中任何其他节点的 f 值) ≤ C*
```
所以 f(G') ≤ C*。

**步骤 4**：又因为 h(G') = 0（G' 是目标），g(G') = f(G') - h(G') = f(G') ≤ C*。

但 C* 是最小代价，所以 g(G') ≥ C*。

综合得 g(G') = C*。∎

### 6.6.3 一致性条件

**定义**：h 是**一致的**，如果 ∀n,n'（n'是n的后继）：h(n) ≤ c(n,n') + h(n')，且 h(goal) = 0。

**意义**：
- 一致性 ⇒ 可采纳性（但反过来不成立）
- 一致性保证 A* 不会重复扩展节点
- 一致性保证 f 值沿路径单调不减

# 第7章 随机算法

## 7.1 生活类比：抛硬币

当你面对两个选择无法判断哪个更好时，抛硬币决定——这就是最简单的随机算法。虽然这次可能选错，但长期来看（重复多次）结果会趋近于正确。

## 7.2 四类随机算法

| 类型 | 正确性 | 运行时间 | 典型例子 |
|------|--------|----------|----------|
| **Las Vegas** | 总是正确 | 随机变化 | 随机快排（期望 O(n log n)） |
| **Monte Carlo** | 可能错误 | 固定 | 随机最小割、素数测试 |
| **Sherwood** | 总是正确 | 消除最坏情况 | 随机化快排（比固定pivot更好） |
| **随机数值** | 近似 | 固定 | 蒙特卡洛积分、π 的估计 |

### ⚠️ 常见坑

很多同学混淆 Las Vegas 和 Monte Carlo：
- Las Vegas：结果一定对，但运气不好可能很慢
- Monte Carlo：速度一定快，但结果可能不对
- Sherwood：本质是"随机化输入顺序"来消除最坏情况

## 7.3 随机快速排序

### 7.3.1 与普通快排的区别

**普通快排**：pivot = A[r]（最后一个元素）→ 最坏 O(n²)

**随机快排**：pivot = 随机选取 → 期望 O(n log n)，不受输入分布影响

### 7.3.2 期望复杂度完整推导（🔥🔥 必考）

设 A 排序后为 z₁, z₂, ..., zₙ。定义指示变量：

**Xᵢⱼ = 1** 当且仅当 zᵢ 和 zⱼ 在快速排序执行过程中被比较过（i < j）。

**关键观察**：zᵢ 和 zⱼ 被比较 ⇔ 在集合 {zᵢ, zᵢ₊₁, ..., zⱼ} 中，第一个被选为 pivot 的恰好是 zᵢ 或 zⱼ。

原因：如果第一个 pivot 不是 zᵢ 或 zⱼ，而是中间的某个 zₖ（i < k < j），则 zᵢ 和 zⱼ 会被划分到不同的子数组，永远不再比较。

**计算 P(Xᵢⱼ = 1)**：
集合 {zᵢ, ..., zⱼ} 有 j-i+1 个元素，每个元素被选为第一个 pivot 的概率相等。两个"好"的 pivot（zᵢ 和 zⱼ）带来比较，所以：

```
P(Xᵢⱼ = 1) = 2 / (j-i+1)
```

**期望比较次数**：

```
E[T] = Σ_{i=1}^{n-1} Σ_{j=i+1}^{n} P(Xᵢⱼ = 1)
     = Σ_{i=1}^{n-1} Σ_{j=i+1}^{n} 2/(j-i+1)
```

令 k = j-i+1（两个元素之间的距离+1）：

```
E[T] = Σ_{i=1}^{n-1} Σ_{k=2}^{n-i+1} 2/k
     ≤ Σ_{i=1}^{n-1} Σ_{k=1}^{n} 2/k
     = (n-1) · 2Hₙ
     = O(n log n)
```

其中 Hₙ = Σ(1/k) ≈ ln n + γ 是调和数。

### 7.3.3 📝 评分重点

考试中要写清楚"为什么 P = 2/(j-i+1)"——必须说明"当且仅当 zᵢ 或 zⱼ 是第一个 pivot"这个关键观察。

## 7.4 随机最小割（Karger 算法）

### 7.4.1 问题

**输入**：无向连通图 G = (V, E)
**输出**：最小割——删除最少边数使图不连通

### 7.4.2 算法

```
Karger(G):
    while |V| > 2:
        从 E 中随机选择一条边 e = (u, v)
        收缩边 e：合并 u 和 v（合为超节点，保留所有连边）
    返回剩余的超节点之间的边数（= 割的大小）
```

### 7.4.3 成功概率分析（🔥🔥 必考定理及证明）

设图 G 的最小割有 k 条边。

**定理**：单次 Karger 算法找到最小割的概率 ≥ 2 / [n(n-1)]。

**证明**（逐步推导）：

设算法有 n-2 轮收缩（从 n 个节点缩到 2 个）。

设 Eᵢ 为事件"第 i 轮收缩没有选中最小割中的边"。

我们要计算：P(成功) = P(E₁ ∩ E₂ ∩ ... ∩ E_{n-2})

**关键分析**：在第 i 轮时：
- 剩余节点数：n - i + 1
- 每个超节点度数至少为 k（因为最小割为 k，每个超节点至少连出 k 条边）
- 所以当前图中至少 k(n-i+1)/2 条边（握手引理）
- 最小割中还有 k 条边在当前图中被动过吗？实际上，收缩操作不影响最小割边集——只要没选中它们

因此：
```
P(第i轮选中最小割边 | 前i-1轮都没选) ≤ k / [k(n-i+1)/2] = 2/(n-i+1)
```

所以：
```
P(成功) = Π_{i=1}^{n-2} (1 - 2/(n-i+1))
        = Π_{j=3}^{n} (1 - 2/j)         （令 j = n-i+1）
        = Π_{j=3}^{n} (j-2)/j
        = (1×2×3×...×(n-2)) / (3×4×5×...×n)
        = (1×2) / ((n-1)×n)
        = 2 / [n(n-1)]
```

∎

### 7.4.4 提高成功率

重复运行 T 次，取最小割。单次失败概率 ≤ 1 - 2/n²。

重复 T = n² ln n 次：
P(全部失败) = (1 - 2/n²)^{n² ln n} ≈ e^{-2 ln n} = 1/n² → 0

所以以高概率找到最小割。总时间复杂度：O(T × m) = O(m n² log n)。

## 7.5 随机 MAX-3-CNF

### 7.5.1 问题

**输入**：3-CNF 公式（每个子句恰好3个文字），共 m 个子句
**输出**：变量赋值，使尽可能多的子句被满足

### 7.5.2 简单随机算法

```
Random-Max-3CNF:
    对每个变量 x：
        以 1/2 概率设为 True，1/2 概率设为 False
    返回此赋值
```

### 7.5.3 期望近似比分析

每个子句有 3 个文字。随机赋值下：
- **子句被满足的概率** = 1 - (1/2)³ = 1 - 1/8 = 7/8
- 期望满足子句数 = 7m/8
- 最优解最多满足 m 个子句

**定理**：上述随机算法的期望近似比为 7/8。

**证明**：设 OPT = 最优解满足的子句数 ≤ m。则：
```
E[ALG] / OPT ≥ (7m/8) / m = 7/8
```

**引申**：存在确定性算法可以达到 3/4 近似比。7/8 是 MAX-3SAT 的经典近似比。

## 7.6 LazySelect 算法与概率工具

### 7.6.1 Markov 不等式

若 X ≥ 0，则对任意 a > 0：
```
P(X ≥ a) ≤ E[X] / a
```

### 7.6.2 Chebyshev 不等式

```
P(|X - E[X]| ≥ a) ≤ Var(X) / a²
```

### 7.6.3 在算法分析中的应用

常用于证明"以高概率不超过某个界"。例如：
- 随机快排以高概率在 O(n log n) 完成
- 随机最小割重复 O(n² log n) 次以高概率成功

# 第8章 近似算法

## 8.1 生活类比：航拍地图拍摄

你想给整个城市拍一张高清航拍。如果要求每个建筑都完美无瑕，可能需要从100个角度拍摄（=求出精准最优解）。但如果接受"建筑细节可能有点模糊，但整体布局清楚"（=近似最优），可能只拍10个角度就够了。

## 8.2 核心概念

### 8.2.1 Ratio Bound（近似比）

**最小化问题**（如顶点覆盖、TSP）：
```
ρ(n) = A(I) / OPT(I) ≥ 1
```
算法的解不会比最优解差超过 ρ(n) 倍。

**最大化问题**（如 MAX-3CNF）：
```
ρ(n) = OPT(I) / A(I) ≥ 1
```

### 8.2.2 相对误差

```
ε(n) ≤ ρ(n) - 1
```

### 8.2.3 PTAS 与 FPTAS

| 缩写 | 全称 | 含义 |
|------|------|------|
| PTAS | Polynomial-Time Approximation Scheme | 对任意 ε>0，存在 (1+ε)-近似算法，但运行时间可能是 n^{1/ε} 这种 |
| FPTAS | Fully PTAS | 对任意 ε>0，存在 (1+ε)-近似算法，且运行时间是 n 和 1/ε 的多项式 |

例如：0/1 背包有 FPTAS，但一般 TSP 没有 PTAS（除非 P=NP）。

## 8.3 顶点覆盖 2-近似

### 8.3.1 问题定义

**输入**：无向图 G = (V, E)
**输出**：最小顶点覆盖 C ⊆ V（每条边至少有一个端点在 C 中）

### 8.3.2 算法

```
Approx-Vertex-Cover(G):
    C = ∅
    E' = E
    while E' ≠ ∅:
        任取一条边 (u, v) ∈ E'
        C = C ∪ {u, v}
        从 E' 中移除所有与 u 或 v 关联的边
    return C
```

### 8.3.3 近似比证明（🔥🔥 必考）

**定理**：Approx-Vertex-Cover 是 2-近似算法。

**证明**：

设 A 为算法选出的边集（算法每步取的那条边）。A 中的边两两不共享端点（因为选了 (u,v) 后删除了 u 和 v 的所有关联边）。所以 A 是一个**匹配**。

对于 A 中每条边，最优解 C* 至少需要选其中一个端点 → |C*| ≥ |A|。

算法输出 |C| = 2|A|。

因此：
```
|C| = 2|A| ≤ 2|C*|
```

近似比 ≤ 2。∎

### 8.3.4 📝 关键理解

这个证明的美妙之处在于构造了"算法选择的匹配大小"和"最优覆盖大小"之间的桥梁。匹配的每条边至少贡献一个最优覆盖顶点。

### 8.3.5 紧的例子

完全二分图 K_{n,n}：算法选 2n 个顶点（尽量全部），最优只需 n 个（取一边），近似比 = 2n/n = 2。所以界是紧的。

## 8.4 装箱问题 First-Fit

### 8.4.1 问题

**输入**：n 个物品，每个大小 aᵢ ∈ (0, 1]
**输出**：用最少的容量为 1 的箱子装下所有物品

### 8.4.2 First-Fit 算法

按顺序处理每个物品，把物品放入第一个能容纳它的箱子。如果没有箱子能放，开新箱。

### 8.4.3 近似比

**定理**：First-Fit 的近似比 ≤ 2。即 First-Fit(I) ≤ 2·OPT(I)。

**证明思路**：如果 First-Fit 使用了 m 个箱子，则前 m-1 个箱子的总装入量 > (m-1)/2。否则一半容量的空隙可以被利用。

更精确：至少 (m-1) 个箱子装入量 > 1/2。

因此所有物品总大小 > (m-1)/2，即 m-1 < 2∑aᵢ。
又因为 OPT ≥ ∑aᵢ（最优也需要至少总大小个箱子），
所以 m-1 < 2·OPT → m ≤ 2·OPT。

### 8.4.4 更紧的界

实际分析可得 First-Fit ≤ ⌈1.7·OPT⌉。但考试中知道 ≤ 2 即可。

## 8.5 TSP 三角不等式 2-近似

### 8.5.1 条件

TSP 满足三角不等式：对任意三个城市 u, v, w，有 d(u,v) ≤ d(u,w) + d(w,v)。

**如果没有三角不等式**：一般 TSP 没有常数近似比（除非 P=NP）。

### 8.5.2 算法

```
Approx-TSP-Tour(G):
    1. 计算 G 的最小生成树 MST T
    2. 对 T 进行 Full Walk：每个节点重复，每条边走两次
       → 得到一个 Euler 回路
    3. 对 Euler 回路的顶点序列，跳过重复访问的顶点（shortcut）
    4. 返回结果序列作为 Hamilton 回路
```

### 8.5.3 近似比证明

**步骤 1**：最优 TSP 回路去掉一条边 → 得到一棵生成树。
∴ w(T) ≤ w(最优回路)

**步骤 2**：Full Walk 代价 = 2·w(T)

**步骤 3**：Shortcut 利用三角不等式 → 代价不超过 Full Walk。

∴ 算法代价 ≤ 2·w(最优回路)。近似比 ≤ 2。

### 8.5.4 Christofides 算法（1.5-近似）

改进：在 MST 的奇度节点上找最小权完美匹配 → 得到 1.5-近似。

## 8.6 集合覆盖贪心近似

### 8.6.1 问题

**输入**：全集 X，子集族 F（每个 S∈F 是 X 的子集），代价函数 c: F → R⁺
**输出**：覆盖 X 的最小代价子集族

### 8.6.2 贪心算法

```
Greedy-Set-Cover(X, F):
    U = X
    C = ∅
    while U ≠ ∅:
        选择 S∈F 使 c(S)/|S∩U| 最小（单位覆盖成本最低）
        U = U - S
        C = C ∪ {S}
    return C
```

### 8.6.3 近似比

**定理**：贪心集合覆盖的近似比为 H(max|S|)，其中 H(d) = 1 + 1/2 + ... + 1/d ≤ ln d + 1。

**证明思路**（不要求完整，理解关键步骤）：
- 将 OPT 的代价分散到每个元素
- 证明贪心每步选择的"单位代价"不超过最优的单位代价
- 调和级数 H(|S|) 来自覆盖剩余元素的过程

## 8.7 最小完工时间调度

### 8.7.1 问题

**输入**：m 台相同的机器，n 个任务（任务 j 需要时间 pⱼ）
**输出**：调度使最大完工时间（makespan）最小

### 8.7.2 Graham 算法（List Scheduling）

按任意顺序处理任务。每次把下一个任务分配给当前负载最小的机器。

**近似比**：2 - 1/m。

# 第9章 在线算法

## 9.1 生活类比：排队结账

你在超市选了一个收银台排队。你看不到每条队伍的实际等待时间——只能根据当前信息（眼前排的人数、购物车满不满）做决定。一旦选了就定了（不能后悔）。

这就是在线算法："边看边决策"，没有全局信息。

## 9.2 核心概念

### 9.2.1 在线 vs 离线

| 维度 | 在线算法 | 离线算法 |
|------|----------|----------|
| 输入 | 逐步到达，逐次决策 | 一次性获得全部输入 |
| 决策 | 不可撤销 | 全局优化 |
| 性能 | 用竞争比衡量 | 用最优值衡量 |

### 9.2.2 竞争比（Competitive Ratio）

```
若存在常数 c 和 b，使得对任意输入序列 σ：
    C_ON-LINE(σ) ≤ c · C_OFF-LINE(σ) + b

则称在线算法是 c-竞争的（c-competitive）。
```

**直观理解**：在线算法的代价不会超过离线最优的 c 倍（再加上一个常数）。

## 9.3 在线 MST 问题

### 9.3.1 问题

**输入**：图的节点逐次到达，每次新节点到达时连带它到之前所有节点的边权重
**输出**：逐步构建最小生成树（每次新节点到达时更新）

### 9.3.2 贪心在线算法

每次新节点 v 到达时：连接 v 到当前生成树中权重最小的那条边。

### 9.3.3 竞争比分析（🔥 核心引理）

**引理**：在线算法生成的 MST 中，第 k 长的边 ≤ 2L/k，其中 L 是离线最优 MST 的总权重。

**直觉**：离线 MST 的 k 条最长边之和 ≤ L（因为总权重是 L）。所以每条长边平均 ≤ L/k。在线算法的每条长边不超过这个平均值的两倍。

**竞争比**：由引理推导得 O(log n)。

## 9.4 随机在线 MST

R(m) 算法：不总是连最近点，而是以一定概率随机探索。

竞争比：θ(n)。

## 9.5 凸包在线近似

在线算法用正 m 边形近似凸包，每条边方向对齐到最近的 m 等分方向。

**近似误差**：
```
ε ≤ sec(π/2m) - 1
```
当 m 增大时，误差趋近于 0。

# 专题：贪心 vs 动态规划 —— 终极选择指南

## S.1 这个专题为什么重要

这是考试中最致命的陷阱。看到一个选择题/大题，你不知道该用贪心还是DP。选错了，整道题全错。

## S.2 四步判断法

```
Step 1: 问题是否具有"优化子结构"？
        否 → 不是DP也不是贪心（可能是搜索/近似）
        是 → 继续

Step 2: 问题是否具有"贪心选择性质"？
        验证方法：交换论证法（下面详述）
        是 → 贪心可解！
        无法验证/不确定 → 继续

Step 3: DP 尝试用贪心，失败了吗？
        经典反例：0/1 背包问题
        - 贪心（按价值密度）：可能失败
        - DP：总是正确

Step 4: 如果贪心可以，需要证明！
        交换论证法 / 拟阵理论
```

## S.3 交换论证法模板（🔥🔥🔥 必背）

这是证明"贪心选择性质"的通用方法：

```
要证明：贪心选择 x 必在某个最优解中。

步骤1: 设 A* 是一个最优解

步骤2: 若 x ∈ A* → 证毕

步骤3: 若 x ∉ A*：
        - 在 A* 中找一个元素 y，使得 y 与 x 具有相似地位
          （例如：活动选择中 y 是 A* 中最早结束的那个；
                 MST 中 y 是最优树上横跨同一条割的边）
        - 构造 A' = (A* - {y}) ∪ {x}
        
步骤4: 证明 A' 也是可行解
        （利用 x 的"贪心性质"证明 x 不会与 A* 中的其他元素冲突）

步骤5: 证明 w(A') ≥ w(A*)（最大化问题）或 w(A') ≤ w(A*)（最小化问题）
        （利用 x 比 y "更好"或"不差"的性质）

步骤6: 故 A' 也是最优解，且包含 x。∎
```

## S.4 活动选择问题的交换论证法（完整背诵版）

**贪心选择**：选最早结束的活动（设为活动 1）。

**证明**：

设 A* = {a_k₁, a_k₂, ..., a_k_m} 是一个最优解，按结束时间排序。

**情况 1**：k₁ = 1。则 A* 包含活动 1，证毕。

**情况 2**：k₁ ≠ 1。

构造 A' = (A* - {a_k₁}) ∪ {a₁}。

**(A) 证明 A' 是可行解**：
- A* - {a_k₁} 中的所有活动互不相容（它们是 A* 的子集）
- 由于 a₁ 是所有活动中最早结束的：f₁ ≤ f_k₁
- a_k₁ 与 A* - {a_k₁} 相容 → 对于任何 a ∈ A* - {a_k₁}，有 s_a ≥ f_k₁ ≥ f₁
- 因此 a 的开始时间 ≥ f₁，a 不与 a₁ 冲突
- 故 A' 是可行解 ✓

**(B) 证明 |A'| = |A*|**：
|A'| = |A*| - 1 + 1 = |A*| ✓

**(C) 结论**：A' 是包含活动 1 的最优解。∎

## S.5 0/1 背包：贪心为什么失败（🔥 必考反例）

**物品**：物品1 (w=2, v=3, 密度=1.5)；物品2 (w=3, v=5, 密度≈1.67)；物品3 (w=4, v=6, 密度=1.5)

**容量 C = 8**。

**贪心（按价值密度降序）**：
① 物品2 (w=3, v=5) → 剩余容量 5
② 物品1 (w=2, v=3) → 剩余容量 3
③ 物品3 装不下
总价值 = 8

**DP 最优解**：物品1 + 物品3 = 2+4=6 ≤ 8，价值 = 3+6 = 9 > 8

贪心失败！因为选了密度最高的物品 2 后，剩下的容量刚好不够放下物品 3，而物品 1+3 的组合更好。

**结论**：0/1 背包**没有**贪心选择性质，必须用 DP。

## S.6 部分背包：贪心为什么成功

**与 0/1 背包的区别**：物品可以分割（如金粉可以分装）。

**贪心策略**：按价值密度降序，尽量拿满每个物品，最后一个可能只拿部分。

**为什么贪心成功**：拿金粉不存在"拿了2kg就放不下后面的3kg"的问题——可以拿 2.5kg。O(n log n)，排序主导。

## S.7 MST 安全边定理（完整背诵版）

**定义回顾**：
- **割 (S, V-S)**：顶点集的一个划分
- **边集 A 尊重割**：A 中没有边横跨此割
- **轻边**：横跨此割的最小权重边

**定理**：设 G = (V, E) 连通无向，A ⊆ E 包含在某棵 MST 中，(S, V-S) 是尊重 A 的任意割，(u, v) 是横跨此割的轻边，则 (u, v) 是 A 的安全边。

**证明**（交换论证）：

设 T 是包含 A 的一棵 MST。

**情况 1**：(u, v) ∈ T。则 T 即所求，证毕。

**情况 2**：(u, v) ∉ T。

因为 T 是生成树，加入 (u, v) 后 T ∪ {(u, v)} 包含恰好一个环 C。

由于 (u, v) 横跨割 (S, V-S)，环 C 上必有另一条边 (x, y) 也横跨此割（否则环无法从 S 到 V-S 再回到 S）。

因为 (u, v) 是轻边：w(u, v) ≤ w(x, y)。

构造 T' = (T - {(x, y)}) ∪ {(u, v)}。

- T' 仍是生成树（去掉 C 上的 (x, y)，打破环）✓
- w(T') = w(T) - w(x, y) + w(u, v) ≤ w(T)
- 又 T 是 MST，所以 w(T') = w(T)，T' 也是 MST ✓
- 由于 A 尊重割，(x, y) 不在 A 中，所以 A ⊆ T' ✓

故 (u, v) 是安全边。∎

# 考前必背公式清单

## 数学基础 (ch2)

```
θ(g): ∃c₁,c₂>0,n₀>0: ∀n≥n₀, c₁g(n) ≤ f(n) ≤ c₂g(n)
O(g):  ∃c>0,n₀>0:   ∀n≥n₀, 0 ≤ f(n) ≤ cg(n)
Ω(g):  ∃c>0,n₀>0:   ∀n≥n₀, 0 ≤ cg(n) ≤ f(n)
o(g):  ∀c>0, ∃n₀>0: ∀n≥n₀, 0 ≤ f(n) < cg(n)
ω(g):  ∀c>0, ∃n₀>0: ∀n≥n₀, 0 ≤ cg(n) < f(n)
```

极限法：L = lim f(n)/g(n) → L=0⇒o/O, L∈(0,∞)⇒θ, L=∞⇒ω/Ω

Master 定理（T(n)=aT(n/b)+f(n)）：
```
情况1: f(n)=O(n^(log_b a - ε)) → T(n)=θ(n^(log_b a))
情况2: f(n)=θ(n^(log_b a))     → T(n)=θ(n^(log_b a)·log n)
情况3: f(n)=Ω(n^(log_b a + ε)) + 正则条件 → T(n)=θ(f(n))
```

## 分治 (ch3)

```
Karatsuba:      T(n)=3T(n/2)+O(n) → θ(n^log₂3)≈θ(n^1.59)
最近点对:       T(n)=2T(n/2)+O(n) → θ(n log n)
中位数之中位数: T(n)≤T(⌈n/5⌉)+T(7n/10)+O(n) → θ(n)
快排期望:       θ(n log n)
```

Partition 不变量（四区域）：[p..i]≤x, [i+1..j-1]>x, [j..r-1]未处理, [r]=x

## DP (ch4)

```
LCS:  C[i,j] = C[i-1,j-1]+1            (xᵢ=yⱼ)
              = max(C[i-1,j],C[i,j-1])  (xᵢ≠yⱼ)

矩阵链: m[i,j] = min_{i≤k<j} {m[i,k]+m[k+1,j]+p_{i-1}p_kp_j}

0/1背包: m(i,j) = max( 不选:m(i+1,j),  选:m(i+1,j-wᵢ)+vᵢ  )

OBST: E(i,j) = min_{i≤r≤j} {E(i,r-1)+E(r+1,j)+W(i,j)}
```

## 贪心 (ch5)

活动选择贪心策略：按 fᵢ 递增排序，每次选最早结束的相容活动

交换论证法模板：
```
① 设 A* 是最优解
② 若贪心选择 x∈A* → 证毕
③ 若否，造 A'=(A*-{y})∪{x}
④ 证 A' 可行（利用 f_x≤f_y 或 w(x)≤w(y)）
⑤ |A'|=|A*| → A' 也是最优解 ∎
```

安全边定理：轻边横跨尊重 A 的割 → 安全边
