# 第四章 动态规划算法的设计与分析原理 —— 复习笔记

------

## 一、动态规划算法总论（4.1）

### 1.1 研究动机（Why）

**分治法的问题：** 当子问题**不是相互独立**的时候，分治法会**重复计算公共子问题**，效率很低。

> **核心思想：** 用**空间换时间**——将已求解的子问题结果保存起来，避免重复计算。

### 1.2 适用范围（What）

动态规划适用于同时满足以下两个条件的**优化问题**：

| 条件                                      | 含义                                     |
| ----------------------------------------- | ---------------------------------------- |
| **最优子结构（Optimal Substructure）**    | 问题的最优解**包含**了子问题的最优解     |
| **重叠子问题（Overlapping Subproblems）** | 求解过程中，很多子问题的解**被多次使用** |

### 1.3 算法设计步骤（How）—— 五步法

```
① 分析优化解的结构（划分子问题、验证最优子结构、子问题重叠性）
② 递归地定义最优解的代价（建立递归方程）
③ 递归地划分问题，直至不可划分
④ 自底向上求解各个子问题：
   - 计算优化解代价并保存
   - 获取构造最优解的信息
⑤ 根据构造最优解的信息构造优化解
```

### 1.4 实例说明：最短路径问题

给定图 S→{A,B,C}→{D,E,F}→T：

```
d(S,T) = min{1+d(A,T), 2+d(B,T), 5+d(C,T)}
```

- **自底向上**先求 d(D,T)、d(E,T)、d(F,T)
- 再求 d(A,T)、d(B,T)、d(C,T)
- 最后求 d(S,T)

## 二、最长公共子序列（LCS）（4.2）

### 2.1 问题定义

- **子序列：** 从序列中**按原顺序**（可不连续）取出若干元素组成的序列
- **公共子序列：** Z 既是 X 的子序列，也是 Y 的子序列
- **LCS问题：**
  - **输入：** X = (x₁, x₂, ..., xₘ)，Y = (y₁, y₂, ..., yₙ)
  - **输出：** X 与 Y 的**最长**公共子序列 Z = (z₁, z₂, ..., zₖ)

**记号：** Xᵢ = (x₁, ..., xᵢ) 称为 X 的**第 i 前缀**

### 2.2 最优子结构

**定理（最优子结构）：** 设 X=(x₁,...,xₘ)、Y=(y₁,...,yₙ)，LCS_XY = (z₁,...,zₖ) 是 X 与 Y 的 LCS，则：

| 情况                 | 结论                                                         |
| -------------------- | ------------------------------------------------------------ |
| ⑴ xₘ = yₙ            | zₖ = xₘ = yₙ，且 LCS_{X_{m-1}Y_{n-1}} 是 X_{m-1} 和 Y_{n-1} 的 LCS |
| ⑵ xₘ ≠ yₙ 且 zₖ ≠ xₘ | LCS_XY 是 X_{m-1} 和 Y 的 LCS                                |
| ⑶ xₘ ≠ yₙ 且 zₖ ≠ yₙ | LCS_XY 是 X 和 Y_{n-1} 的 LCS                                |

**证明思路（反证法）：**

- ⑴ 若 zₖ ≠ xₘ，则可将 xₘ=yₙ 追加到 Z 末尾得到长度 k+1 的公共子序列，与 Z 是 LCS 矛盾。
- ⑵ 由 zₖ ≠ xₘ 知 Z 也是 X_{m-1} 与 Y 的公共子序列；若存在更长的，则它也是 X 与 Y 的公共子序列，矛盾。

**子问题重叠性：** LCS_{X_{m-1}Y} 和 LCS_{XY_{n-1}} 都可能包含 LCS_{X_{m-1}Y_{n-1}}，即子问题被重复计算。

### 2.3 递归方程

设 **C[i, j]** = Xᵢ 与 Yⱼ 的 LCS 的长度：

$$
C[i,j] = \begin{cases} 0 & \text{if } i=0 \text{ 或 } j=0 \ C[i-1,j-1]+1 & \text{if } i,j>0 \text{ 且 } x_i = y_j \ \max(C[i,j-1],\ C[i-1,j]) & \text{if } i,j>0 \text{ 且 } x_i \neq y_j \end{cases}
$$

**计算依赖关系：**

```
C[i-1,j-1]   C[i-1,j]
C[i,j-1]     C[i,j]  ← 由左上、上、左三个值决定
```

### 2.4 算法

**数据结构：**

- `C[0:m, 0:n]`：C[i,j] 存储 Xᵢ 与 Yⱼ 的 LCS 长度
- `B[1:m, 1:n]`：B[i,j] 记录构造最优解的方向信息（↖、↑、←）

```
LCS-length(X, Y)
  m ← length(X);  n ← length(Y)
  For i=0 To m Do  C[i,0] ← 0
  For j=1 To n Do  C[0,j] ← 0
  For i=1 To m Do
    For j=1 To n Do
      If x_i = y_j Then
        C[i,j] ← C[i-1,j-1] + 1;  B[i,j] ← "↖"
      Else If C[i-1,j] ≥ C[i,j-1] Then
        C[i,j] ← C[i-1,j];  B[i,j] ← "↑"
      Else
        C[i,j] ← C[i,j-1];  B[i,j] ← "←"
  Return C and B
```

### 2.5 构造最优解

从 B[m,n] 开始按指针回溯：

- B[i,j] = "↖"：xᵢ = yⱼ 是 LCS 的一个元素，递归到 (i-1, j-1)
- B[i,j] = "↑"：递归到 (i-1, j)
- B[i,j] = "←"：递归到 (i, j-1)

```
Print-LCS(B, X, i, j)
  If i=0 or j=0 Then Return
  If B[i,j] = "↖" Then
    Print-LCS(B, X, i-1, j-1);  Print x_i
  Else If B[i,j] = "↑" Then
    Print-LCS(B, X, i-1, j)
  Else
    Print-LCS(B, X, i, j-1)
```

### 2.6 实例演示

X = (A, B, C, B, D, A, B)，Y = (B, D, C, A, B, A)

填表结果（C 数组）：

|      | ∅    | B    | D    | C    | A    | B    | A    |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| ∅    | 0    | 0    | 0    | 0    | 0    | 0    | 0    |
| A    | 0    | 0    | 0    | 0    | 1    | 1    | 1    |
| B    | 0    | 1    | 1    | 1    | 1    | 2    | 2    |
| C    | 0    | 1    | 1    | 2    | 2    | 2    | 2    |
| B    | 0    | 1    | 1    | 2    | 2    | 3    | 3    |
| D    | 0    | 1    | 2    | 2    | 2    | 3    | 3    |
| A    | 0    | 1    | 2    | 2    | 3    | 3    | 4    |
| B    | 0    | 1    | 2    | 2    | 3    | 4    | 4    |

LCS 长度 = C[7,6] = 4，LCS = (B, C, B, A)

### 2.7 复杂度分析

| 项目             | 复杂度                   |
| ---------------- | ------------------------ |
| 计算代价时间     | O(mn)（两层循环）        |
| 构造最优解时间   | O(m+n)（沿对角线回溯）   |
| **总时间复杂度** | **O(mn)**                |
| **空间复杂度**   | **O(mn)**（数组 C 和 B） |

## 三、矩阵链乘法（4.3）

### 3.1 问题定义

- **输入：** 矩阵链 ⟨A₁, A₂, ..., Aₙ⟩，其中 Aᵢ 是 pᵢ₋₁ × pᵢ 的矩阵
- **输出：** 确定计算 A₁A₂...Aₙ 的**最小乘法次数**的加括号方式
- **代价定义：** 若 A 是 p×q 矩阵，B 是 q×r 矩阵，则 AB 需要 **p·q·r** 次标量乘法

**示例：** A₁(10×100)，A₂(100×5)，A₃(5×50)

| 计算顺序 | 代价                              |
| -------- | --------------------------------- |
| (A₁A₂)A₃ | 10×100×5 + 10×5×50 = **7,500**    |
| A₁(A₂A₃) | 100×5×50 + 10×100×50 = **75,000** |

> 差异达 **10倍**！

**解空间大小：** p(n) = C(n-1) = Catalan数 = Ω(4ⁿ/n^{3/2})，穷举不可行。

### 3.2 最优子结构

**定理：** 若计算 A₁ₙ 的最优顺序在位置 k 处断开，即 A₁ₙ = A₁ₖ · Aₖ₊₁ₙ，则：

- A₁ₖ 的计算顺序必为 A₁ₖ 的最优解
- Aₖ₊₁ₙ 的计算顺序必为 Aₖ₊₁ₙ 的最优解

**子问题重叠性：** 例如 A₁A₂A₃A₄ 的不同划分中，子问题 (A₃A₄)、(A₁A₂) 等被重复出现。

### 3.3 递归方程

设 **m[i, j]** = 计算 Aᵢⱼ 的最小乘法次数：

$$
m[i,j] = \begin{cases} 0 & \text{if } i=j \ \displaystyle\min_{i \le k < j}{m[i,k] + m[k+1,j] + p_{i-1} \cdot p_k \cdot p_j} & \text{if } i<j \end{cases}
$$

**直觉理解：**

- 在位置 k 处断开：(Aᵢ...Aₖ)(Aₖ₊₁...Aⱼ)
- Aᵢₖ 是 pᵢ₋₁×pₖ 矩阵，Aₖ₊₁ⱼ 是 pₖ×pⱼ 矩阵
- 合并代价 = pᵢ₋₁·pₖ·pⱼ

### 3.4 自底向上计算

**按链长 l 递增的顺序计算（l = j - i + 1）：**

```
                m[1,5]              ← l=5
            m[1,4]  m[2,5]          ← l=4
        m[1,3]  m[2,4]  m[3,5]     ← l=3
    m[1,2]  m[2,3]  m[3,4]  m[4,5] ← l=2
m[1,1]  m[2,2]  m[3,3]  m[4,4]  m[5,5]  ← l=1 (全为0)
```

### 3.5 算法

```
Matrix-Chain-Order(p)
  n ← length(p) - 1
  For i=1 To n Do  m[i,i] ← 0          // 链长l=1
  For l=2 To n Do                        // 枚举链长
    For i=1 To n-l+1 Do                  // 枚举起点
      j ← i+l-1                          // 终点
      m[i,j] ← ∞
      For k=i To j-1 Do                  // 枚举断点
        q ← m[i,k] + m[k+1,j] + p_{i-1}·p_k·p_j
        If q < m[i,j] Then
          m[i,j] ← q;  s[i,j] ← k       // s记录最优断点
  Return m and s
```

### 3.6 构造最优解

```
Print-Optimal-Parens(s, i, j)
  If j = i Then Print "A"_i
  Else
    Print "("
    Print-Optimal-Parens(s, i, s[i,j])
    Print-Optimal-Parens(s, s[i,j]+1, j)
    Print ")"
```

调用 `Print-Optimal-Parens(s, 1, n)` 输出最优加括号方式。

### 3.7 复杂度分析

| 项目             | 复杂度                                     |
| ---------------- | ------------------------------------------ |
| 计算代价时间     | O(n³)（l, i, k 三层循环，每层至多 n-1 步） |
| 构造最优解时间   | O(n)                                       |
| **总时间复杂度** | **O(n³)**                                  |
| **空间复杂度**   | **O(n²)**（数组 m 和 s）                   |

## 四、凸多边形最优三角剖分（4.4）

### 4.1 问题定义

- **多边形：** P = (v₀, v₁, ..., vₙ)，n+1 个顶点
- **三角剖分：** 用**弦**（非相邻顶点连线）将多边形划分为不相交的三角形
- **优化三角剖分问题：**
  - **输入：** 简单多边形 P 和代价函数 W
  - **输出：** 使得所有三角形代价之和 ∑W(s) 最小的三角剖分

### 4.2 最优子结构

**定理：** 设 P=(v₀,...,vₙ) 的最优三角剖分 Tₚ 包含三角形 v₀vₖvₙ，则：

$$T_P = T(v_0,...,v_k) \cup T(v_k,...,v_n) \cup {v_0v_k, v_kv_n}$$

其中：

- T(v₀,...,vₖ) 是 P₁=(v₀,...,vₖ) 的**最优**三角剖分
- T(vₖ,...,vₙ) 是 P₂=(vₖ,...,vₙ) 的**最优**三角剖分

**子问题重叠性：** 不同划分方案共享相同的子多边形三角剖分。

### 4.3 递归方程

设 **t[i, j]** = ⟨vᵢ₋₁, vᵢ, ..., vⱼ⟩ 的最优三角剖分代价：

$$
t[i,j] = \begin{cases} 0 & \text{if } i=j \ \displaystyle\min_{i \le k < j}{t[i,k] + t[k+1,j] + w(\triangle v_{i-1}v_k v_j)} & \text{if } i<j \end{cases}
$$

### 4.4 与矩阵链乘法的关系

> **结构完全一致！** 只需将矩阵链乘法中的 pᵢ₋₁·pₖ·pⱼ 替换为 w(△vᵢ₋₁vₖvⱼ) 即可。

### 4.5 复杂度

| 项目           | 复杂度    |
| -------------- | --------- |
| **时间复杂度** | **O(n³)** |
| **空间复杂度** | **O(n²)** |

## 五、0/1 背包问题（4.5）

### 5.1 问题定义

- **输入：** n 种物品，背包承重 C；物品 i 重量 wᵢ，价值 vᵢ（均为正数）
- **输出：** 选择向量 (x₁, x₂, ..., xₙ)，xᵢ ∈ {0, 1}
  - 约束：∑wᵢxᵢ ≤ C
  - 目标：最大化 ∑vᵢxᵢ

**暴力方法：** 每个物品装或不装，共 2ⁿ 种方案，总代价 O(n·2ⁿ)。

### 5.2 最优子结构

**子问题定义：** Pᵢ = [{i, i+1, ..., n}, Cᵢ]

其中 Cᵢ = C - ∑_{k=1}^{i-1} wₖxₖ（剩余容量）

**定理：** 如果 Sᵢ = (yᵢ, yᵢ₊₁, ..., yₙ) 是子问题 Pᵢ 的最优解，则 (yᵢ₊₁, ..., yₙ) 是子问题 Pᵢ₊₁ = [{i+1,...,n}, Cᵢ - wᵢyᵢ] 的最优解。

**证明（反证法）：** 若 (yᵢ₊₁,...,yₙ) 不是 Pᵢ₊₁ 的最优解，则存在更优解 (zᵢ₊₁,...,zₙ)，则 (yᵢ, zᵢ₊₁,...,zₙ) 比 Sᵢ 更优，矛盾。

**子问题重叠性：** 当 wᵢ 均为整数时，不同的物品选择路径可能产生相同的剩余容量，导致大量重叠子问题。

### 5.3 递归方程

设 **m(i, j)** = 子问题 [{i, i+1, ..., n}, j] 的最优解代价（最大总价值）：

$$
m(i,j) = \begin{cases} 0 & i=n,\ 0 \le j < w_n \ v_n & i=n,\ j \ge w_n \ m(i+1, j) & i<n,\ 0 \le j < w_i \ \max{m(i+1,j),\ m(i+1,j-w_i)+v_i} & i<n,\ j \ge w_i \end{cases}
$$

**直觉理解：**

- 当剩余容量 j < wᵢ 时：物品 i **装不下**，只能不装
- 当 j ≥ wᵢ 时：取"不装"和"装"中的**较大值**
  - 不装：m(i+1, j)
  - 装：m(i+1, j-wᵢ) + vᵢ

### 5.4 算法

```
// 自底向上填表（从第 n 行到第 1 行）
For j=0 To min(w_n-1, C) Do  m[n,j] ← 0
For j=w_n To C Do             m[n,j] ← v_n

For i=n-1 To 2 Do
  For j=0 To min(w_i-1, C) Do
    m[i,j] ← m[i+1, j]
  For j=w_i To C Do
    m[i,j] ← max{m[i+1,j], m[i+1,j-w_i]+v_i}

// 处理第1个物品
If C < w_1 Then m[1,C] ← m[2,C]
Else m[1,C] ← max{m[2,C], m[2,C-w_1]+v_1}
```

### 5.5 构造最优解

```
从 m(1,C) 开始：
  If m(1,C) = m(2,C) Then x₁=0, 继续由 m(2,C) 构造
  Else x₁=1, 继续由 m(2,C-w₁) 构造
  
一般地，对物品 i：
  If m(i,j) = m(i+1,j) Then xᵢ=0, 从 m(i+1,j) 继续
  Else xᵢ=1, 从 m(i+1,j-wᵢ) 继续
```

### 5.6 复杂度分析

| 项目             | 复杂度    |
| ---------------- | --------- |
| 计算代价时间     | O(nC)     |
| 构造最优解时间   | O(n)      |
| **总时间复杂度** | **O(nC)** |
| **空间复杂度**   | **O(nC)** |

> ⚠️ **这是伪多项式时间算法！**
>
> - C 的输入规模是 log C 位，所以 O(nC) = O(n·2^{log C})
> - 当 C = 2ⁿ 时，T(n) = O(n·2ⁿ)
> - 当 wᵢ 不限定为正整数时，该 DP 方法不直接适用
> - 0/1 背包问题是 **NP-完全** 问题

## 六、最优二叉搜索树（4.6）

### 6.1 问题定义

- **关键字集合：** K = {k₁, k₂, ..., kₙ}，k₁ < k₂ < ... < kₙ
- **虚拟键（叶节点）：** D = {d₀, d₁, ..., dₙ}
  - dᵢ 对应区间 (kᵢ, kᵢ₊₁)
  - d₀ 对应 (-∞, k₁)，dₙ 对应 (kₙ, +∞)
- **概率：**
  - 搜索 kᵢ 的概率为 pᵢ
  - 搜索 dᵢ 的概率为 qᵢ
  - ∑pᵢ + ∑qⱼ = 1

**搜索树的期望代价：**

$$E(T) = \sum_{i=1}^{n} p_i \cdot (DEP_T(k_i)+1) + \sum_{j=0}^{n} q_j \cdot (DEP_T(d_j)+1)$$

- **目标：** 构造使 E(T) 最小的二叉搜索树

### 6.2 最优子结构

**观察：** K = {k₁,...,kₙ} 的最优 BST 的根必为某个 kᵣ：

```
        k_r
       /   \
  {k₁,...,k_{r-1}}  {k_{r+1},...,kₙ}
```

**定理（最优子结构）：** 如果最优 BST T 含有子树 T'（包含关键字 {kᵢ,...,kⱼ}），则 T' 必为关于 {kᵢ,...,kⱼ} 子问题的最优解。

**证明（反证法）：** 若 T' 不是最优，则存在更优子树 T''，用 T'' 替换 T 中的 T' 得到比 T 更优的搜索树，矛盾。

**子问题重叠性：** 不同根的选择导致相同的子集 {kᵢ,...,kⱼ} 重复出现。

### 6.3 递归方程

设 **E(i, j)** = {kᵢ,...,kⱼ} 的最优 BST 的期望搜索代价：

**辅助量：**

$$W(i,j) = \sum_{l=i}^{j} p_l + \sum_{l=i-1}^{j} q_l$$

递推计算：
$$W(i, i-1) = q_{i-1}, \quad W(i,j) = W(i,j-1) + p_j + q_j$$

> W(i,j) 表示子树中所有节点概率之和，当子树作为某节点的子树"下沉一层"时，期望代价增加 W(i,j)。

**递归方程：**

$$
E(i,j) = \begin{cases} q_{i-1} & \text{if } j = i-1 \text{（仅含叶节点 } d_{i-1}\text{）} \\ \displaystyle\min_{i \le r \le j}{E(i,r-1) + E(r+1,j) + W(i,j)} & \text{if } j \ge i \end{cases}
$$

**推导过程：**

- 选 kᵣ 为根，左子树含 {kᵢ,...,k_{r-1}}，右子树含 {k_{r+1},...,kⱼ}
- 左右子树放入 Tᵢⱼ 时，每个节点深度 +1
- 代价增量 = W(i,r-1) + W(r+1,j) + pᵣ = W(i,j)
- 因此 E(i,j) = E(i,r-1) + E(r+1,j) + W(i,j)

### 6.4 自底向上计算

按子问题规模 l = j - i + 1 递增：

```
l=0:  E(1,0)=q₀, E(2,1)=q₁, ..., E(n+1,n)=qₙ
l=1:  E(1,1), E(2,2), ..., E(n,n)
l=2:  E(1,2), E(2,3), ..., E(n-1,n)
  ...
l=n:  E(1,n)  ← 最终答案
```

### 6.5 算法

```
Optimal-BST(p, q, n)
  // 初始化 l=0 的情况
  For i=1 To n+1 Do
    E(i, i-1) ← q_{i-1}
    W(i, i-1) ← q_{i-1}
  
  // 按子问题规模 l 递增填表
  For l=1 To n Do
    For i=1 To n-l+1 Do
      j ← i+l-1
      E(i,j) ← ∞
      W(i,j) ← W(i,j-1) + p_j + q_j
      For r=i To j Do           // 枚举根节点
        t ← E(i,r-1) + E(r+1,j) + W(i,j)
        If t < E(i,j) Then
          E(i,j) ← t
          Root(i,j) ← r         // 记录最优根
  Return E and Root
```

**数据结构：**

- `E[1:n+1, 0:n]`：存储最优期望搜索代价
- `W[1:n+1, 0:n]`：存储概率和
- `Root[1:n, 1:n]`：记录每个子问题最优解的根

### 6.6 构造最优解

利用 Root 数组递归构造：

- Root(1,n) = r → kᵣ 是整棵树的根
- Root(1,r-1) → 左子树的根
- Root(r+1,n) → 右子树的根
- 递归直到子问题为空

### 6.7 复杂度分析

| 项目             | 复杂度                                             |
| ---------------- | -------------------------------------------------- |
| 计算代价时间     | O(n³)（l, i, r 三层循环，每层至多 n 步）           |
| 构造最优解时间   | O(n)                                               |
| **总时间复杂度** | **O(n³)**                                          |
| **空间复杂度**   | **O(n²)**（两个 (n+1)×(n+1) 数组 + 一个 n×n 数组） |

