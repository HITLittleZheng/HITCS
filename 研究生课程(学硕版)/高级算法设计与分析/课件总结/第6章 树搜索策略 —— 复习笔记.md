# 第六章 树搜索策略 —— 复习笔记

------

## 一、树搜索方法的研究动机

### 核心思想

很多问题的解可以表示为**树**的形式，解为树的**节点**或**路径**，求解问题可以转化为**树搜索问题**。

### 典型例子

| 问题                   | 解的形式 | 说明                                              |
| ---------------------- | -------- | ------------------------------------------------- |
| SAT（可满足性问题）    | 路径     | 每层对一个布尔变量赋值T/F，叶节点对应一组完整赋值 |
| 8-Puzzle（八数码问题） | 节点     | 每次移动产生一个子节点，目标状态对应的节点即为解  |

**SAT问题定义：**

- **输入：** n个布尔变量 $x_1, x_2, \dots, x_n$，关于它们的k个析取式
- **输出：** 是否存在一种赋值使所有k个析取式皆为真
- **树表示：** 二叉树，每层对应一个变量的T/F赋值，共 $2^n$ 个叶节点

## 二、简单树搜索策略

------

### 2.1 宽度优先搜索（Breadth-First Search, BFS）

#### 数据结构：队列（Queue）

#### 算法步骤

```
1. 构造仅由根组成的队列 Q
2. IF Q的第一个元素x是目标节点
3.   THEN 输出解，停止
4.   ELSE 从Q中删除x，把x的所有子节点加入Q的末尾
5. IF Q空 THEN 无解
6.   ELSE goto 2
```

#### 时间复杂度（搜索树共n个节点）

| 情况     | 复杂度                                                       |
| -------- | ------------------------------------------------------------ |
| **最好** | $O(1)$（目标节点为根）                                       |
| **最坏** | $O(n)$                                                       |
| **平均** | $\sum_{i=1}^{n} p_i \cdot Cost(i)$，$p_i$ 为节点i是目标的概率 |

#### 特点

- 按**层次**逐层扩展
- 保证找到**最浅**的目标节点（若边权相同则为最短路径）

------

### 2.2 深度优先搜索（Depth-First Search, DFS）

#### 数据结构：栈（Stack）

#### 算法步骤

```
1. 构造由根组成的单元素栈 S
2. IF Top(S) 是目标节点
3.   THEN 输出解，停止
4.   ELSE T = Top(S), Pop(S)
5.        把T的所有子节点压入栈顶
6. IF S空 THEN 无解
7.   ELSE goto 2
```

#### 时间复杂度（搜索树共n个节点）

| 情况     | 复杂度                             |
| -------- | ---------------------------------- |
| **最好** | $O(1)$                             |
| **最坏** | $O(n)$                             |
| **平均** | $\sum_{i=1}^{n} p_i \cdot Cost(i)$ |

#### 特点

- 沿一条路径**尽可能深入**，到底后回退
- 空间消耗比BFS小（只存储当前路径）

------

### 2.3 回溯搜索（Backtracking Search）

#### 核心思想

- 是**穷举搜索的第一次改进**
- 按**深度优先策略**搜索解空间树
- 在扩展节点处用**约束函数**剪去不满足约束条件的子树
- 或用**界限函数**剪去得不到最优解的子树
- 若当前节点肯定不包含解 → 跳过该子树，向祖先**回溯**
- 否则 → 进入子树，继续深度优先搜索

#### 两类常见解空间树

| 类型       | 适用问题                    | 叶节点数 | 遍历时间      |
| ---------- | --------------------------- | -------- | ------------- |
| **子集树** | 从n个元素中找满足性质的子集 | $2^n$    | $\Theta(2^n)$ |
| **排列树** | 确定n个元素满足性质的排列   | $n!$     | $\Theta(n!)$  |

#### 例：n=3的0-1背包问题（背包容量30）

- 问题实例：$w_1=16, v_1=45$；$w_2=15, v_2=25$；$w_3=15, v_3=25$
- 构造子集树，每层决定是否选取物品i（1或0）
- 在搜索过程中，若当前重量已超过容量 → **剪枝回溯**
- 最优解代价为50

------

## 三、最优树搜索策略

------

### 3.1 爬山法（Hill Climbing）

#### 核心思想

- **优化的深度优先搜索**
- 遇到多个可扩展节点时，使用**贪心方法（启发式测度）**确定扩展顺序
- 具有**局部优化**观念

#### 算法步骤

```
1. 构造由根组成的单元素栈 S
2. IF Top(S) 是目标节点 THEN 停止
3. Pop(S)
4. 将S（此处应为被弹出节点）的子节点按启发测度
   从大到小的顺序压入S（使最小值在栈顶，优先扩展）
5. IF S空 THEN 失败 ELSE goto 2
```

#### 启发式测度举例（8-Puzzle）

- $f(n) = W(n)$：节点n中**处于错误位置的方块数**
- 每次优先扩展 $f(n)$ 值最小的子节点

#### 特点

- 仅在**兄弟节点**之间选择最优（局部），不回头比较其他分支
- 可能陷入局部最优

------

### 3.2 最佳优先搜索（Best-First Search）

#### 核心思想

- 结合深度优先和广度优先的优点
- 根据评价函数，在**目前产生的所有节点**中选择评价函数值最小的节点扩展
- 具有**全局优化**观念（与爬山法的局部优化相对）

#### 数据结构：堆（Heap / 优先队列）

#### 算法步骤

```
1. 使用评价函数构造一个堆H，首先构造由根组成的单元素堆
2. IF H的根r是目标节点 THEN 停止
3. 从H中删除r，把r的子节点插入H
4. IF H空 THEN 失败 ELSE goto 2
```

#### 与爬山法的对比

| 特性     | 爬山法               | Best-First                   |
| -------- | -------------------- | ---------------------------- |
| 数据结构 | 栈                   | 堆（优先队列）               |
| 选择范围 | 当前兄弟节点（局部） | 所有已生成未扩展节点（全局） |
| 优化观念 | 局部                 | 全局                         |

------

### 3.3 分支界限策略（Branch-and-Bound）

#### 核心思想

- 用于有效求解**组合优化问题**
- 三要素：
  1. **分支机制**：使用前述任意搜索策略（如爬山法）产生分支
  2. **产生界限**：通过发现可能解（可行解），获得优化解代价的**上界**
  3. **剪枝**：若某节点的代价**下界**已超过当前最优上界，则剪掉该分支

#### 多阶段图搜索问题示例

- 输入：多阶段图
- 输出：从 $v_0$ 到 $v_3$ 的最短路径
- 用爬山法搜索，找到第一个可能解后得到上界，然后剪枝

## 四、人员分配问题（Personnel Assignment Problem）

### 4.1 问题定义

- **输入：**
  - 人的有序集合 $P = {P_1, P_2, \dots, P_n}$，$P_1 < P_2 < \dots < P_n$
  - 工作的偏序集合 $J = {J_1, J_2, \dots, J_n}$
  - 代价矩阵 $[C_{ij}]$，$C_{ij}$ 是 $P_i$ 被分配工作 $J_j$ 的代价
- **输出：**
  - 分配矩阵 $[X_{ij}]$，$X_{ij}=1$ 表示 $P_i$ 被分配 $J_j$
  - 使 $\sum_{i,j} C_{ij}X_{ij}$ 最小
- **约束：**
  - 不同人分配不同工作（一一对应）
  - 分配函数 $f: P \to J$ 满足偏序约束：若 $f(P_i) \succeq f(P_j)$，则 $P_i \geq P_j$；若 $i < j$，则 $f(P_i) \preceq f(P_j)$ 或不可比

### 4.2 最优子结构

**命题1：** $P_1 \to J_{k_1}, P_2 \to J_{k_2}, \dots, P_n \to J_{k_n}$ 是一个可行解，当且仅当 $J_{k_1}, J_{k_2}, \dots, J_{k_n}$ 是 $J$ 的一个**拓扑排序序列**。

→ 解空间 = 工作集合所有拓扑排序序列的集合

### 4.3 转换为树搜索问题

**拓扑序列树生成算法：**

```
输入：偏序集合S，树根root
输出：由S的所有拓扑排序序列构成的树
1. 生成树根root
2. 选择偏序集中没有前驱元素的所有元素，作为root的子节点
3. For root的每个子节点v Do
4.     S = S - {v}
5.     把v作为根，递归地处理S
```

### 4.4 计算代价下界

**命题2：** 把代价矩阵**每行（列）各元素减去同一个数**，不影响优化解。

**下界计算方法：**

1. 代价矩阵每行减去该行最小值
2. 再每列减去该列最小值
3. 使每行每列至少有一个0
4. **下界 = 所有减去的数之和**

### 4.5 分支界限搜索算法

```
1. 建立根节点，权值为解代价下界
2. 使用爬山法，类似拓扑排序序列树生成算法求解：
   每产生一个节点，其权值 = 加工后代价矩阵对应元素 + 父节点权值
3. 一旦发现一个可行解，将其代价作为界限
4. 循环进行分支界限搜索：
   - 剪掉不能导致优化解的子树（下界 > 当前最优上界）
   - 使用爬山法继续扩展
   - 直至发现优化解
```

## 五、旅行商问题（Traveling Salesman Problem, TSP）

### 5.1 问题定义

- **输入：** 有向连通图 $G=(V,E)$，每对节点间有非负加权边（代价矩阵）
- **输出：** 从任一节点出发，经过**每个节点恰好一次**，回到出发点的**最小代价回路**（Hamilton回路）

### 5.2 转换为树搜索问题

构造**二叉搜索树**：

- **根节点** = 所有解的集合
- 选择一条边 $(i,j)$ 进行**划分**：
  - **左子树** = 包含边 $(i,j)$ 的所有解集合
  - **右子树** = 不包含边 $(i,j)$ 的所有解集合
- 递归划分至得到具体解

### 5.3 代价下界计算

与人员分配问题类似：

1. 代价矩阵每行减去行最小值
2. 再每列减去列最小值
3. **下界 = 所有减去的数之和**

### 5.4 划分边的选择策略

选择边 $(i,j)$ 满足：

- $Cost(i,j) = 0$（在化简后的矩阵中）
- **使右子树代价下界增加最大**（贪心）
- 使左子树代价下界不变（或增加最少）

**右子树下界增量计算：**

- 不包含 $(i,j)$ → 必须包含一条从 $i$ 出发的其他边（行 $i$ 的次小值）+ 一条进入 $j$ 的其他边（列 $j$ 的最小值，排除 $(i,j)$）

### 5.5 左右子树代价矩阵构造

**左子树（包含边 $(i,j)$）：**

1. 删除矩阵的第 $i$ 行和第 $j$ 列
2. 将 $C[j,i]$ 设为 $\infty$（防止形成子回路）
3. 若已有部分路径 $i_1\text{-}i_2\text{-}\dots\text{-}i_m$ 和 $j_1\text{-}j_2\text{-}\dots\text{-}j_n$，加入 $(i_m, j_1)$ 后，须将 $C[j_n, i_1]$ 设为 $\infty$（避免环）
4. 检查是否有行/列没有0，若有则继续化简，更新下界

**右子树（不包含边 $(i,j)$）：**

1. 将 $C[i,j]$ 设为 $\infty$
2. 检查是否需要继续化简，更新下界

### 5.6 分支界限搜索算法

```
1. 根节点为所有解集合
2. 使用代价矩阵，计算根节点的代价下界
3. 使用爬山策略扩展当前树节点直至找到一个解：
   a. 选择划分边(x,y)：使右子树下界增加最大，左子树下界不变
   b. 构造左右子树，计算各自代价下界
   c. 构造左右子树的代价矩阵，修改代价下界
4. 用找到解的代价作为上界进行剪枝
5. 继续爬山搜索，直到找到最优解
```

### 5.7 注意事项

- 构造路径过程中须避免形成**子回路（环）**
- 若已有路径片段 $i_1\text{-}\dots\text{-}i_m$ 和 $j_1\text{-}\dots\text{-}j_n$，加入边 $(i_m, j_1)$ 后，必须将 $C[j_n, i_1]$ 设为 $\infty$

## 六、A* 算法

### 6.1 基本思想

- 使用 **Best-First策略** 求解优化问题
- 与分支界限的区别：
  - 分支界限：通过剪枝排除非最优分支
  - **A\*算法：一旦选中目标节点，即为最优解，算法立即停止**

### 6.2 代价函数

对于任意节点 $n$：

| 符号                     | 含义                                        |
| ------------------------ | ------------------------------------------- |
| $g(n)$                   | 从根到 $n$ 的**实际代价**（已知）           |
| $h^*(n)$                 | 从 $n$ 到目标节点的**最优路径代价**（未知） |
| $f^*(n) = g(n) + h^*(n)$ | 经过 $n$ 到目标的**最优总代价**（未知）     |
| $h(n)$                   | $h^*(n)$ 的**估计值**（启发函数）           |
| $f(n) = g(n) + h(n)$     | 节点 $n$ 的**评价函数**（已知，用于排序）   |

**关键约束：** $h(n) \leq h^*(n)$（估计值不超过真实值，即**可采纳启发函数**）

→ 因此 $f(n) = g(n) + h(n) \leq g(n) + h^*(n) = f^*(n)$

### 6.3 $h(n)$ 的估计方法

- 从节点 $n$ 出发的所有边中，选择**最小代价边**的代价作为 $h(n)$
- 例如：若从 $V_1$ 出发有代价为2和3的两条边，则 $h(V_1) = \min{2, 3} = 2$
- 这保证了 $h(n) \leq h^*(n)$

### 6.4 算法步骤（A* 三条规则）

```
(1) 使用 Best-First 规则（优先队列/堆）
(2) 代价函数 f(n) = g(n) + h(n)
    其中 g(n) = 根到n的路径代价
    h(n) = h*(n)的估计值，且对所有n满足 h(n) ≤ h*(n)
(3) 当选择到的节点是目标节点时，算法停止，
    该目标节点即为最优解
```

**详细执行过程：**

```
1. 将根节点放入优先队列H（按f值排序）
2. 取出H中f值最小的节点r
3. IF r是目标节点 THEN 输出解，停止（这就是最优解！）
4. ELSE 扩展r，计算所有子节点的f值，插入H
5. IF H空 THEN 无解 ELSE goto 2
```

### 6.5 求解最短路径问题示例

**输入图：** S → V1(2), S → V2(3), S → V3(4), V1 → V4(2), V1 → V2(3), V2 → V4(2), V2 → V5(2), V3 → V5(2), V4 → V5(1), V4 → T(3), V5 → T(5)

**搜索过程：**

| 步骤 | 扩展节点 | 生成子节点及f值           | 堆中待扩展                                      |
| ---- | -------- | ------------------------- | ----------------------------------------------- |
| 1    | S        | V1: f=4, V2: f=5, V3: f=6 | V1(4), V2(5), V3(6)                             |
| 2    | V1       | V4: f=5, V2': f=7         | V2(5), V4(5), V3(6), V2'(7)                     |
| 3    | V2       | V4': f=6, V5: f=10        | V4(5), V3(6), V4'(6), V2'(7), V5(10)            |
| 4    | V4       | V5': f=10, T: f=7         | V3(6), V4'(6), T(7), V2'(7), V5(10), V5'(10)    |
| 5    | V3       | V5'': f=11                | V4'(6), T(7), V2'(7), V5(10), V5'(10), V5''(11) |
| 6    | V4'      | V5''': f=11, T': f=8      | **T(7)**, T'(8), V2'(7)...                      |
| 7    | **T**    | T是目标节点 → **停止**    | —                                               |

**最优解：** S → V1 → V4 → T，代价 = 2+2+3 = **7**

### 6.6 A* 算法正确性证明

**定理：** 使用Best-First策略搜索树，如果A*选择的节点是目标节点，则该节点对应的解是最优解。

**证明：**

1. 设 $t$ 是选中的目标节点
2. 假设存在一个更优解 $s$，需要途经某节点 $n$（$n$ 尚未被扩展）
3. 由Best-First策略：$f(t) \leq f(n)$（$t$ 先于 $n$ 被选中）
4. 由可采纳条件 $h(n) \leq h^*(n)$：$f(n) = g(n) + h(n) \leq g(n) + h^*(n) = f^*(n)$
5. 而 $f^*(n)$ 是经过 $n$ 的最优解代价，$f^*(n) \leq f^*(s)$（$s$ 是经过 $n$ 的某个解）
6. 所以 $f(t) \leq f^*(s)$
7. 又因为 $t$ 是目标节点：$h(t) = h^*(t) = 0$，故 $f(t) = g(t) = f^*(t)$（$t$ 对应解的实际代价）
8. 所以 $f^*(t) \leq f^*(s)$
9. 若 $s$ 是最优解，则 $f^*(s) \leq f^*(t)$，因此 $f^*(t) = f^*(s)$
10. 即 $t$ 对应的解也是最优解 $\blacksquare$

## 七、各算法对比总结

| 算法       | 数据结构   | 策略              | 优化性               | 适用场景              |
| ---------- | ---------- | ----------------- | -------------------- | --------------------- |
| BFS        | 队列       | 逐层扩展          | 最浅解               | 等权图最短路径        |
| DFS        | 栈         | 尽深扩展          | 无保证               | 空间受限、解在深处    |
| 回溯       | 栈+剪枝    | DFS+约束/界限剪枝 | 可找最优             | 子集树/排列树问题     |
| 爬山法     | 栈（有序） | 启发式DFS         | 局部最优             | 有启发信息的问题      |
| Best-First | 堆         | 全局最优节点优先  | 全局较优             | 有评价函数的问题      |
| 分支界限   | 堆+界限    | Best-First+剪枝   | 最优解               | 组合优化（TSP、分配） |
| A*         | 堆         | Best-First, f=g+h | **最优解（有证明）** | 最短路径等优化问题    |

### 关键区分点

- **爬山法 vs Best-First：** 局部（兄弟间）vs 全局（所有待扩展节点间）
- **分支界限 vs A\*：** 分支界限靠剪枝保证最优；A*靠 $h(n)\leq h^*(n)$ 保证选中目标即最优
- **A\*的核心条件：** $h(n) \leq h^*(n)$（可采纳性），这是正确性的关键

# 人员分配问题 —— 详细解析

------

## 一、问题定义（白话版）

**场景：** 公司有 n 个人要分配到 n 个岗位，一人一岗，要求总代价最小。

**但有约束：** 岗位之间有"等级关系"（偏序），等级高的岗位必须分配给资历高的人。

------

## 二、形式化定义

### 输入

- **人集合**：$P = {P_1, P_2, P_3, P_4}$，有**全序** $P_1 < P_2 < P_3 < P_4$（按资历从低到高）
- **工作集合**：$J = {J_1, J_2, J_3, J_4}$，有**偏序**关系（用 Hasse 图表示）
- **代价矩阵**：$C[i][j]$ = 把 $P_i$ 分配到 $J_j$ 的代价

### 输出

- 一一对应的分配方案，使总代价最小

------

## 三、偏序约束到底是什么意思？

### 用具体例子说明

假设工作的 Hasse 图如下：

```
    J3      J4
    |        
    J1      J2
```

含义：$J_1 \preceq J_3$（$J_1$ 是 $J_3$ 的下级），$J_2$ 和 $J_4$ 与其他工作**不可比**。

### 约束规则

> **如果工作 $J_a$ 的等级低于 $J_b$（即 $J_a \preceq J_b$），那么分配到 $J_a$ 的人的资历必须低于分配到 $J_b$ 的人。**

翻译成数学语言：

- 若 $J_a \preceq J_b$，且 $P_i \to J_a$，$P_j \to J_b$，则必须 $i < j$

### 举例判断

| 分配方案                                             | 是否可行 | 原因                                                         |
| ---------------------------------------------------- | -------- | ------------------------------------------------------------ |
| $P_1 \to J_1, P_2 \to J_3, P_3 \to J_2, P_4 \to J_4$ | ✅        | $J_1 \preceq J_3$，分配给 $P_1, P_2$，满足 $1<2$             |
| $P_1 \to J_3, P_2 \to J_1, P_3 \to J_2, P_4 \to J_4$ | ❌        | $J_1 \preceq J_3$，但 $J_3$ 给了 $P_1$，$J_1$ 给了 $P_2$，$1<2$ 但高级工作给了低资历人 |

------

## 四、核心定理：解空间 = 拓扑排序

### 定理（命题1）

> 分配 $P_1 \to J_{k_1}, P_2 \to J_{k_2}, \dots, P_n \to J_{k_n}$ 是**可行解**
> $\iff$ 序列 $J_{k_1}, J_{k_2}, \dots, J_{k_n}$ 是工作偏序集的一个**拓扑排序**

### 为什么？

**拓扑排序的定义：** 对于偏序中 $J_a \preceq J_b$，在序列中 $J_a$ 必须出现在 $J_b$ 前面。

**约束的要求：** 若 $J_a \preceq J_b$，则 $J_a$ 必须分配给编号更小的人（即 $J_a$ 在序列中更早出现）。

→ 两者完全一致！

### 回到例子

Hasse 图：$J_1 \preceq J_3$，$J_2$ 和 $J_4$ 独立

所有拓扑排序（即所有可行解）：

```
J1, J2, J3, J4    → P1→J1, P2→J2, P3→J3, P4→J4
J1, J2, J4, J3    → P1→J1, P2→J2, P3→J4, P4→J3
J1, J3, J2, J4    → P1→J1, P2→J3, P3→J2, P4→J4
J1, J3, J4, J2    → P1→J1, P2→J3, P3→J4, P4→J2
J2, J1, J3, J4    → P1→J2, P2→J1, P3→J3, P4→J4
J2, J1, J4, J3    → ...
J1, J4, J2, J3    → ...
... 等等
```

**关键：** $J_1$ 必须在 $J_3$ 前面，其他随意。

------

## 五、转换为树搜索（拓扑序列树）

### 建树规则

```
第1层（给P1分配）：选"当前没有前驱"的工作作为子节点
第2层（给P2分配）：去掉已选的，再选"没有前驱"的工作
...依此类推
```

### 具体建树过程

Hasse 图：$J_1 \preceq J_3$

```
                        root
                      /      \
                   J1          J2          ← 没有前驱的：J1, J2, J4
                  /|\          /|\            （J3有前驱J1，不能选）
                J2 J3 J4    J1  J4
               ...           ...
```

**第1层**（给 $P_1$）：可选 $J_1, J_2, J_4$（$J_3$ 有前驱 $J_1$，不可选）

**第2层**（给 $P_2$，假设第1层选了 $J_1$）：

- 去掉 $J_1$ 后，$J_3$ 的前驱消失了
- 可选：$J_2, J_3, J_4$

**第2层**（给 $P_2$，假设第1层选了 $J_2$）：

- 去掉 $J_2$ 后，$J_3$ 的前驱 $J_1$ 还在
- 可选：$J_1, J_4$

------

## 六、计算代价下界

### 核心原理（命题2）

> 代价矩阵每行（或每列）**同时减去一个常数**，不影响最优解的位置。

**直觉理解：** 若每行减去行最小值，相当于把"基础代价"提取出来，所有方案的相对优劣不变。

### 下界计算步骤

**例：** 代价矩阵

$$C = \begin{pmatrix} 5 & 3 & 6 & 7 \\ 4 & 7 & 2 & 5 \\ 8 & 5 & 3 & 4 \\ 6 & 9 & 5 & 2 \end{pmatrix}$$

**Step 1：每行减去行最小值**

| 行   | 最小值 | 操作      |
| ---- | ------ | --------- |
| 行1  | 3      | 各元素减3 |
| 行2  | 2      | 各元素减2 |
| 行3  | 3      | 各元素减3 |
| 行4  | 2      | 各元素减2 |

$$C' = \begin{pmatrix} 2 & 0 & 3 & 4 \\ 2 & 5 & 0 & 3 \\ 5 & 2 & 0 & 1 \\ 4 & 7 & 3 & 0 \end{pmatrix}$$

**Step 2：每列减去列最小值**

| 列   | 最小值 | 操作      |
| ---- | ------ | --------- |
| 列1  | 2      | 各元素减2 |
| 列2  | 0      | 不变      |
| 列3  | 0      | 不变      |
| 列4  | 0      | 不变      |

$$C'' = \begin{pmatrix} 0 & 0 & 3 & 4 \\ 0 & 5 & 0 & 3 \\ 3 & 2 & 0 & 1 \\ 2 & 7 & 3 & 0 \end{pmatrix}$$

**下界 = 所有减去的数之和 = (3+2+3+2) + (2+0+0+0) = 12**

**含义：** 无论怎么分配，总代价**至少**为 12。

------

## 七、分支界限搜索完整过程

### 整体策略

```
1. 用爬山法（启发式DFS）沿树向下搜索
2. 每个节点计算代价下界
3. 找到第一个完整解 → 得到上界
4. 回溯检查其他分支：
   - 若某节点下界 ≥ 当前最优上界 → 剪枝
   - 否则继续搜索
5. 直到所有有希望的分支都检查完毕
```

### 具体例子

假设代价矩阵（4人4工作，$J_1 \preceq J_3$）：

$$C = \begin{pmatrix}
      & J_1 & J_2 & J_3 & J_4 \\
  P_1 & 5   & 3   & 6   & 7   \\
  P_2 & 4   & 7   & 2   & 5   \\
  P_3 & 8   & 5   & 3   & 4   \\
  P_4 & 6   & 9   & 5   & 2
\end{pmatrix}$$

**根节点下界** = 12（如上计算）

------

### 搜索第1层（给 $P_1$ 分配）

可选：$J_1, J_2, J_4$（$J_3$ 有前驱不可选）

**用爬山法：** 选使代价增量最小的分支

- 选 $J_1$：代价贡献 = $C''[1][1] = 0$（用化简后的矩阵）
- 选 $J_2$：代价贡献 = $C''[1][2] = 0$
- 选 $J_4$：代价贡献 = $C''[1][4] = 4$

爬山法选代价贡献最小的，比如选 $J_1$（代价=0）

**节点下界** = 根下界 + 代价贡献 = 12 + 0 = 12

选了 $J_1$ 后：

- 删除第1行、第1列
- 子矩阵需要重新检查是否每行每列有0

------

### 搜索第2层（给 $P_2$ 分配，已选 $J_1$）

去掉 $J_1$ 后，$J_3$ 无前驱了。可选：$J_2, J_3, J_4$

继续用化简后的子矩阵计算代价贡献，选最小的...

------

### 找到第一个完整解

假设沿爬山法一路走到底，得到方案：

$P_1 \to J_1, P_2 \to J_3, P_3 \to J_2, P_4 \to J_4$

总代价 = $5 + 2 + 5 + 2 = 14$

→ **当前上界 = 14**

------

### 剪枝回溯

回溯到之前的分支节点：

- 若某节点的下界 ≥ 14 → **剪掉**（不可能比当前解更好）
- 若下界 < 14 → 继续探索

最终找到所有下界 < 当前上界的分支中，代价最小的方案 = **最优解**。

------

## 八、图解总结

```
                    root (下界=12)
                 /        |         \
          P1→J1(12)   P1→J2(12)   P1→J4(16) ← 剪枝(若>上界)
          /    \          |
     P2→J3(12) P2→J2(15) ...
      /    \
  P3→J2(14) P3→J4(13)
     |         |
  P4→J4(14) P4→J2(17) ← 剪枝
   [解1=14]
   
  ↑ 找到解1后上界=14
  继续检查其他 <14 的分支...
```

------

## 九、关键点总结

| 要点                         | 说明                                                         |
| ---------------------------- | ------------------------------------------------------------ |
| **为什么要拓扑排序？**       | 偏序约束要求：等级低的工作必须分配给编号小的人，这恰好是拓扑排序的定义 |
| **树的每一层是什么？**       | 第 $i$ 层决定 $P_i$ 分配哪个工作                             |
| **每层能选什么？**           | 当前剩余工作中**没有前驱**的工作（前驱已被删除也算）         |
| **下界怎么算？**             | 矩阵行列化简，减去的数之和 = 下界                            |
| **为什么化简不影响最优解？** | 每行减同一个数，所有方案代价同时减少相同量，相对优劣不变     |
| **什么时候剪枝？**           | 节点下界 ≥ 已知最优解代价（上界）→ 剪                        |

# 旅行商问题（TSP）—— 分支界限法详细解析

------

## 一、问题定义

### 白话版

一个旅行商要从某城市出发，**经过所有城市恰好一次**，最后回到出发城市，要求**总路程最短**。

### 形式化定义

- **输入：** 完全有向图 $G=(V,E)$，代价矩阵 $C[i][j]$（$i$ 到 $j$ 的距离）
- **输出：** 一条**Hamilton回路**（经过每个顶点恰好一次的回路），使总代价最小
- **对角线：** $C[i][i] = \infty$（自己到自己不允许）

------

## 二、具体例子

### 代价矩阵（5个城市）

$$C = \begin{pmatrix}
& 1 & 2 & 3 & 4 & 5 \\
1 & \infty & 20 & 30 & 10 & 11 \\
2 & 15 & \infty & 16 & 4 & 2 \\
3 & 3 & 5 & \infty & 2 & 4 \\
4 & 19 & 6 & 18 & \infty & 3 \\
5 & 16 & 4 & 7 & 16 & \infty
\end{pmatrix}$$

------

## 三、第一步：计算根节点下界

### 原理

> 任何Hamilton回路中，**每个城市恰好离开一次**（每行用一个元素）、**每个城市恰好到达一次**（每列用一个元素）。
>
> 因此，每行减去行最小值、每列减去列最小值后，总代价的减少量就是所有解代价的**公共下界**。

### Step 1：每行减去行最小值

| 行   | 最小值 | 操作后                 |
| ---- | ------ | ---------------------- |
| 行1  | 10     | $\infty, 10, 20, 0, 1$ |
| 行2  | 2      | $13, \infty, 14, 2, 0$ |
| 行3  | 2      | $1, 3, \infty, 0, 2$   |
| 行4  | 3      | $16, 3, 15, \infty, 0$ |
| 行5  | 4      | $12, 0, 3, 12, \infty$ |

行最小值之和 = $10 + 2 + 2 + 3 + 4 = 21$

化简后矩阵：
$$C' = \begin{pmatrix}
\infty & 10 & 20 & 0 & 1 \\
13 & \infty & 14 & 2 & 0 \\
1 & 3 & \infty & 0 & 2 \\
16 & 3 & 15 & \infty & 0 \\
12 & 0 & 3 & 12 & \infty
\end{pmatrix}$$

### Step 2：每列减去列最小值

| 列   | 最小值 | 操作后                  |
| ---- | ------ | ----------------------- |
| 列1  | 1      | $\infty, 12, 0, 15, 11$ |
| 列2  | 0      | 不变                    |
| 列3  | 3      | $17, 11, \infty, 12, 0$ |
| 列4  | 0      | 不变                    |
| 列5  | 0      | 不变                    |

列最小值之和 = $1 + 0 + 3 + 0 + 0 = 4$

最终化简矩阵：
$$C'' = \begin{pmatrix}
\infty & 10 & 17 & 0 & 1 \\
12 & \infty & 11 & 2 & 0 \\
0 & 3 & \infty & 0 & 2 \\
15 & 3 & 12 & \infty & 0 \\
11 & 0 & 0 & 12 & \infty
\end{pmatrix}$$

### 根节点下界

$$\text{下界} = 21 + 4 = \boxed{25}$$

**含义：** 无论走哪条回路，总代价至少为 25。

------

## 四、第二步：选择划分边

### 核心思想

选一条边 $(i,j)$ 把所有解分成两类：

- **左子树：** 包含边 $(i,j)$ 的所有Hamilton回路
- **右子树：** 不包含边 $(i,j)$ 的所有Hamilton回路

### 选择标准

从化简矩阵中**值为0的元素**中选择，选使**右子树下界增加最大**的那条边。

### 为什么选值为0的？

- 值为0说明这条边在化简后是"免费"的
- 如果不走这条"免费"边（右子树），代价必然增加
- 增加越多 → 右子树越差 → 越容易被剪枝

### 计算每个0的"惩罚值"

对于 $C''[i][j] = 0$，如果不走边 $(i,j)$：

- 从城市 $i$ 必须走另一条出去的边 → 行 $i$ 中除 $(i,j)$ 外的**最小值**
- 进入城市 $j$ 必须从另一个城市来 → 列 $j$ 中除 $(i,j)$ 外的**最小值**
- **惩罚值 = 行$i$次小值 + 列$j$次小值**

### 具体计算

找出所有 $C''[i][j] = 0$ 的位置：

| 位置                                             | 行次小值 | 列次小值 | 惩罚值 |
| ------------------------------------------------ | -------- | -------- | ------ |
| $(1,4)$：行1除(1,4)外最小=1，列4除(1,4)外最小=0  | 1        | 0        | **1**  |
| $(2,5)$：行2除(2,5)外最小=2，列5除(2,5)外最小=0  | 2        | 0        | **2**  |
| $(3,1)$：行3除(3,1)外最小=0，列1除(3,1)外最小=11 | 0        | 11       | **11** |
| $(3,4)$：行3除(3,4)外最小=0，列4除(3,4)外最小=0  | 0        | 0        | **0**  |
| $(4,5)$：行4除(4,5)外最小=3，列5除(4,5)外最小=0  | 3        | 0        | **3**  |
| $(5,2)$：行5除(5,2)外最小=0，列2除(5,2)外最小=0  | 0        | 0        | **0**  |
| $(5,3)$：行5除(5,3)外最小=0，列3除(5,3)外最小=11 | 0        | 11       | **11** |

**最大惩罚值 = 11**，对应 $(3,1)$ 和 $(5,3)$

**选择边 $(3,1)$ 作为划分边**（或选 $(5,3)$，两者等价）

------

## 五、第三步：构造左右子树

### 右子树（不包含边 $(3,1)$）

**操作：** 将 $C''[3][1]$ 设为 $\infty$

$$\text{右子树矩阵第3行} = (\infty, 3, \infty, 0, 2)$$

检查：行3最小值变为0（没变），列1最小值变为11（原来是0，现在最小是11）

→ 列1需要再减去11？不对，让我重新算：

列1原来：$\infty, 12, 0, 15, 11$，去掉(3,1)的0后变成 $\infty, 12, \infty, 15, 11$，最小值=11

所以列1要减去11。

**右子树下界 = 25 + 11 = 36**

（注：惩罚值就是右子树下界的增量，所以右子树下界 = 根下界 + 惩罚值）

### 左子树（包含边 $(3,1)$）

**操作步骤：**

#### ① 删除第3行和第1列

含义：城市3已经确定去城市1了，所以：

- 不需要再考虑从城市3出发去其他地方（删行3）
- 不需要再考虑其他城市到城市1（删列1）

剩余矩阵（4×4）：

$$\begin{pmatrix}
& 2 & 3 & 4 & 5 \\
1 & 10 & 17 & 0 & 1 \\
2 & \infty & 11 & 2 & 0 \\
4 & 3 & 12 & \infty & 0 \\
5 & 0 & 0 & 12 & \infty
\end{pmatrix}$$

#### ② 设置 $C[1][3] = \infty$（防止子回路）

**为什么？** 现在有路径片段 $3 \to 1$。如果允许 $1 \to 3$，就形成了子回路 $1 \to 3 \to 1$，不是Hamilton回路。

**一般规则：** 若当前已有路径片段 $a \to b \to \dots \to c$，则必须设 $C[c][a] = \infty$，防止提前成环。

修改后：

$$\begin{pmatrix}
& 2 & 3 & 4 & 5 \\
1 & 10 & \infty & 0 & 1 \\
2 & \infty & 11 & 2 & 0 \\
4 & 3 & 12 & \infty & 0 \\
5 & 0 & 0 & 12 & \infty
\end{pmatrix}$$

#### ③ 检查是否需要继续化简

- 行1：最小值=0 ✓
- 行2：最小值=0 ✓
- 行4：最小值=0 ✓
- 行5：最小值=0 ✓
- 列2：最小值=0 ✓
- 列3：最小值=0 ✓
- 列4：最小值=0 ✓
- 列5：最小值=0 ✓

每行每列都有0，**不需要额外化简**。

**左子树下界 = 25 + 0 = 25**（没有额外化简量）

------

## 六、第四步：用爬山法继续搜索

### 选择扩展节点

| 节点                | 下界 |
| ------------------- | ---- |
| 左子树（含边3→1）   | 25   |
| 右子树（不含边3→1） | 36   |

**爬山法选下界小的 → 扩展左子树**

### 在左子树中继续选择划分边

对左子树的4×4矩阵，找所有值为0的位置，计算惩罚值，选最大的...

（重复上述过程）

------

## 七、搜索过程图解

```
                        根节点
                     所有Hamilton回路
                       下界 = 25
                      /          \
                 含(3,1)        不含(3,1)
                下界=25          下界=36
               /       \
          含(?,?)     不含(?,?)
          下界=25      下界=?
         /      \
       ...      ...
      /
  找到完整回路
  代价 = 28
  ← 上界更新为28
  
  回溯：检查所有下界<28的节点
  → 右子树下界36>28 → 剪枝！
  → 其他...
```

------

## 八、防止子回路的详细说明

### 什么是子回路？

5个城市的Hamilton回路应该是：$1 \to ? \to ? \to ? \to ? \to 1$

但如果搜索过程中允许：

- 选了边 $3 \to 1$
- 又选了边 $1 \to 3$

就形成了 $1 \to 3 \to 1$ 这个长度为2的小环，剩下的3个城市无法全部经过。

### 一般规则

**已确定的路径片段：** 假设目前已经选定的边形成了若干路径片段：

```
片段1: a → b → c
片段2: d → e
```

如果现在要加入边 $(c, d)$，把两个片段连起来变成：

```
a → b → c → d → e
```

则必须设 $C[e][a] = \infty$，因为如果走 $e \to a$ 就会形成完整环 $a \to b \to c \to d \to e \to a$，但此时还有其他城市没经过！（除非这已经是包含所有城市的完整回路）

### 具体跟踪

| 步骤 | 选定边  | 当前路径片段                               | 需要禁止的边     |
| ---- | ------- | ------------------------------------------ | ---------------- |
| 1    | $(3,1)$ | $3 \to 1$                                  | $C[1][3]=\infty$ |
| 2    | $(1,4)$ | $3 \to 1 \to 4$                            | $C[4][3]=\infty$ |
| 3    | $(5,3)$ | $5 \to 3 \to 1 \to 4$ 和单独的 $5$ 不对... | 看连接情况       |

让我更仔细地说明：

假设依次选择了：

- 边 $(3,1)$：路径片段 $3 \to 1$，禁止 $C[1][3]=\infty$
- 边 $(1,4)$：连接到片段变成 $3 \to 1 \to 4$，禁止 $C[4][3]=\infty$
- 边 $(4,5)$：变成 $3 \to 1 \to 4 \to 5$，禁止 $C[5][3]=\infty$

此时只剩城市2没在这条路径上。要完成回路必须：$2 \to 3$ 和 $5 \to 2$，即 $2 \to 3 \to 1 \to 4 \to 5 \to 2$。

------

## 九、完整搜索过程（简化演示）

### 初始状态

- 根节点下界 = 25
- 化简矩阵如上

### 第1次划分：选边 $(3,1)$

- 左子树（含3→1）：下界 = 25
- 右子树（不含3→1）：下界 = 25 + 11 = 36
- **扩展左子树**

### 第2次划分（在左子树中）：对4×4矩阵找划分边

假设计算后选边 $(1,4)$（惩罚值最大）：

- 左子树（含3→1, 1→4）：下界 = 25
- 右子树（含3→1, 不含1→4）：下界 = 25 + ?
- **扩展左子树**

已有路径：$3 \to 1 \to 4$，禁止 $C[4][3]=\infty$

### 第3次划分：继续...

假设选边 $(4,5)$：

- 路径变为 $3 \to 1 \to 4 \to 5$，禁止 $C[5][3]=\infty$

### 第4次划分：

剩余2×2矩阵，只有一种选择（假设为边 $(5,2)$ 和 $(2,3)$）

得到完整回路：$1 \to 4 \to 5 \to 2 \to 3 \to 1$

计算总代价 = $10 + 3 + 4 + 5 + 3 = 25$？（需要在原始矩阵上算）

实际代价 = $C[1][4] + C[4][5] + C[5][2] + C[2][3] + C[3][1] = 10 + 3 + 4 + 16 + 3 = 36$

hmm 让我用原始矩阵重新算...

$C[3][1]=3, C[1][4]=10, C[4][5]=3, C[5][2]=4, C[2][3]=16$

总代价 = $3+10+3+4+16 = 36$

→ **上界 = 36**

### 剪枝回溯

- 右子树（不含3→1）下界=36，等于上界 → 不能改善，**剪枝**
- 检查其他分支...

最终确定最优解。

## 十、算法完整流程总结

```
算法：TSP的分支界限法

输入：n×n 代价矩阵 C
输出：最小代价Hamilton回路

1. 化简矩阵（行列减最小值），得到根节点下界 LB
2. REPEAT:
   a. 在当前化简矩阵中，找所有值为0的位置
   b. 对每个0位置(i,j)，计算惩罚值 = 行i次小值 + 列j次小值
   c. 选惩罚值最大的(i,j)作为划分边
   d. 构造左子树（含边i→j）：
      - 删第i行第j列
      - 设C[j→i方向]=∞（防子回路）
      - 化简，更新下界
   e. 构造右子树（不含边i→j）：
      - 设C[i][j]=∞
      - 化简，更新下界 = 父节点下界 + 惩罚值
   f. 用爬山法选下界小的子树继续扩展
3. 找到完整回路 → 记录为当前最优解（上界）
4. 回溯，剪掉下界≥上界的分支
5. UNTIL 所有分支处理完毕
6. 输出最优解
```

# 一、A* 算法 —— 完整详解

------

## 1.1 解决什么问题？

> 在**加权图**中找从**起点S**到**终点T**的**最短路径**。

与分支界限的区别：

- 分支界限：找到解后还要**回溯验证**，通过剪枝排除非最优
- **A\*：一旦从堆中取出的是目标节点，立刻停止，保证就是最优解**

------

## 1.2 三个关键函数

对于搜索树中的任意节点 $n$：

```
g(n)  = 从起点S到节点n的【已走过的实际代价】（精确已知）
h*(n) = 从节点n到终点T的【真实最短代价】（未知，想知道但算不出）
h(n)  = 从节点n到终点T的【估计代价】（我们自己设计的）

f*(n) = g(n) + h*(n)  ← 经过n的真实最优代价（未知）
f(n)  = g(n) + h(n)   ← 经过n的估计代价（用这个来排序）
```

### 核心约束（可采纳条件）

$$\boxed{h(n) \leq h^*(n) \quad \text{对所有节点 } n}$$

**白话：估计值永远不能高估真实值（乐观估计）**

### 为什么这个条件重要？

$$h(n) \leq h^*(n) \implies f(n) = g(n) + h(n) \leq g(n) + h^*(n) = f^*(n)$$

即：**评价函数值 ≤ 真实最优代价**，永远是下界。

------

## 1.3 $h(n)$ 怎么设计？

### 方法：取从节点n出发的所有边中的最小边权

$$h(n) = \min_{(n,v) \in E} w(n, v)$$

**为什么满足 $h(n) \leq h^\*(n)$？**

- $h^*(n)$ 是从 $n$ 到 $T$ 的最短路径代价，至少要走一条边出去
- 最短路径的第一条边代价 $\geq$ 所有出边中的最小值
- 所以 $h(n) \leq h^*(n)$ ✓

### 例子

```
节点V1有出边：V1→V4 代价2，V1→V2 代价3
则 h(V1) = min{2, 3} = 2
```

## 1.5 正确性证明（为什么取出T就能停？）

### 定理

> 在A*算法中，若从堆中取出的节点是目标节点T，则对应路径是最优的。

### 证明

**反证法：** 假设取出的T不是最优的，存在一条更好的路径。

```
设取出的T对应路径代价为 f(T) = g(T) + 0 = g(T)
假设存在更优解，经过某个还在堆中的节点n
```

**推导：**

```
① f(T) ≤ f(n)          ← 因为T先于n被取出（堆中T的f最小）
② f(n) = g(n) + h(n) 
       ≤ g(n) + h*(n)   ← 可采纳条件 h(n) ≤ h*(n)
       = f*(n)           ← 经过n的真实最短路径代价
③ 而经过n的任何到T的路径代价 ≥ f*(n)
④ 所以 f(T) ≤ f(n) ≤ f*(n) ≤ 任何经过n的解的代价

⑤ 又因为T是终点：h(T) = h*(T) = 0
   所以 f(T) = g(T) = T对应解的真实代价

⑥ 结论：T对应解的代价 ≤ 任何其他解的代价 → T就是最优解 ✓
```

**关键：** 全靠 $h(n) \leq h^*(n)$ 这一条件！

------

## 1.6 $h(n)$ 的设计对效率的影响

| $h(n)$ 的选择                      | 效果                                          |
| ---------------------------------- | --------------------------------------------- |
| $h(n) = 0$（对所有n）              | 退化为**Dijkstra算法**（只按g排序），正确但慢 |
| $h(n)$ 越接近 $h^*(n)$（但不超过） | 搜索效率越高，扩展节点越少                    |
| $h(n) = h^*(n)$（完美估计）        | 直接沿最优路径走，不扩展任何多余节点          |
| $h(n) > h^*(n)$（违反条件）        | **可能找不到最优解！**                        |

# 二、回溯法 —— 完整详解

------

## 2.1 核心思想

```
穷举搜索太慢
    ↓ 改进
回溯法 = 深度优先搜索 + 剪枝
    ↓
在搜索过程中，一旦发现当前路径不可能产生解（或最优解）
就立即回头（回溯），不再继续深入
```

------

## 2.2 两类解空间树

### 子集树（Subset Tree）

**适用问题：** 从n个元素中选一个**子集**

**结构：** 每层决定第 $i$ 个元素"选"还是"不选"

```
           root
          /    \
        选x1   不选x1
        / \     / \
     选x2 不选  选x2 不选
     ...
```

- 叶节点数：$2^n$
- 遍历时间：$\Theta(2^n)$
- **典型问题：0-1背包**

### 排列树（Permutation Tree）

**适用问题：** 确定n个元素的一个**排列**

**结构：** 每层从剩余元素中选一个

```
           root
         / | \
        x1 x2 x3     ← 第1个位置选谁
       /\   |  ...
     x2 x3  x1 x3    ← 第2个位置选谁
     |   |
    x3  x2            ← 第3个位置
```

- 叶节点数：$n!$
- 遍历时间：$\Theta(n!)$
- **典型问题：TSP、人员分配**

------

## 2.3 例子1：0-1背包问题

### 问题

| 物品 | 重量 $w_i$ | 价值 $v_i$ |
| ---- | ---------- | ---------- |
| 1    | 16         | 45         |
| 2    | 15         | 25         |
| 3    | 15         | 25         |

**背包容量 $W = 30$**

### 搜索树（子集树）

```
                          root
                       剩余容量=30
                        /        \
                   选物品1        不选物品1
                   w=16,v=45      w=0,v=0
                  剩余容量=14     剩余容量=30
                  /       \       /       \
            选物品2    不选物品2  选物品2  不选物品2
            w=15>14!    v=45    w=15,v=25  v=0
            ❌剪枝    剩余=14   剩余=15    剩余=30
                      /    \    /    \     /    \
                   选3  不选3 选3 不选3  选3  不选3
                  15>14  v=45 v=50 v=25  v=25  v=0
                  ❌剪枝 ★    ★★   ★     ★
```

### 详细过程

| 步骤 | 节点    | 决策      | 当前重量    | 当前价值 | 剩余容量 | 操作                               |
| ---- | ------- | --------- | ----------- | -------- | -------- | ---------------------------------- |
| 1    | A       | 选物品1   | 16          | 45       | 14       | 进入左子树                         |
| 2    | B       | 选物品2？ | 16+15=31>30 | —        | —        | **剪枝！回溯**                     |
| 3    | B       | 不选物品2 | 16          | 45       | 14       | 进入右子树                         |
| 4    | D       | 选物品3？ | 16+15=31>30 | —        | —        | **剪枝！回溯**                     |
| 5    | D       | 不选物品3 | 16          | 45       | 14       | **叶节点，记录解={1}，价值45**     |
| 6    | 回溯到A | 不选物品1 | 0           | 0        | 30       | 进入右子树                         |
| 7    | E       | 选物品2   | 15          | 25       | 15       | 进入左子树                         |
| 8    | F       | 选物品3   | 15+15=30≤30 | 50       | 0        | **叶节点，记录解={2,3}，价值50** ★ |
| 9    | 回溯    | 不选物品3 | 15          | 25       | 15       | 叶节点，价值25                     |
| 10   | 回溯到E | 不选物品2 | 0           | 0        | 30       | 继续                               |
| 11   | G       | 选物品3   | 15          | 25       | 15       | 叶节点，价值25                     |
| 12   | G       | 不选物品3 | 0           | 0        | 30       | 叶节点，价值0                      |

**最优解：选物品2和物品3，总价值 = 50，总重量 = 30**

### 剪枝效果

- 不剪枝：需要检查 $2^3 = 8$ 个叶节点
- 剪枝后：只检查了 5 个叶节点，跳过了 3 个分支

------

## 2.4 回溯法的两种剪枝函数

### ① 约束函数（Constraint Function）

> 检查当前选择是否**违反约束条件**

- 0-1背包：当前重量 > 背包容量 → 剪枝
- N皇后：当前放置的皇后互相攻击 → 剪枝

### ② 界限函数（Bound Function）

> 检查当前路径是否**有可能得到比已知解更好的解**

- 0-1背包：当前价值 + 剩余物品的价值上界 ≤ 已知最优价值 → 剪枝
- TSP：当前路径代价 + 剩余代价下界 ≥ 已知最短回路 → 剪枝

------

## 2.5 例子2：N皇后问题（回溯经典）

### 问题

在 $n \times n$ 棋盘上放 $n$ 个皇后，使得任意两个皇后不同行、不同列、不同对角线。

### 4皇后搜索过程

```
第1行：尝试列1
  第2行：尝试列1 ❌同列
         尝试列2 ❌同对角线
         尝试列3 ✓
    第3行：尝试列1 ❌对角线冲突
           尝试列2 ❌同列（与第2行的列3不冲突，但与第1行对角线冲突）
           尝试列3 ❌同列
           尝试列4 ❌对角线
           全部失败 → 回溯！
  第2行：尝试列4 ✓
    第3行：尝试列1 ❌
           尝试列2 ✓
      第4行：尝试列1 ❌
             尝试列2 ❌
             尝试列3 ❌
             尝试列4 ❌
             全部失败 → 回溯！
    第3行：继续...
    ...
```

## 2.6 回溯法通用框架

### 子集树框架

```python
def backtrack_subset(i):      # 第i层，决定第i个元素选不选
    if i > n:                 # 到达叶节点
        记录/更新解
        return
    
    # 尝试"选"
    if 约束函数(i, 选) and 界限函数(i, 选):
        x[i] = 1
        backtrack_subset(i + 1)
    
    # 尝试"不选"
    if 约束函数(i, 不选) and 界限函数(i, 不选):
        x[i] = 0
        backtrack_subset(i + 1)
```

**时间复杂度：** 最坏 $O(2^n)$，实际通过剪枝大幅减少

### 排列树框架

```python
def backtrack_permutation(i):  # 第i个位置放什么
    if i > n:
        记录/更新解
        return
    
    for j in 剩余可选元素:
        if 约束函数(i, j) and 界限函数(i, j):
            x[i] = j
            标记j为已使用
            backtrack_permutation(i + 1)
            取消标记j              # 回溯
```

**时间复杂度：** 最坏 $O(n!)$，实际通过剪枝大幅减少

## 2.7 回溯法 vs 分支界限法

| 特性           | 回溯法                   | 分支界限法                       |
| -------------- | ------------------------ | -------------------------------- |
| **搜索策略**   | 深度优先（DFS）          | 最佳优先（Best-First）或广度优先 |
| **数据结构**   | 栈（递归调用栈）         | 堆（优先队列）                   |
| **搜索方式**   | 一条路走到底，不行就回头 | 全局选最有希望的节点扩展         |
| **空间复杂度** | $O(n)$（只存当前路径）   | $O(2^n)$（存所有活节点）         |
| **找解速度**   | 可能较慢（走了很多弯路） | 通常更快（直奔最优方向）         |
| **适用场景**   | 空间受限，或只需找一个解 | 需要找最优解，空间充足           |

# 三、所有算法终极对比表

| 算法           | 数据结构 | 扩展策略      | 剪枝        | 最优性保证           | 时间              | 空间   |
| -------------- | -------- | ------------- | ----------- | -------------------- | ----------------- | ------ |
| **BFS**        | 队列     | 逐层          | 无          | 最浅解               | $O(n)$            | $O(n)$ |
| **DFS**        | 栈       | 最深          | 无          | 无                   | $O(n)$            | $O(d)$ |
| **回溯**       | 递归栈   | DFS+剪枝      | 约束+界限   | 可保证               | $O(2^n)$或$O(n!)$ | $O(n)$ |
| **爬山法**     | 有序栈   | 启发式DFS     | 无          | **局部**最优         | $O(n)$            | $O(d)$ |
| **Best-First** | 堆       | 全局最优节点  | 无          | 较好                 | $O(n\log n)$      | $O(n)$ |
| **分支界限**   | 堆+剪枝  | 全局最优+剪枝 | 界限剪枝    | **全局最优**         | 指数级            | 指数级 |
| **A\***        | 堆       | $f=g+h$       | $h\leq h^*$ | **全局最优（证明）** | 视$h$而定         | 指数级 |
