# 第8章 近似算法 复习笔记

## 8.1 近似算法概述

### 背景

- 许多组合优化问题是NP完全/NP难问题，不太可能存在多项式时间精确算法
- 近似算法：在多项式时间内求得近似最优解

### 近似比（Ratio Bound）

- 对于最小化问题：近似比 ρ = C / C*
- 对于最大化问题：近似比 ρ = C* / C
- 其中 C 为近似解代价，C* 为最优解代价
- 近似比 ρ ≥ 1，越接近1越好

### 近似算法的分类

| 类型         | 说明                     |
| ------------ | ------------------------ |
| 基于组合优化 | 利用问题结构特性         |
| 基于贪心策略 | 每步做局部最优选择       |
| 基于局部优化 | 通过局部修改逐步改进     |
| 基于动态规划 | 通过rounding缩小状态空间 |
| 基于线性规划 | LP松弛+舍入/primal-dual  |

## 8.2 基于组合优化的近似算法

### 8.2.1 顶点覆盖问题（Vertex Cover）

#### 问题定义

- **输入**：无向图 G=(V, E)
- **输出**：C⊆V，满足：
  1. 对∀(u,v)∈E，u∈C 或 v∈C 或 {u,v}⊆C
  2. C 是满足条件(1)的**最小**集合
- **性质**：顶点覆盖问题是NP完全问题

#### 算法：APPROX-Vertex-Cover

```
APPROX-Vertex-Cover(G)
1. C = ∅
2. E' = E[G]
3. While E' ≠ ∅ Do
4.    任取 (u,v) ∈ E'
5.    C = C ∪ {u, v}
6.    从E'中删除所有与u或v相连的边
7. Return C
```

**基本思想**：每次任取一条边，将两个端点都加入覆盖集，然后删除所有与这两个端点关联的边。

#### 时间复杂度

$$T(G) = O(|E|)$$

#### 近似比分析

**定理**：APPROX-Vertex-Cover 的近似比为 **2**。

**证明**：

1. 令 A = {(u,v) | (u,v)是算法第4步选中的边}
2. A 中无相邻接边（因为选中一条边后，邻接边都被删除）
3. 每次第5步增加两个结点到C，所以 |C| = 2|A|
4. 设 C* 是最优解，C* 必须覆盖 A 中每条边
5. 由于 A 中无邻接边，C* 至少包含 A 中每条边的一个端点
6. 因此 |A| ≤ |C*|
7. 所以 |C| = 2|A| ≤ 2 |C* | ，即 |C|/|C*| ≤ 2

### 8.2.2 装箱问题（Bin Packing）- First Fit Decreasing

#### 问题定义

- **输入**：n个物品，每个物品体积 ∈ (0,1]，箱子容量为1
- **输出**：用最少数量的箱子装下所有物品
- **性质**：NP完全问题

#### 算法思想（First Fit）

按顺序考虑每个物品，将其放入第一个能容纳它的箱子中；若没有能容纳的箱子则开启新箱子。

#### 时间复杂度

$$T(n) = O(n^2)$$

#### 近似比

近似比为 2（即使用的箱子数不超过最优解的2倍）

### 8.2.3 旅行商问题（TSP with Triangle Inequality）

#### 问题定义

- **输入**：
  - 完全无向图 G=(V,E)
  - 代价函数 C: E → 非负整数，满足三角不等式：C(u,w) ≤ C(u,v) + C(v,w)
- **输出**：具有最小代价的Hamilton环
- **注意**：不满足三角不等式的TSP问题不存在常数近似比的近似算法（除非P=NP）

#### 算法：APPROX-TSP-TOUR

```
APPROX-TSP-TOUR(G, C)
1. 选择一个 r∈V[G] 作为生成树的根
2. 调用 MST-Prim(G, C, r) 生成最小生成树 T
3. 先序遍历 T，形成有序结点表 L
4. 按照 L 中的顺序访问各结点，形成哈密顿环
```

**基本思想**：

1. 构造最小生成树
2. 先序遍历最小生成树，构造TSP的解

#### 时间复杂度

- 第2步：O(|V|²)（Prim算法）
- 第3步：O(|V|)
- 第4步：O(|V|)
- **总计：T(G) = O(|V|²)**

#### 近似比分析

**定理**：APPROX-TSP-TOUR 的近似比为 **2**。

**证明**：

1. 设 H* 是最优Hamilton环，H 是算法产生的近似解
2. 从 H* 中删除一条边 → 得到生成树 T'
3. 因为 T 是最小生成树：C(T) ≤ C(T') ≤ C(H*)
4. T 的 full walk W 经过每条边两次：C(W) = 2C(T) ≤ 2C(H*)
5. W 不是Hamilton环（某些结点访问多次）
6. 根据三角不等式，删除重复访问不增加代价
7. 先序遍历（删除重复访问）得到 H：C(H) ≤ C(W)
8. 因此 **C(H) ≤ 2C(H\*)**

## 8.3 基于贪心策略的近似算法

### 8.3.1 最小时间调度问题（Minimum Makespan Scheduling）

#### 问题定义

- **输入**：m台相同机器，n个作业（各作业处理时间已知）
- **输出**：将作业分配给机器，使最大完成时间（makespan）最小
- **性质**：NP完全问题

#### 算法（贪心调度 - LPT/List Scheduling）

```
贪心调度算法:
对每个作业，将其分配给当前负载最小的机器
```

**贪心选择**：选择具有最短任务队列（当前完成时间最早）的机器

#### 时间复杂度

$$T = O(nm)$$

#### 近似比

**定理**：贪心调度算法的近似比为 **2**。

------

### 8.3.2 最小集合覆盖问题（Minimum Set Cover）

#### 问题定义

- **输入**：有限集 X，X 的子集族 F，X = ∪_{S∈F} S
- **输出**：C⊆F，满足：
  1. X = ∪_{S∈C} S（覆盖所有元素）
  2. |C| 最小
- **性质**：NP完全问题

#### 算法：Greedy-Set-Cover

```
Greedy-Set-Cover(X, F)
1. U ← X          // U是尚未被覆盖的元素集
2. C ← ∅
3. While U ≠ ∅ Do
4.    Select S∈F 使得 |S∩U| 最大    // 贪心选择
5.    U ← U - S
6.    C ← C ∪ {S}
7. Return C
```

**贪心选择**：每次选择能覆盖最多未被覆盖元素的子集

#### 时间复杂度

- 循环次数至多 |X| 次（最多min(|X|,|F|)次）
- 每次计算 S∩U 需要 O(|X|)
- 第4步需要 O(|F|·|X|)
- **总计：T(X,F) = O(|F|·|X|·min(|X|,|F|))**

#### 近似比分析

**定理**：Greedy-Set-Cover 的近似比为 H(max{|S| : S∈F})，其中 H(d) = Σ_{i=1}^{d} 1/i（调和级数）。

**推论**：近似比为 ln|X| + 1。

**证明思路**：

1. 定义代价分摊：当 Si 被选入 C 时，将代价1均匀分配到 Si 首次覆盖的所有元素上
2. 对于被 Si 首次覆盖的元素 x：
   $$c_x = \frac{1}{|S_i - (S_1 \cup S_2 \cup ... \cup S_{i-1})|}$$
3. 算法代价 = Σ_{x∈X} c_x
4. 证明对任意 S∈F：Σ_{x∈S} c_x ≤ H(|S|)
5. 因此：|C| = Σ_{x∈X} c_x ≤ Σ_{S∈C*} Σ_{x∈S} c_x ≤ |C*|·H(max|S|)

**关键步骤的详细证明**：

令 u_i = |S - (S_1∪...∪S_i)| 为选择 S_i 后 S 中未覆盖的点数，u_0 = |S|

由贪心选择性质：Si 覆盖的新元素数 ≥ u_{i-1}（因为 S 可以覆盖 u_{i-1} 个新元素）

$$\sum_{x\in S} c_x = \sum_{i=1}^{k} \frac{u_{i-1}-u_i}{u_{i-1}} \leq \sum_{i=1}^{k} \sum_{j=u_i+1}^{u_{i-1}} \frac{1}{j} \leq \sum_{j=1}^{u_0} \frac{1}{j} = H(|S|)$$



这个证明是算法界非常经典的“势能法/代价分摊法”的代表作。它之所以难懂，是因为它没有直接去算贪心算法选了几个集合，而是玩了一个“会计游戏”。
不要被那些求和符号吓倒，我们用最直观的花钱买东西的比喻，把这三步拆解得明明白白。
核心思想：换个角度看问题
假设你要覆盖所有元素，每次选一个集合，你要支付 1 块钱。
贪心算法的总花费 = 选了几个集合 = $|\mathcal{C}|$
最优算法的总花费 = 选了几个集合 = $|\mathcal{C}^*|$
我们想证明 $\frac{|\mathcal{C}|}{|\mathcal{C}^*|} \leq H(d)$。
直接比太难了。证明用了一个巧妙的中间人：每个元素 $x$ 身上揣的钱 $c_x$。
Step 1 通俗版：贪心是怎么发钱的？
规则：贪心每次选一个集合 $S_i$，掏出 1 块钱。这 1 块钱只分给那些第一次被覆盖的元素，大家平分。
如果 $S_i$ 首次覆盖了 5 个新元素，那这 1 块钱就分成 5 份，每个元素拿到 $c_x = 1/5$。
如果 $S_i$ 首次覆盖了 100 个新元素，那每个元素拿到 $c_x = 1/100$。
关键观察：贪心一共掏了 $k$ 块钱（选了 $k$ 个集合），每一块钱都分给了元素。所以：
贪心选的集合数 = 所有元素手里的钱之和，即 $|\mathcal{C}| = \sum c_x$。
*现在，问题转化成了：只要证明每个元素手里的钱不多，贪心总花费就不多。*
Step 2 通俗版：随便拿个集合，它里面的元素总共能有多少钱？（核心！）
这是整个证明最精彩、也最绕的地方。
我们随便从子集族里抓一个集合 $S$，假设它有 $d$ 个元素。我们来算算这 $d$ 个元素手里总共拿了多少钱（$\sum_{x \in S} c_x$）。
注意：这 $d$ 个元素，不一定都是被我们抓出来的这个集合 $S$ 覆盖的！它们可能是被贪心算法之前选的其他集合覆盖的。
想象一个时间线，贪心算法一轮一轮地选集合。对于这 $d$ 个元素：
初始时，这 $d$ 个都没被覆盖，设 $u_0 = d$。
第 1 轮，可能覆盖了 2 个，剩下没覆盖的 $u_1 = d-2$。
第 2 轮，可能覆盖了 1 个，剩下没覆盖的 $u_2 = d-3$。
……
直到最后全被覆盖，$u_k = 0$。
关键逻辑：为什么它们拿到的钱不多？
因为贪心算法是“贪心”的！
看第 $i$ 轮。此时这 $d$ 个元素里，还有 $u_{i-1}$ 个没被覆盖。
如果我们抓出来的这个集合 $S$ 此刻能上场，它能覆盖 $u_{i-1}$ 个新元素。
但是！贪心算法没选 $S$，它选了 $S_i$！
为什么选 $S_i$？因为 $S_i$ 覆盖的新元素比 $S$ 更多（或至少一样多）！
所以，第 $i$ 轮被覆盖的那些元素，他们是跟着大佬 $S_i$ 混的，分钱是按 $S_i$ 的人数平分的。由于 $S_i$ 人数比 $S$ 多，所以分到手里的钱 $c_x$ 比跟着 $S$ 混要少！
这就是那个核心不等式：$|S_i \cap U_i| \geq u_{i-1}$ 的直觉。
$|S_i \cap U_i|$ 是分钱的人数（大佬 $S_i$ 带来的新兄弟）
$u_{i-1}$ 是原本可能的人数（集合 $S$ 剩下的兄弟）
人数多，分到的钱 $\frac{1}{|S_i \cap U_i|}$ 自然 $\leq \frac{1}{u_{i-1}}$。
接下来是最神奇的放缩：把零碎的钱凑成调和级数
在第 $i$ 轮，这 $d$ 个元素里有 $(u_{i-1} - u_i)$ 个人拿到了钱，每个人最多拿到 $\frac{1}{u_{i-1}}$。
所以第 $i$ 轮，这 $d$ 个人拿到的总钱数 $\leq \frac{u_{i-1} - u_i}{u_{i-1}}$。
这个分数看着烦，怎么变？拆开它！
假设 $u_{i-1} = 5$，$u_i = 2$。说明这轮覆盖了 3 个人。
这 3 个人每人最多拿 $1/5$，总共拿 $3 \times (1/5) = 3/5$。
但是，这 3 个人是在剩余的 5 个人里挑出来的。本来这 3 个人应该排在第 3、4、5 个被覆盖。
我们给他们涨工资！排在第 3 个的给 $1/3$，第 4 个给 $1/4$，第 5 个给 $1/5$。
$$ \frac{1}{5} + \frac{1}{5} + \frac{1}{5} \leq \frac{1}{3} + \frac{1}{4} + \frac{1}{5} $$
这不就是常识吗：分母越小，值越大。
写成公式就是：
$$\frac{u_{i-1} - u_i}{u_{i-1}} = \sum_{j=u_i+1}^{u_{i-1}} \frac{1}{u_{i-1}} \leq \sum_{j=u_i+1}^{u_{i-1}} \frac{1}{j}$$
拼图游戏：
第 1 轮，剩下 $u_1$ 到 $u_0$ 的人拿钱，钱数 $\leq \frac{1}{u_1+1} + \dots + \frac{1}{u_0}$
第 2 轮，剩下 $u_2$ 到 $u_1$ 的人拿钱，钱数 $\leq \frac{1}{u_2+1} + \dots + \frac{1}{u_1}$
……
因为这些数是 $d, d-1, d-2 \dots 1$ 递减的，所以这些区间完美首尾相接，没有任何重叠！
把所有轮次加起来，正好是从 1 加到 $d$：
$$ \leq \frac{1}{1} + \frac{1}{2} + \frac{1}{3} + \dots + \frac{1}{d} = H(d) $$
Step 2 结论：随便抓一个大小为 $d$ 的集合 $S$，它里面的元素手里总钱数绝对不超过 $H(d)$！
Step 3 通俗版：收网算账
现在我们让最优解 $\mathcal{C}^*$ 出场。
最优解选了 $|\mathcal{C}^*|$ 个集合，就覆盖了全场。
全场元素的钱 $\sum_{x \in X} c_x$ 怎么算？
我们可以按最优解的集合来分类算！
$$ \sum_{x \in X} c_x \leq \sum_{S \in \mathcal{C}^*} \left( \sum_{x \in S} c_x \right) $$
*(注意：因为有的元素同时属于最优解的好几个集合，这样算会重复计算，所以是 $\leq$)*
由 Step 2 我们知道，任何一个集合 $S$ 里的元素总钱数 $\sum_{x \in S} c_x \leq H(|S|) \leq H(d)$。
所以：
$$ \sum_{x \in X} c_x \leq \sum_{S \in \mathcal{C}^*} H(d) $$
最优解有几个集合？有 $|\mathcal{C}^*|$ 个！所以就是 $|\mathcal{C}^*|$ 个 $H(d)$ 相加：
$$ \sum_{x \in X} c_x \leq |\mathcal{C}^*| \cdot H(d) $$
别忘了 Step 1 说的：贪心选的集合数 $|\mathcal{C}| = \sum_{x \in X} c_x$。
代入进去：
$$ |\mathcal{C}| \leq |\mathcal{C}^*| \cdot H(d) $$
两边同除以 $|\mathcal{C}^*|$，近似比 $\frac{|\mathcal{C}|}{|\mathcal{C}^*|} \leq H(d)$ 出炉！
总结一句话记住这个证明：
贪心付钱，元素收钱；因为贪心选的集合人更多，所以元素分到的钱更少；把元素按被覆盖顺序排好，每人最多拿倒数，正好凑成调和级数；最后拿最优解的集合去圈这些元素，发现钱的总数超不过最优解的个数乘以 $H(d)$。

### 8.3.3 次模函数最大化（Submodular Function Maximization）

#### 次模函数定义

给定有限集 U，集合函数 f: 2^U → ℝ 是**次模函数**，如果满足以下等价条件之一：

1. **条件1（集合凸性）**：f(A) + f(B) ≥ f(A∪B) + f(A∩B)，对∀A,B⊆U
2. **条件2（边缘增益递减）**：f(A∪{x}) - f(A) ≥ f(B∪{x}) - f(B)，对∀A⊆B⊆U 且 x∈U-B

**直观含义**：

- 条件1：集合运算后的函数值 ≤ 函数值的组合（类似凸性）
- 条件2：x 在小集合上的边际增益 ≥ x 在大集合上的边际增益（递减边际收益）

#### 次模函数的性质

**(B) 基本性质**：若 f 是次模函数，则

1. f(B∪A) - f(A) ≤ Σ_{x∈B} [f(A∪{x}) - f(A)]
2. f(B∪A) - f(A) ≤ Σ_{x∈B\A} [f(A∪{x}) - f(A)]
3. 若 A⊆B，则 f(B) - f(A) ≤ Σ_{x∈B\A} [f(A∪{x}) - f(A)]

**(O) 运算封闭性**：

1. h(X) = cf(X)（c≥0）是次模函数
2. h(X) = f(X) + g(X) 是次模函数
3. h(X) = f(X∩T) 是次模函数
4. h(X) = f(U-X) 是次模函数
5. 若 f 单调且 c≥0，则 h(X) = min{c, f(X)} 是单调次模函数

**(I) 标准化**：f(X) 是次模 ⟺ g(X) = f(X) - f(∅) 是次模

**(II) 集合增量增益递减**：f(X∪A) - f(X) ≥ f(Y∪A) - f(Y)，对 X⊆Y⊆U 且 A⊆U-Y

**(IV) 亚可加性**：标准化次模函数满足 f(A∪B) ≤ f(A) + f(B)

#### 次模函数实例

| 实例     | 定义                        | 说明                         |
| -------- | --------------------------- | ---------------------------- |
| 加权函数 | f(A) = Σ_{x∈A} w(x)         | 线性函数（既是次模也是超模） |
| 预算函数 | f(A) = min{Σ_{x∈A} w(x), b} | 有上界的加权函数             |
| 秩函数   | f(A) = dim(span(A))         | 向量集张成空间的维数         |
| 覆盖函数 | f(A) =                      | ∪_{s∈A} s                    |
| 割函数   | f(A) = Σ_{uv∈δ(A)} w(uv)    | 图的割的权重                 |

#### 问题：基数约束下的单调次模最大化

**问题**：

- 输入：空间 U，非负单调次模函数 f: 2^U → ℝ⁺，约束 |A| ≤ k
- 输出：max f(A)，s.t. |A| ≤ k

**贪心算法**：

```
1. A ← ∅
2. While |A| < k Do
3.    x* ← argmax_{x∈U-A} {f(A∪{x}) - f(A)}    // 贪心选择
4.    A ← A ∪ {x*}
5. Return A
```

**近似比**：f(A) ≥ (1 - 1/e) · f(A_opt)，即近似比为 **(1-1/e)⁻¹ ≈ 1.58**

**证明**：

令 A_i = {x_1,...,x_i} 为第 i 轮后的结果：

$$f(A_i) - f(A_{i-1}) = f(A_{i-1}\cup{x_i}) - f(A_{i-1})$$
$$\geq \max_{x\in A_{OPT}} [f(A_{i-1}\cup{x}) - f(A_{i-1})]$$ (贪心选择)
$$\geq \frac{1}{k} \sum_{x\in A_{OPT}} [f(A_{i-1}\cup{x}) - f(A_{i-1})]$$ (最大值≥平均值)
$$\geq \frac{1}{k} [f(A_{OPT}\cup A_{i-1}) - f(A_{i-1})]$$ (次模性质B)
$$\geq \frac{1}{k} [f(A_{OPT}) - f(A_{i-1})]$$ (单调性)

因此：f(A_OPT) - f(A_i) ≤ (1-1/k)[f(A_OPT) - f(A_{i-1})]

递推得：f(A_OPT) - f(A_k) ≤ (1-1/k)^k [f(A_OPT) - f(∅)]

由于 (1-1/k)^k ≤ 1/e，所以 **f(A) ≥ (1-1/e)f(A_OPT)**

**注意**：当代价函数不满足单调性时，近似比没有保障。

#### 问题：背包约束下的次模最大化

**问题**：

- 输入：空间 U，加权函数 w，非负次模函数 f，预算 b
- 输出：max f(A)，s.t. w(A) ≤ b

**算法**：

```
1. A ← ∅
2. While U ≠ ∅ Do
3.    x* ← argmax_{x∈U} {[f(A∪{x})-f(A)] / w(x)}    // 性价比贪心
4.    If w(A∪{x*}) ≤ b Then A ← A∪{x*}
5.    U ← U - {x*}
6. x₀ ← argmax_{x∈U} f({x})
7. S ← A 和 {x₀} 中次模函数值较大者
8. Return S
```

**近似比**：f(S) ≥ 0.5(1-e⁻¹)f(A_opt)

#### 问题：拟阵约束下的次模最大化

**问题**：

- 输入：空间 U，非负次模函数 f，拟阵约束 (U, I)
- 输出：max f(A)，s.t. A∈I

**算法**：

```
1. A ← ∅
2. While U ≠ ∅ Do
3.    x* ← argmax_{x∈U} {f(A∪{x})-f(A)}
4.    If A∪{x*} ∈ I Then A ← A∪{x*}
5.    U ← U - {x*}
6. Return A
```

**近似比**：f(A) ≥ 0.5·f(A_opt)，即近似比为 **2**

## 8.4 基于局部搜索的近似算法

### 8.4.1 最大割问题（Maximum Cut）

#### 问题定义

- **输入**：加权图 G=(V,E)，权值函数 W: E→ℕ
- **输出**：S⊆V，使得割 W(S) = Σ_{u∈S, v∈V-S} W(uv) 最大

#### 关键概念

**局部修改操作**：交换顶点 v 在 S 和 V-S 中的位置

**cost函数**：

- 若 v∈S：cost(v,S) = Σ_{u∈S: uv∈E} W(uv) - Σ_{u∈V-S: vu∈E} W(vu)
- 若 v∈V-S：cost(v,S) = Σ_{u∈V-S: vu∈E} W(vu) - Σ_{u∈S: uv∈E} W(uv)

当 cost(v,S) > 0 时，交换 v 的位置可以增加割的权重。

#### 算法：ApproxMaxCut

```
ApproxMaxCut(G(V,E))
输入: 加权图 G=(V,E)，权值函数 W
输出: S⊆V 使得割最大

1. S ← {u}           // u是V中任意顶点
2. Repeat
3.    任取使得 cost(v,S)>0 的顶点v，交换v在S和V-S中的位置
4. Until 不存在 v∈V 使得 cost(v,S)>0
5. 输出 S
```

#### 算法终止性

- 每次交换操作使割的权重至少增加1
- 割的权重上界为 Σ_{u,v∈V} W(uv)
- 因此算法一定在有限步内终止

#### 近似比分析

**定理**：ApproxMaxCut 的近似比为 **2**。

**证明**：

算法终止时，对∀v∈V，cost(v,S) ≤ 0

**情况1**：v∈S，cost(v,S) ≤ 0 意味着：
$$\sum_{u\in S} W(uv) \leq \sum_{u\in V-S} W(vu)$$

由此可得：
$$\frac{1}{2}\sum_{u\in V} W(uv) \leq \sum_{u\in V-S} W(vu)$$

**情况2**：v∈V-S，类似地：
$$\frac{1}{2}\sum_{u\in V} W(uv) \leq \sum_{u\in S} W(uv)$$

将所有顶点的不等式相加：
$$\sum_{uv\in E} W(uv) \leq \frac{1}{2}\sum_{v\in V}\sum_{u\in V} W(uv) \leq 2W(S)$$

因此：W(S*) ≤ Σ_{uv∈E} W(uv) ≤ 2W(S)

即 **W(S\*)/W(S) ≤ 2**

## 8.5 基于动态规划的近似算法

### 8.5.1 0-1背包问题的FPTAS

#### 问题回顾：0-1背包问题

**递归方程**：
$$m(i,j) = \begin{cases} 0 & \text{if } i=n, j<w_n \ v_n & \text{if } i=n, j\geq w_n \ m(i+1,j) & \text{if } j<w_i \ \max{m(i+1,j), m(i+1,j-w_i)+v_i} & \text{if } j\geq w_i \end{cases}$$

#### FPTAS算法：ApproxPacking

**基本思想**：通过 Rounding（舍入）缩小价值的取值范围，从而降低DP的时间复杂度。

```
ApproxPacking(W[1:n], V[1:n], C, ε)
输入：容量C，重量数组W，价值数组V，误差参数ε
输出：0-1背包问题的(1-ε)-近似解

1. V_max = max_{1≤i≤n} v_i
2. δ = ε·V_max/n
3. For i=1 to n Do
       V[i] ← ⌊V[i]/δ⌋            // Rounding down
4. 以(W[1:n], V[1:n], C)为输入，调用DP算法，得到(x₁,...,xₙ)
5. 输出(x₁,...,xₙ)作为原始问题的近似解
```

**Rounding处理**：

- V_max = max v_i
- δ = εV_max/n
- v'_i = ⌊v_i/δ⌋
- 有 δv'_i ≤ v_i ≤ δ(v'_i + 1)

#### 时间复杂度分析

Rounding后的总价值上界：
$$V' = \sum v'*i = \sum \lfloor v_i/\delta \rfloor \leq \sum v_i/\delta \leq n \cdot V*{max}/\delta = n^2/\varepsilon$$

DP算法时间复杂度：O(n·V') = **O(n³/ε)**

这是关于 n 和 1/ε 的多项式 → **FPTAS**

#### (1-ε)-近似性证明

设：

- (z₁,...,zₙ) 是原始问题的最优解，Z = Σv_iz_i ≥ V_max
- (x₁,...,xₙ) 是算法返回的解

$$X = \sum v_i x_i \geq \sum \delta v'_i x_i = \delta \sum v'_i x_i \geq \delta \sum v'*i z_i$$
$$\geq \sum (v_i - \delta)z_i = \sum v_iz_i - n\delta = Z - \varepsilon V*{max} \geq Z - \varepsilon Z = (1-\varepsilon)Z$$

因此 **X ≥ (1-ε)Z**，即算法是 (1-ε)-近似的。

### 8.5.2 子集和问题（Subset Sum）

#### 问题定义

- **输入**：S = {x₁, x₂, ..., xₙ}（正整数），目标值 t
- **输出**：A⊆S，使得 Σ_{x∈A} x ≤ t 且 Σ_{x∈A} x 最大

#### 精确算法（指数时间）

```
Exact-Subset-Sum(S, t)
1. n ← |S|
2. L₀ ← <0>
3. For i=1 To n Do
4.    L_i ← Merge-List(L_{i-1}, L_{i-1}+x_i)
5.    删除L_i中所有大于t的元素
6. Return L_n中最大元素
```

**递归关系**：L_i = L_{i-1} ∪ (L_{i-1} + x_i)（≤t的部分）

**时间复杂度**：O(2ⁿ)（最坏情况下 |L_i| = 2ⁱ）

#### 修剪算法 Trim

**思想**：对于多个相近的元素，只保留一个代表。

**修剪参数** δ（0<δ<1）：对于L中被删除的元素y，L'中存在z满足 y(1-δ) ≤ z ≤ y

```
Trim(L=<y₁,...,yₘ>, δ)    // y_i已排序
1. L' ← <y₁>
2. last ← y₁
3. For i=2 To m Do
4.    If y_i(1-δ) > last Then    // last不能代表y_i
5.       将y_i加入L'末尾
6.       last ← y_i
7. Return L'
```

时间复杂度：O(m)

#### FPTAS：Approx-Subset-Sum

```
Approx-Subset-Sum(S, t, ε)
输入：S={x₁,...,xₙ}, t≥0, 0<ε<1

1. n ← |S|
2. L₀ ← <0>
3. For i=1 To n Do
4.    L_i ← Merge-List(L_{i-1}, L_{i-1}+x_i)
5.    L_i ← Trim(L_i, ε/n)    // 修剪参数δ=ε/n
6.    从L_i中删除大于t的元素
7. z ← L_n中最大值
8. Return z
```

#### 性能分析

**定理**：Approx-Subset-Sum 是子集和问题的相对误差为 ε 的**完全多项式时间近似模式（FPTAS）**。

**近似性证明**（往证 C*(1-ε) ≤ z）：

对 i 作归纳法证明：∀y∈P_i, y≤t，∃z'∈L_i 使得 (1-ε/n)ⁱy ≤ z' ≤ y

由于 C*∈P_n，存在 z'∈L_n 使得 (1-ε/n)ⁿC* ≤ z' ≤ z ≤ C*

由 (1-ε/n)ⁿ ≥ 1-ε（当n≥1），得 **(1-ε)C\* ≤ z**

**时间复杂度分析**：

修剪后 L_i 中相邻元素满足 y_j/y_{j-1} > 1/(1-ε/n)

$$|L_i| \leq 2 + \log_{1/(1-\varepsilon/n)} t \leq 2 + n\ln t/\varepsilon$$

算法运行时间是 |L_i| 的多项式 → **O(n²·ln t/ε)** → n和1/ε的多项式

### 8.5.3 装箱问题的近似模式

#### 算法框架

```
ApproxBinPacking(I, ε)
输入：装箱问题实例I，相对误差参数ε<1

1. (I', I_down, I_up) ← Transform(I, ε)    // 变换
2. S' ← DynamicSearch(I_up, 1/ε²)          // DP精确求解
3. S ← SolutionTrans(S', I, ε)              // 解转换
4. 输出S
```

#### 变换算法 Transform(I, ε)

```
1. 删除I中所有体积小于ε的物品，得I'，n=|I'|
2. 将I'中物品按体积递增排序，分为K=1/ε²组，每组n/K=nε²个
3. 各组内物品体积修改为组内最大体积 → I_up
4. 各组内物品体积修改为组内最小体积 → I_down
5. 输出I', I_down, I_up
```

时间复杂度：O(n log n)

#### 关键性质

1. 每个箱子至多容纳 L=1/ε 个物品
2. I_down 和 I_up 中物品体积至多有 K=1/ε² 个不同取值
3. Opt(I_down) ≤ Opt(I')，且 n·ε ≤ Opt(I')
4. **Opt(I_up) ≤ (1+ε)·Opt(I')**

#### 性质4的证明

由 I_down 的最优解 S' 构造 I_up 的可行解：

1. 删除 S' 中属于 I_down 第1组的物品
2. 对 i≥2，将 I_down 第 i 组物品替换为 I_up 第 i-1 组物品
3. I_up 最后每个剩余物品用新箱子（最多 nε² 个）

Opt(I_up) ≤ Opt(I_down) + nε² ≤ Opt(I') + ε·Opt(I') = (1+ε)Opt(I')

#### 解转换 SolutionTrans

```
1. For I中体积小于ε的每个物品i Do
2.    If S中存在箱子能容纳i Then 装入该箱子
3.    Else 开启新箱子装入i
4. 输出S
```

#### 近似比

ApproxBinPacking 的近似比为 **1+2ε**（当 ε≤0.5 时，1/(1-ε) ≤ 1+2ε）

#### 总时间复杂度

O(m^{1/ε²})

## 8.6 基于线性规划的近似算法

### 8.6.1 线性规划基础

#### 线性规划标准形式

**最小化问题（原始问题）**：
$$\min \sum_{j=1}^n c_j x_j$$
$$\text{s.t.} \sum_{j=1}^n a_{ij} x_j \geq b_i, \quad i=1,...,m$$
$$x_j \geq 0, \quad j=1,...,n$$

**最大化问题（对偶问题）**：
$$\max \sum_{i=1}^m b_i y_i$$
$$\text{s.t.} \sum_{i=1}^m a_{ij} y_i \leq c_j, \quad j=1,...,n$$
$$y_i \geq 0, \quad i=1,...,m$$

#### 对偶定理

| 定理           | 内容                                                 |
| -------------- | ---------------------------------------------------- |
| **弱对偶定理** | 若x,y分别是原、对偶问题的可行解，则 cx ≥ bᵀy         |
| **强对偶定理** | 原问题最优值有限 ⟺ 对偶问题最优值有限，且 cx* = bᵀy* |

#### 互补松弛条件

x和y分别是原始和对偶问题的最优解 ⟺ 同时满足：

**原始互补松弛**：对∀j：x_j = 0 或 Σᵢ a_{ij}y_i = c_j

**对偶互补松弛**：对∀i：y_i = 0 或 Σⱼ a_{ij}x_j = b_i

#### 松弛互补松弛条件（命题1）

若x, y分别是原、对偶问题的可行解，满足：

- **α-松弛原始条件**：x_j=0 或 c_j/α ≤ Σᵢ a_{ij}y_i ≤ c_j
- **β-松弛对偶条件**：y_i=0 或 b_i ≤ Σⱼ a_{ij}x_j ≤ β·b_i

则：Σ c_jx_j ≤ α·β·Σ b_iy_i

------

### 8.6.2 整数规划与LP松弛

#### 组合优化问题 → 整数线性规划

**例：最小顶点覆盖**

整数规划（IP）：
$$\min \sum_{v\in V} x_v$$
$$\text{s.t.} \quad x_u + x_v \geq 1, \quad \forall (u,v)\in E$$
$$x_v \in {0, 1}$$

**LP松弛**：将 x_v ∈ {0,1} 放宽为 x_v ∈ [0,1]

#### 从LP解到近似解的两种方法

| 方法                   | 特点                                    |
| ---------------------- | --------------------------------------- |
| **舍入法**             | 精确求解LP，将分数解舍入为整数解        |
| **Primal-Dual Schema** | 不需要精确求解LP，构造原始-对偶可行解对 |

------

### 8.6.3 集合覆盖的LP松弛与Primal-Dual算法

#### 集合覆盖问题的LP表示

**原始问题（LP松弛）**：
$$\min \sum_{S\in F} c(S) \cdot x_S$$
$$\text{s.t.} \quad \sum_{S: e\in S} x_S \geq 1, \quad \forall e\in X$$
$$x_S \in [0,1]$$

**对偶问题**：
$$\max \sum_{e\in X} y_e$$
$$\text{s.t.} \quad \sum_{e\in S} y_e \leq c(S), \quad \forall S\in F$$
$$y_e \geq 0$$

#### 关系

- OPT_f（分数最优解代价）≤ OPT（整数最优解代价）
- 对偶问题任意可行解的代价是 OPT 的下界

#### Primal-Dual Schema 集合覆盖算法

```
Primal-Dual Set Cover:
1. x ← 0    // 原始问题初始解
2. y ← 0    // 对偶问题初始可行解
3. U ← ∅    // 已覆盖元素集合
4. While U ≠ X Do
5.    取 e_i ∈ X - U
6.    增加 y_{e_i} 直到 ∃S∈F 使得 Σ_{e∈S} y_e = c(S)（S变紧）
7.    对所有变紧的集合S：x_S=1，U=U∪S
8. 输出所有 x_S=1 的集合
```

#### 性能分析

**引理1**：算法结束后，x和y分别是原始问题和对偶问题的可行解。

**引理2**：算法结束时 x 和 y 满足：

1. 对∀S∈F：x_S≠0 ⟹ Σ_{e∈S} y_e = c(S)
2. 对∀e∈X：y_e≠0 ⟹ Σ_{S:e∈S} x_S ≤ f（f为元素的最大频率）

**定理**：Primal-Dual集合覆盖算法的近似比为 **f**（X中元素的最大频率）。

**证明**：

由引理2，满足命题1的条件（α=1, β=f），因此：
$$\text{cost}(C) = \sum_{S:x_S=1} c(S) \leq 1\cdot f \cdot \sum_{e\in X} y_e$$

由于y是对偶可行解：Σ_{e∈X} y_e ≤ OPT

因此：**cost(C) ≤ f · OPT**

### 8.6.4 最大流-最小割（Max-Flow Min-Cut）

#### 最大流问题

- **输入**：有向图G=(V,E)，源s，汇t，容量c(e)
- **输出**：从s到t的最大流
- **约束**：流量约束 f(e)≤c(e)，守恒约束（除s,t外各点流入=流出）

#### 最小割问题

- **输入**：同上
- **输出**：s-t最小割（删除后s到t不可达的边集，使容量和最小）

#### 最大流最小割定理

$$\text{最大流} = \text{最小割}$$

这是LP对偶理论的一个经典应用（强对偶定理）。

## 关键概念辨析

### PTAS vs FPTAS

- **PTAS**（多项式时间近似模式）：对任意固定ε>0，算法在输入规模n的多项式时间内运行
- **FPTAS**（完全多项式时间近似模式）：算法运行时间是n和1/ε的多项式

### 近似算法设计范式

1. **组合优化**：利用问题的组合结构（如最小生成树→TSP）
2. **贪心策略**：每步选择局部最优，分析全局性能
3. **局部搜索**：从初始解出发，不断局部改进直到无法改进
4. **动态规划+Rounding**：缩小状态空间，损失少量精度
5. **线性规划**：LP松弛+舍入或Primal-Dual Schema
