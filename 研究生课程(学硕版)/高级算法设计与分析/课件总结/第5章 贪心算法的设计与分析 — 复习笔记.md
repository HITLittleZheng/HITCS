# 第五章 贪心算法的设计与分析 — 复习笔记

------

## 一、贪心算法的基本要素

### 1.1 基本思想

- 求解**最优化问题**，算法包含一系列步骤
- **每一步**都作出在当前看来**最好**的选择（局部最优）
- 希望通过局部最优选择达到**全局最优解**
- ⚠️ 贪心算法**不一定**总能产生最优解

### 1.2 贪心算法产生最优解的两个条件

| 条件                                     | 含义                                             |
| ---------------------------------------- | ------------------------------------------------ |
| **最优子结构（优化子结构）**             | 一个优化问题的优化解包含其（剩余）子问题的优化解 |
| **贪心选择性（Greedy-choice property）** | 全局最优解可以通过局部最优选择得到               |

### 1.3 贪心选择性的证明方法

1. **归纳法**：对算法步数或问题规模归纳
2. **交换论证法**：在保证最优性不变的前提下，从一个最优解逐步替换，最终得到贪心算法的解

### 1.4 与动态规划的比较

| 比较项       | 动态规划                           | 贪心算法                               |
| ------------ | ---------------------------------- | -------------------------------------- |
| **选择依赖** | 每步选择依赖于子问题的解           | 每步先做当前最好选择，不依赖子问题的解 |
| **求解方向** | 自底向上，先解小子问题再解大子问题 | 自顶向下，逐步贪心选择，缩减子问题规模 |
| **适用条件** | 最优子结构 + 子问题重叠性          | 最优子结构 + 贪心选择性                |
| **关系**     | 可用DP时，贪心不一定适用；反之亦然 | —                                      |

**经典对比实例：0-1背包 vs 部分背包**

- 两者都具有最优子结构
- **部分背包**：可用贪心（按每磅价值 $v_i/w_i$ 降序取）→ 正确
- **0-1背包**：贪心**不正确**，需用动态规划

### 1.5 贪心算法的设计步骤（非常重要）

1. **设计贪心选择方法**：确定贪心选择策略 + 剩余子问题结构
2. **证明最优子结构**
3. **证明贪心选择性**（决定能否得到全局最优解）
4. **按贪心策略设计算法**

## 二、活动选择问题（Activity-Selection Problem）

### 2.1 问题定义

- **输入**：$S = {1, 2, \dots, n}$ 是 $n$ 个活动的集合，每个活动 $i$ 有起始时间 $s_i$ 和终止时间 $f_i$（$s_i < f_i$），所有活动共享一个资源（同时只能为一个活动使用）
- **相容活动**：活动 $i$ 和 $j$ 相容，当且仅当 $s_j \geq f_i$ 或 $s_i \geq f_j$（时间区间不重叠）
- **输出**：$S$ 的**最大相容活动集合**（包含活动数最多）

### 2.2 贪心选择方法

- **贪心策略**：每次选择**结束时间 $f_i$ 最小**的活动（使我们能选更多后续活动）
- **剩余子问题**：$S_i = {j \in S \mid s_j \geq f_i}$

### 2.3 最优子结构的证明

**定理1**：设 $S = {1,2,\dots,n}$，$f_1 \leq f_2 \leq \dots \leq f_n$，设 $A$ 是 $S$ 的活动选择问题的一个优化解且包括活动 1，则 $A' = A - {1}$ 是 $S' = {i \in S \mid s_i \geq f_1}$ 的活动选择问题的优化解。

**证明**（反证法）：

- $A'$ 中活动显然相容
- 若 $A'$ 不是 $S'$ 的最优解，则存在 $S'$ 的最优解 $B'$，$|B'| > |A'|$
- 令 $B = {1} \cup B'$，对 $i \in S'$，$s_i \geq f_1$，所以 $B$ 中活动相容
- $|B| = |B'| + 1 > |A'| + 1 = |A|$，与 $A$ 最大矛盾 ∎

### 2.4 贪心选择性的证明

**引理1**：设 $f_1 \leq f_2 \leq \dots \leq f_n$，则 $S$ 的活动选择问题的**某个**优化解包括活动 1。

**证明**（交换论证法）：

- 设 $A$ 是一个优化解，其第一个活动为 $k$，第二个活动为 $j$
- 若 $k = 1$，结论成立
- 若 $k \neq 1$，令 $B = A - {k} \cup {1}$
- 由于 $f_1 \leq f_k \leq s_j$，$B$ 中活动相容
- $|B| = |A|$，所以 $B$ 也是优化解，且包含活动 1 ∎

**定理2**（完整贪心选择性）：设 $l_i$ 是 $S_i$ 中具有最小结束时间的活动，则 $A = {l_1, l_2, \dots, l_k}$（逐步选最小结束时间活动）。通过对 $|A|$ 归纳证明。

### 2.5 算法

```
Greedy-Activity-Selector(S, F)
    n ← length(S)
    A ← {1}         // 选择第一个活动（结束最早）
    j ← 1
    For i ← 2 To n Do
        If s_i ≥ f_j
        Then A ← A ∪ {i}; j ← i
    Return A
```

**前提**：活动已按结束时间 $f_1 \leq f_2 \leq \dots \leq f_n$ 排序。

### 2.6 时间复杂度

| 情况             | 时间复杂度                                                  |
| ---------------- | ----------------------------------------------------------- |
| 已按结束时间排序 | $\Theta(n)$                                                 |
| 未排序           | $\Theta(n\log n)$（排序） + $\Theta(n)$ = $\Theta(n\log n)$ |

### 2.7 正确性

由定理1（最优子结构）+ 定理2（贪心选择性）+ 算法按贪心策略选择 → 算法产生最优解。

## 三、Huffman 编码

### 3.1 问题定义

**基本概念**：

- **二进制字符编码**：每个字符用 0/1 串表示
- **固定长编码**：每个字符用相同长度 0/1 串
- **可变长编码**：频繁字符用短码，不频繁用长码
- **前缀编码**：无任何字符的编码是另一个字符编码的前缀

**编码树**：

- **叶结点**：字符及其频数
- **内结点**：子树叶结点频数之和
- **边标记**：左 0，右 1

**编码树代价**：

$$B(T) = \sum_{c \in C} f(c) \cdot d_T(c)$$

其中 $f(c)$ 是字符 $c$ 的频数，$d_T(c)$ 是叶子 $c$ 在树 $T$ 中的深度（编码长度）。

**问题**：

- **输入**：字母表 $C = {c_1, c_2, \dots, c_n}$，频数表 $F = {f(c_1), \dots, f(c_n)}$
- **输出**：具有最小 $B(T)$ 的前缀编码树

### 3.2 贪心选择方法

- **贪心策略**：每次选择**频数最低的两个结点** $x$ 和 $y$，构造子树
- **剩余子问题**：$C' = C - {x, y} \cup {z}$，其中 $f(z) = f(x) + f(y)$

```
         z
        / \
       x   y
```

### 3.3 最优子结构的证明

**引理1**：设 $T$ 是 $C$ 的最优前缀树，$x$、$y$ 是 $T$ 中任意两个相邻叶结点，$z$ 是它们的父结点，则 $T' = T - {x,y}$ 是 $C' = C - {x,y} \cup {z}$（$f(z) = f(x)+f(y)$）的最优前缀编码树。

**证明关键**：

- 建立代价关系：$B(T) = B(T') + f(x) + f(y)$
  - 因为 $d_T(x) = d_T(y) = d_{T'}(z) + 1$
- 反证：若 $T'$ 不是 $C'$ 的最优编码树，存在 $T''$ 使 $B(T'') < B(T')$
- 将 $x$、$y$ 加入 $T''$ 作为 $z$ 的子结点得 $T'''$
- $B(T''') = B(T'') + f(x) + f(y) < B(T') + f(x) + f(y) = B(T)$
- 与 $T$ 最优矛盾 ∎

### 3.4 贪心选择性的证明

**引理2**：设 $x$、$y$ 是 $C$ 中**频数最小**的两个字符，则存在 $C$ 的一个最优前缀树，使 $x$ 与 $y$ 的编码具有**相同最大长度**，且仅末位不同（即为兄弟叶结点）。

**证明**（交换论证法）：

- 设 $T$ 是最优前缀树，$b$、$c$ 是 $T$ 中最大深度的两个兄弟
- 不失一般性 $f(b) \geq f(c)$，$f(x) \leq f(y)$
- 因为 $x$、$y$ 是最低频数：$f(x) \leq f(y) \leq f(b) \leq f(c)$
- 交换 $b$ 和 $x$ 得 $T'$：
  $$B(T) - B(T') = (f(b)-f(x))(d_T(b)-d_T(x)) \geq 0$$
- 交换 $c$ 和 $y$ 得 $T''$：同理 $B(T') - B(T'') \geq 0$
- 所以 $B(T) \geq B(T'')$，又 $T$ 最优 → $B(T) = B(T'')$
- $T''$ 也是最优前缀树，且 $x$、$y$ 是深度最大的兄弟 ∎

### 3.5 算法

```
Huffman(C, F)
1.  n ← |C|; Q ← 根据F对C建最小堆; T为空树
2.  For i ← 1 To n-1 Do
3.      z ← Allocate-Node()
4.      left[z] ← x ← Extract-min(Q)    // 取出最小频数结点
5.      right[z] ← y ← Extract-min(Q)   // 取出次小频数结点
6.      f(z) ← f(x) + f(y)
7.      Insert(Q, z, f(z))
8.  Return Extract-min(Q)               // 返回树根
```

### 3.6 构造过程示例

字母频数：f:5, e:9, c:12, b:13, d:16, a:45

| 步骤 | 操作                     | 说明     |
| ---- | ------------------------ | -------- |
| 1    | 合并 f(5) 和 e(9) → 14   | 最小两个 |
| 2    | 合并 c(12) 和 b(13) → 25 | 最小两个 |
| 3    | 合并 14 和 d(16) → 30    | 最小两个 |
| 4    | 合并 25 和 30 → 55       | 最小两个 |
| 5    | 合并 a(45) 和 55 → 100   | 最终树根 |

### 3.7 时间复杂度

| 步骤                                                     | 复杂度            |
| -------------------------------------------------------- | ----------------- |
| 建堆                                                     | $O(n)$            |
| 循环 $n-1$ 次，每次 Extract-min 和 Insert 各 $O(\log n)$ | $O(n \log n)$     |
| **总计**                                                 | **$O(n \log n)$** |

## 四、最小生成树问题

### 4.1 问题定义

- **生成树**：设 $G = (V, E)$ 是边加权无向连通图，$G$ 的生成树是 $T = (V, E_T)$，$E_T \subseteq E$，$T$ 是无向树
- **生成树的权**：$W(T) = \sum_{(u,v) \in T} W(u,v)$
- **最小生成树**：$W(T)$ 最小的生成树

**问题**：

- **输入**：无向连通图 $G = (V,E)$，权函数 $W$
- **输出**：$G$ 的最小生成树

### 4.2 基本概念

| 概念       | 定义                                                         |
| ---------- | ------------------------------------------------------------ |
| **划分**   | $V$ 的划分 $(S, V-S)$                                        |
| **交叉边** | 若 $u \in S$，$v \in V-S$，则 $(u,v)$ 是划分的交叉边         |
| **尊重A**  | 若边集 $A$ 中没有边是划分的交叉边，则该划分尊重 $A$          |
| **轻边**   | 划分的交叉边中权值最小的边                                   |
| **安全边** | 若 $A \cup {(u,v)}$ 仍是某最小生成树的边子集，则 $(u,v)$ 对 $A$ 安全 |

### 4.3 贪心选择方法

- **初始**：$A = \emptyset$，构造森林 $G_A = (V, A)$
- **循环**：选择连接 $G_A$ 中两棵树的**最小安全边**加入 $A$，直至 $G_A$ 是一棵生成树

**一般算法框架**：

```
Generic-MST(G, W)
1. A ← ∅
2. While A 不是生成树 Do
3.     寻找一个最小安全边 (u,v)
4.     A ← A ∪ {(u,v)}
5. Return A
```

### 4.4 最优子结构

**定理1**：设 $T$ 是 $G$ 的最小生成树，$T$ 包含子树 $T_1$（$G_1$ 的生成树）和 $T_2$（$G_2$ 的生成树），则 $T_1$ 是 $G_1$ 的最小生成树，$T_2$ 是 $G_2$ 的最小生成树。

### 4.5 贪心选择性（安全边定理）

**定理2**（核心定理）：设 $G = (V,E)$ 是边加权无向连通图，$A \subseteq E$ 包含在 $G$ 的某最小生成树中，$(S, V-S)$ 是 $G$ 的尊重 $A$ 的任意划分，$(u,v)$ 是 $(S,V-S)$ 的交叉**轻边**，则 $(u,v)$ 对 $A$ 是**安全的**。

**证明**：

- 令 $T$ 是包含 $A$ 的最小生成树
- 若 $(u,v) \in T$，则安全，证毕
- 若 $(u,v) \notin T$：
  - $T$ 中 $u$ 到 $v$ 的路径 $p$ 必有交叉边 $(x,y)$
  - 因划分尊重 $A$，$(x,y) \notin A$
  - 构造 $T' = T - {(x,y)} \cup {(u,v)}$
  - $W(u,v) \leq W(x,y)$（因 $(u,v)$ 是轻边）
  - $W(T') = W(T) - W(x,y) + W(u,v) \leq W(T)$
  - $T$ 是最小生成树 → $W(T') = W(T)$，$T'$ 也是最小生成树
  - $A \cup {(u,v)} \subseteq T'$，所以 $(u,v)$ 安全 ∎

**推论1**：设 $C = (V_C, E_C)$ 是森林 $G_A = (V, A)$ 中的一棵树，$(u,v)$ 是连接 $C$ 和另一棵树的交叉**轻边**，则 $(u,v)$ 对 $A$ 安全。

### 4.6 Kruskal 算法

**基本思想**：按边权从小到大排序，依次考察每条边，若它连接 $G_A$ 中两棵不同的树，则加入 $A$。

```
MST-Kruskal(G, W)
1.  A ← ∅
2.  For v ∈ V[G] Do
3.      Make-Set(v)          // 每个顶点建立单元素集合
4.  按 W 值递增顺序排序 E[G]
5.  For (u,v) ∈ E[G]（按W递增顺序）Do
6.      If Find-Set(u) ≠ Find-Set(v)    // u、v在不同的树中
7.      Then A ← A ∪ {(u,v)}; Union(u,v)
8.  Return A
```

### 4.7 时间复杂度

设 $n = |V|$，$m = |E|$：

| 步骤                              | 复杂度                                                 |
| --------------------------------- | ------------------------------------------------------ |
| 第4步：排序边                     | $O(m \log m)$                                          |
| 第2-3步：$n$ 次 Make-Set          | $O(n)$                                                 |
| 第5-7步：$m$ 次 Find-Set 和 Union | $O(m \cdot \alpha(n))$（$\alpha$ 是反 Ackermann 函数） |
| **总计**                          | **$O(m \log m)$**                                      |

> 注：因 $m \leq n^2$，$\log m \leq 2\log n$，所以也可写为 $O(m \log n)$。

### 4.8 正确性

算法按贪心选择性进行局部优化选择，每次选择的边都是连接两棵不同子树的最小边（即安全边），由定理2保证正确性。

## 五、贪心算法的理论基础——拟阵（Matroid）

### 5.1 拟阵的定义

**Matroid** 是序对 $M = (S, \mathcal{I})$，满足：

1. $S$ 是有限非空集合
2. $\mathcal{I} \subseteq 2^S$，$\mathcal{I}$ 非空（$\mathcal{I}$ 中的子集称为 $S$ 的**独立子集**）
3. **遗传性**：若 $A \in \mathcal{I}$，$B \subseteq A$，则 $B \in \mathcal{I}$
4. **交换性**：若 $A \in \mathcal{I}$，$B \in \mathcal{I}$，$|A| < |B|$，则 $\exists x \in B - A$ 使得 $A \cup {x} \in \mathcal{I}$

> 遗传性和交换性是拟阵最根本的两条性质。

### 5.2 图拟阵实例（Graphic Matroid）

**定义**：设 $G = (V, E)$ 是无向连通图，$|E| \geq 1$，定义：

- $S_G = E$
- $\mathcal{I}_G = {A \mid A \subseteq E,\ G_A = (V, A) \text{ 是森林}}$

**定理**：$M_G = (S_G, \mathcal{I}_G)$ 是一个拟阵。

**证明要点**：

- 非空性、有限性：显然
- 遗传性：森林的子图仍是森林
- 交换性：利用"具有 $k$ 条边的森林有 $|V| - k$ 棵树"的图论定理
  - $|A| < |B|$ → $G_A$ 比 $G_B$ 树更多 → $G_B$ 中存在边 $(u,v)$ 连接 $G_A$ 中不同的树 → $A \cup {(u,v)}$ 仍是森林

### 5.3 拟阵的性质

| 概念                  | 定义                                                         |
| --------------------- | ------------------------------------------------------------ |
| **扩展（extension）** | 若 $A \cup {x} \in \mathcal{I}$，$x \notin A$，则 $x$ 是 $A$ 的扩展 |
| **最大独立子集**      | 没有扩展的独立子集                                           |

**定理**：一个拟阵的**所有最大独立子集具有相同大小**。

- 证明：若存在 $|A| < |B|$，由交换性可扩展 $A$，与 $A$ 最大矛盾。

### 5.4 加权拟阵与最优独立子集

- **加权拟阵**：拟阵 $M = (S, \mathcal{I})$ 配合权函数 $W: S \to \mathbb{R}^+$
- **集合权值**：$W(A) = \sum_{x \in A} W(x)$
- **最优独立子集**：$W(A)$ 最大的独立子集 $A \in \mathcal{I}$

**实际意义**：很多可用贪心算法获得最优解的问题都可以归结为在加权拟阵中寻找最优独立子集。

### 5.5 贪心选择性

**引理1**：若 ${x} \notin \mathcal{I}$，则 $x$ 不是任何独立子集的元素。（由遗传性）

**推论**：任何元素一旦不能被初始选中，则永远不会被选中。

**定理1（贪心选择性）**：设 $S$ 按 $W$ 值递减排序，$x$ 是 $S$ 中第一个满足 ${x} \in \mathcal{I}$ 的元素，则存在最优独立子集 $A$ 使 $x \in A$。

**证明**：

- 设 $B$ 是任意最优独立子集，$x \notin B$
- $S$ 中 $x$ 之前的元素 $y$ 必不在 $B$ 中（否则 ${y} \in \mathcal{I}$ 矛盾）
- 对所有 $z \in B$，$W(x) \geq W(z)$
- 用交换性从 $A = {x}$ 开始，逐步加入 $B$ 中元素，得到 $A = (B - {w}) \cup {x}$
- $W(A) = W(B) - W(w) + W(x) \geq W(B)$，所以 $A$ 也是最优独立子集 ∎

### 5.6 最优子结构

**定理2**：设 $x$ 是第一个被贪心选中的元素，$A$ 是包含 $x$ 的最优独立子集，则 $A' = A - {x}$ 是子问题 $M' = (S', \mathcal{I}')$ 的最优独立子集。其中：

- $S' = S - {y \mid W(y) > W(x)} - {x}$
- $\mathcal{I}' = {B \subseteq S' \mid B \cup {x} \in \mathcal{I}}$

**证明**：反证，若存在更优的 $B$，则 $B \cup {x}$ 比 $A$ 更优，矛盾。

### 5.7 算法

```
Greedy(M, W)
1.  A ← ∅
2.  按权 W 值递减排序 S
3.  For x ∈ S（按 W(x) 递减顺序）Do
4.      If A ∪ {x} ∈ I
5.      Then A ← A ∪ {x}
6.  Return A
```

### 5.8 时间复杂度

$$T(|S|) = O(|S| \log |S| + |S| \cdot f(|S|))$$

其中 $f(|S|)$ 是检查 $A \cup {x} \in \mathcal{I}$ 的时间。

### 5.9 正确性

**定理3**：$\text{Greedy}(M, W)$ 返回 $M$ 的最优独立子集。

证明：引理1（不可选元素排除）+ 定理1（贪心选择性）+ 定理2（最优子结构）。

## 六、任务调度问题

### 6.1 问题定义

- **输入**：
  - 单位时间任务集 $S = {1, 2, \dots, n}$
  - 正整数任务期限 $D = {d_1, d_2, \dots, d_n}$（任务 $i$ 须在 $d_i$ 前完成）
  - 非负权集 $W = {w_1, w_2, \dots, w_n}$（任务 $i$ 不在 $d_i$ 前完成罚款 $w_i$）
- **输出**：$S$ 的一个调度（执行顺序），具有**最小总罚款数**

### 6.2 核心概念

| 概念               | 定义                                           |
| ------------------ | ---------------------------------------------- |
| **早任务**         | 在调度中，任务 $i$ 在 $d_i$ 之前完成           |
| **迟任务**         | 在调度中，任务 $i$ 在 $d_i$ 之后完成           |
| **早任务优先形式** | 早任务总先于迟任务执行                         |
| **规范化形式**     | 早任务优先且按期限单调递增顺序执行各早任务     |
| **独立集**         | 任务集 $A$ 存在一个调度使 $A$ 中任务皆为早任务 |

### 6.3 调度的规范化

1. **早任务优先**：若早任务 $x$ 在迟任务 $y$ 之后，交换它们（不改变早/迟状态）
2. **期限递增排序早任务**：若早任务 $i$、$j$ 完成于时刻 $k$、$k+1$ 且 $d_j < d_i$，交换它们（不改变早/迟状态）

→ 寻找最优调度 ⟺ 寻找**最优早任务集合** $A$

### 6.4 独立集的判定

**$N_t(A)$** = $A$ 中期限 $\leq t$ 的任务数。

**引理1**：以下三条等价：

1. $A$ 是独立集
2. 对 $t = 1, 2, \dots, n$，$N_t(A) \leq t$
3. 按期限递增顺序调度 $A$ 中任务，$A$ 中无迟任务

### 6.5 转换为拟阵问题

**定理1**：设 $S$ 是带期限的单位时间任务集合，$\mathcal{I}$ 为所有独立任务集构成的集族，则 $M = (S, \mathcal{I})$ 是一个**拟阵**。

**证明要点**：

- 非空性、有限性：显然
- 遗传性：若 $A \in \mathcal{I}$，$B \subseteq A$，则 $N_t(B) \leq N_t(A) \leq t$，$B \in \mathcal{I}$
- 交换性：设 $A, B \in \mathcal{I}$，$|B| > |A|$
  - 令 $k = \max{t \mid N_t(B) > N_t(A)}$
  - $B$ 中比 $A$ 多出具有期限 $k+1$ 的任务
  - 选 $x \in B - A$（期限为 $k+1$），令 $A' = A \cup {x}$
  - 对 $1 \leq t \leq k$：$N_t(A') = N_t(A) \leq t$
  - 对 $k < t \leq n$：$N_t(A') \leq N_t(B) \leq t$
  - 所以 $A' \in \mathcal{I}$

### 6.6 权值定义

- 令 $\bar{w} = \max{w_1, w_2, \dots, w_n}$
- 对 $x_i \in S$：$W(x_i) = w_i$（即以罚款作权值，不是直接这么用）

**转换逻辑**：

- 对独立集 $A$：$W(A) = n\bar{w} - \sum_{y \in S-A} w(y)$
- 最大化 $W(A)$ ⟺ 最小化 $\sum_{y \in S-A} w(y)$（即最小化总罚款）

→ 用拟阵上的 Greedy 算法：按权值（$w_i$）递减排序，依次尝试加入独立集。

### 6.7 算法流程

1. 按 $w_i$ 递减顺序排列任务
2. 依次考察每个任务，若加入当前集合 $A$ 后仍为独立集（$N_t(A) \leq t$ 对所有 $t$ 成立），则加入
3. $A$ 即为最优早任务集合
4. 按期限递增顺序调度 $A$ 中任务，再按任意顺序调度迟任务 $S - A$
