### 🟦 问题 1：顶点覆盖（Vertex Cover）近似比 2

#### 📖 1. 问题到底在干嘛？
- **输入**：一张无向图 $G=(V, E)$（一堆点，一堆边）。
- **目标**：挑出**最少数量**的点组成集合 $C$，使得**每条边**至少有一个端点在 $C$ 里。
- **难点**：这是 NP 完全问题。图一大，穷举会爆炸，所以需要近似算法。

#### 🤖 2. 算法具体怎么跑？（APPROX-Vertex-Cover）

```

1. C ← ∅ （结果集先空着）
2. E' ← E （把边复制一份用来操作）
3. While E' 不为空:
   随便挑一条边 (u, v) ∈ E'
   把 u 和 v 都扔进 C
   从 E' 中删掉所有跟 u 或 v 相连的边（因为这条边和它的邻居边已经被 u,v 盖住了）
4. 返回 C
```

**一句话动作**：看到一条边，直接买下两端，然后清空周边，继续找下一条。

#### 💡 3. 直觉：为什么它不会太离谱？
想象你在修路灯覆盖道路。每条路必须至少有一端有灯。
- **算法策略**：看到一条没灯的路，直接两端都装灯。很“土豪”，但绝对能覆盖。
- **最优解策略**：再精明的人，要照亮同一条路，**最少也得装 1 盏灯**。
- 所以算法每照亮一条路，花的钱最多是最优解的 2 倍。只要这些路**不共享端点**，这个 2 倍关系就能锁死全局。

#### 📐 4. 近似比证明（逐行拆解，对应你的笔记）
我们要证：$|C| / |C*| ≤ 2$（$C$ 是算法解，$C*$ 是最优解）

**第①步：定义中间变量 $A$**
- 让 $A$ 表示算法在第 4 步**实际选中过的那批边**的集合。
- ⚠️ 关键性质：$A$ 中的边**两两不相邻**（没有公共端点）。
  - *为什么？* 因为算法每次挑中 $(u,v)$ 后，第 6 步会**删光**所有跟 $u$ 或 $v$ 相连的边。所以后面绝不可能再挑到和 $(u,v)$ 共享端点的边。
  - 这在图论里叫**极大匹配（Maximal Matching）**。

**第②步：算算法花了多少钱（$|C|$）**
- 算法每从 $A$ 里拿一条边，就往 $C$ 里塞 2 个点。
- 所以：$|C| = 2 × |A|$

**第③步：卡住最优解的下限（$|C*|$）**

- 最优解 $C*$ 必须覆盖图中**所有边**，当然也必须覆盖 $A$ 里的所有边。
- $A$ 里的边互不相邻，意味着它们**没有公共端点**。
- 要覆盖 $|A|$ 条互不相邻的边，哪怕最优解再会“一石二鸟”，在这里也做不到（因为边根本不挨着）。所以覆盖 $A$ **至少需要 $|A|$ 个点**。
- 结论：$|C*| ≥ |A|$

**第④步：联立收网**
$
|C| = 2|A|          （算法实际选的点数）
    ≤ 2|C*|         （因为 |A| ≤ |C*|）
→ |C| / |C*| ≤ 2
$
证毕。近似比严格 ≤ 2。

#### 🔍 5. 最容易卡壳的 3 个坑
| 坑点                                                     | 为什么错                                           | 正确理解                                                     |
| :------------------------------------------------------- | :------------------------------------------------- | :----------------------------------------------------------- |
| “最优解一个点能盖多条边，所以 $|C*|$ 可能远小于 $|A|$？” | 混淆了“原图”和“集合A”。                            | 我们只用 $A$ 来卡下限。$A$ 是**互不相交**的边集，最优解在这 $|A|$ 条边上就是无法一石二鸟，最少必须出 $|A|$ 个点。 |
| “为什么不是最大匹配？”                                   | 算法是**贪心**随便挑，不一定是全局边数最多的匹配。 | 我们只需要**极大匹配**（不能继续加边了）就够了。证明不依赖“最大”，只依赖“互不相邻”。 |
| “近似比一定是 2 吗？”                                    | 2 是**上界**，不是定值。                           | 比如星型图，算法选中心+1个叶子(2点)，最优解只选中心(1点)，此时比值=2。很多图比值<2，但最坏情况卡死在2。 |

#### ✅ 总结记忆锚点
> **“挑边不碰头，一边走两点。最优最少一，倍数锁在二。”**

### 🟦 问题 2：装箱问题（Bin Packing）- First Fit → 近似比 2

#### 📖 1. 问题到底在干嘛？
- **输入**：$n$ 个物品，每个物品体积 $s_i ∈ (0, 1]$，箱子容量固定为 $1$。
- **目标**：用最少的箱子把所有物品装下。
- **难点**：物品顺序固定或任意，装箱是经典的 NP 难问题。哪怕物品体积看起来很整，组合起来也可能产生大量“边角料”浪费。

#### 🤖 2. 算法具体怎么跑？（First Fit）
```


1. 初始化：所有箱子为空，当前只有第1个箱子
2. For i = 1 to n:  // 按给定顺序处理每个物品
3. 从第1个箱子开始往后找，找到第一个剩余空间 ≥ s_i 的箱子
4. 把物品 i 放进去
5. If 所有已有箱子都装不下:
6. 开启一个新箱子，把物品 i 放进去
7. Return 使用的箱子总数 B

```

**一句话动作**：来了新物品，从头到尾找“第一个能塞下的坑”，塞不下就开新坑。

#### 💡 3. 直觉：为什么它不会太离谱？
First Fit 看起来很“呆板”，但它有一个隐藏特性：**它极度厌恶“大量半空箱”**。
- 假设你用了 10 个箱子，如果前面 5 个都只装了一半（0.5），那第 6 个箱子里的物品当初来的时候，明明可以塞进前 5 个里的任意一个！但算法说“没塞”，说明前 5 个箱子的剩余空间肯定都 $<$ 第 6 个物品的体积。
- 这意味着：**除了最后一个箱子，不可能出现两个箱子都“半空或更空”的情况**。否则后开的箱子里的物品，早就被塞进前一个半空箱了。
- 既然“半空箱”最多只有 1 个，那其他箱子一定都装了 $> 0.5$。总体积就被这个 $>0.5$ 卡死了，箱子数自然超不过最优解的 2 倍。

#### 📐 4. 近似比证明（逐行拆解）
我们要证：$B / OPT ≤ 2$（$B$ 是算法用的箱子数，$OPT$ 是最优箱子数）

**第①步：核心引理（反证法抓漏洞）**
- **断言**：在 First Fit 的最终装箱结果中，**至多只有一个箱子的已装体积 $≤ 1/2$**。
- **证明**：假设有两个箱子 $B_i$ 和 $B_j$（$i < j$），它们的已装体积都 $≤ 1/2$。
  - 考虑第一个被放进 $B_j$ 的物品 $x$。它的体积 $s_x ≤ 1/2$（否则它单独就超容量了，不可能被装下）。
  - 当算法处理 $x$ 时，$B_i$ 已经存在且剩余空间 $≥ 1 - 1/2 = 1/2$。
  - First Fit 的规则是“找第一个能装下的箱子”。既然 $B_i$ 能装下 $x$，算法绝不可能跳过 $B_i$ 去开 $B_j$。
  - **矛盾！** 所以假设不成立。至多只有一个箱子体积 $≤ 1/2$。

**第②步：算总体积下限**
- 设算法一共用了 $B$ 个箱子。
- 由第①步，至少有 $B-1$ 个箱子的体积 $> 1/2$。
- 所有物品的总体积记为 $V$。显然：
  $V > (B - 1) × 1/2 = (B - 1)/2$

**第③步：卡住最优解的下限**
- 最优解 $OPT$ 也要装下这 $n$ 个物品，总体积也是 $V$。
- 每个箱子容量最多为 $1$，所以最优解至少需要 $V$ 个箱子（体积守恒）：
  $OPT ≥ V$
- 联立第②步：$OPT > (B - 1)/2$
- 整理得：$2·OPT > B - 1$  →  $B < 2·OPT + 1$

**第④步：整数收尾（最关键的一步）**
- $B$ 和 $OPT$ 都是**正整数**。
- $B < 2·OPT + 1$ 在整数域里等价于 $B ≤ 2·OPT$。
- 所以：$B / OPT ≤ 2$。证毕。

#### 🔍 5. 最容易卡壳的 3 个坑
| 坑点                                                         | 为什么错                       | 正确理解                                                     |
| :----------------------------------------------------------- | :----------------------------- | :----------------------------------------------------------- |
| “总体积 $V$ 可能很小啊，比如一堆 $0.1$ 的物品，$OPT$ 也很小？” | 忽略了算法的**顺序填充特性**。 | 证明不依赖具体体积大小，只依赖“半空箱互斥”这个结构性性质。无论体积多小，只要开了 $B$ 个箱，$B-1$ 个箱的体积和必然撑住下限。 |
| “近似比真的是 2 吗？好像 First Fit 实际表现更好？”           | 2 是**理论最坏情况上界**。     | 学术界已证明 First Fit 的实际渐近上界是 $1.7·OPT + 2$。但考试/笔记要求证 $≤2$，用“半空箱引理+体积放缩”是最标准、最安全的打法。 |
| “为什么最后一步能从 $B < 2OPT+1$ 变成 $B ≤ 2OPT$？”          | 混淆了实数不等式和整数不等式。 | 举个例：若 $OPT=3$，则 $2OPT+1=7$。$B < 7$ 且 $B$ 是整数 → $B$ 最大是 $6$，正好 $B ≤ 6 = 2OPT$。整数特性把 $+1$ 的尾巴吃掉了。 |

#### ✅ 总结记忆锚点
> **“首适不贪新，半空难共存。余箱皆过半，体积定乾坤。最优卡下限，二倍锁死门。”**

### 🟦 问题 3：旅行商问题（TSP with Triangle Inequality）→ 近似比 2

#### 📖 1. 问题到底在干嘛？
- **输入**：完全无向图 $G=(V,E)$，每条边有代价 $C$，且**必须满足三角不等式**：$C(u,w) ≤ C(u,v) + C(v,w)$（任意三点，直接走不比绕路贵）。
- **目标**：找一条经过所有顶点**恰好一次**并回到起点的环路（哈密顿环），使总代价最小。
- **注意**：如果不满足三角不等式，该问题**不存在任何常数近似比的多项式算法**（除非 P=NP）。所以三角不等式是本题证明的“保命符”。

#### 🤖 2. 算法具体怎么跑？（APPROX-TSP-TOUR）

```


1. 任选一个顶点 r 作为根
2. 调用 Prim/Kruskal 算法，构造 G 的最小生成树 T
3. 从 r 开始对 T 做“先序遍历”(Preorder Traversal)，记录访问顺序得到列表 L
   （注：遍历时遇到分叉会回溯，所以 L 中会有重复顶点）
4. 按照 L 的顺序依次访问顶点，如果下一个顶点已经访问过，则直接“跳过”(Shortcut)
   连到 L 中更靠后的未访问顶点，最终回到 r 形成哈密顿环 H
5. Return H
 
```

**一句话动作**：先铺最便宜的骨架(MST) → 沿骨架来回走一遍记录顺序 → 遇到重复点走直线抄近道，拼成合法环路。

#### 💡 3. 直觉：为什么它能工作？
想象你要巡逻所有城市，最优解 $H*$ 是一个完美的环。
- **第一步拆环**：如果我从 $H*$ 上随便砍掉一条边，它立刻变成了一棵**生成树** $T'$。
- **第二步比骨架**：算法找的 MST $T$ 是**所有生成树里最便宜的**，所以 $T$ 的代价一定 $≤ T'$ 的代价，也就 $≤ H*$ 的代价。
- **第三步来回走**：如果我沿着 $T$ 的每条边“走一个来回”（DFS），肯定能经过所有城市，但代价是 $2 × C(T)$。这已经不超过 $2 × 最优解$ 了。
- **第四步抄近道**：来回走会重复访问某些城市。此时我们利用**三角不等式**：把“绕弯路”替换成“直接连直线”。抄近道只会让路程变短或不变，绝不会变长。所以最终合法环路的代价 $≤ 来回走的代价 ≤ 2×最优解$。

#### 📐 4. 近似比证明（逐行拆解，对应笔记 8.2.3）
我们要证：$C(H) / C(H*) ≤ 2$（$H$ 是算法解，$H*$ 是最优解）

**第①步：构造中间生成树 $T'$**
- 从最优哈密顿环 $H*$ 中**任意删除一条边**。
- 环断了一条边后，变成了一条经过所有顶点的链，它本质上是一棵**生成树**，记为 $T'$。
- 显然：$C(T') = C(H*) - 删除边的代价 < C(H*)$，所以 $C(T') ≤ C(H*)$。

**第②步：引入最小生成树 $T$ 放缩**
- 算法第2步构造的是**最小生成树** $T$。
- 根据 MST 定义：任何生成树的代价都 $≥ T$ 的代价。
- 所以：$C(T) ≤ C(T') ≤ C(H*)$  →  $C(T) ≤ C(H*)$
- 📌 **关键点**：这一步把“最优环的代价”成功下放到了“算法构造的 MST 代价”上。

**第③步：定义“完全遍历”路径 $W$**
- 对 MST $T$ 进行一次深度优先的**完全遍历**(Full Walk)，记为 $W$。
- $W$ 的特点是：从根出发，遍历每条边**恰好两次**（一去一回），最后回到根。
- 因此：$C(W) = 2 · C(T)$
- 联立第②步：$C(W) = 2·C(T) ≤ 2·C(H*)$

**第④步：三角不等式“剪枝”(Shortcut)**
- $W$ 不是合法的 TSP 解，因为有些顶点被重复访问了。
- 算法按先序遍历列表 $L$ 的顺序访问顶点。当发现下一个顶点 $v$ 已经在当前路径中出现过时，**直接跳过 $v$**，将当前顶点与 $L$ 中下一个未访问的顶点用直线相连。
- ⚠️ **三角不等式保价**：对于任意三点 $a,b,c$，有 $C(a,c) ≤ C(a,b) + C(b,c)$。这意味着“跳过中间点直接连”的代价，**一定 ≤** “绕道经过中间点”的代价。
- 因此，每执行一次 shortcut，总代价只会下降或持平，绝不会上升。
- 最终得到的哈密顿环 $H$ 满足：$C(H) ≤ C(W)$

**第⑤步：联立收网**
$$$
C(H) ≤ C(W)          （三角不等式剪枝，代价不增）
     = 2·C(T)        （完全遍历每条边走两次）
     ≤ 2·C(H*)       （MST代价 ≤ 最优环代价）
→ C(H) / C(H*) ≤ 2
$$$
证毕。近似比严格 ≤ 2。

#### 🔍 5. 最容易卡壳的 3 个坑
| 坑点                                        | 为什么错                   | 正确理解                                                     |
| :------------------------------------------ | :------------------------- | :----------------------------------------------------------- |
| “三角不等式到底在哪一步起作用？”            | 以为只在最后一步用了一次。 | 它作用于**每一次 shortcut**。算法会跳过很多重复点，三角不等式保证**累计跳过后的总代价 ≤ 原始来回走的代价**。没有它，抄近道可能更贵，证明直接失效。 |
| “为什么 $C(T) ≤ C(H*)$？最优环明明更复杂。” | 混淆了“环”和“树”。         | 环去掉一边就是树。MST 是**全局最便宜的树**，当然比“最优环变来的树”更便宜或相等。这是放缩的起点。 |
| “先序遍历会不会漏掉某些点？”                | 担心 shortcut 把点跳没了。 | 先序遍历本身就会按 DFS 顺序访问**所有**节点。shortcut 只是把“已经访问过的节点”从路径中剔除，**不改变访问集合**，只改变连接方式。最终 $H$ 依然包含所有顶点各一次。 |

#### ✅ 总结记忆锚点
> **“最优去一边，生成树最贱。来回走两遍，代价翻两倍。三角保直线，抄近道不贵。二倍锁死环，证明全到位。”**

### 🟦 问题 4：最小时间调度（Minimum Makespan）→ 近似比 2

#### 📖 1. 问题到底在干嘛？
- **输入**：$m$ 台完全相同的机器，$n$ 个作业，每个作业有已知的处理时间 $p_i$。
- **目标**：把作业分配给机器，使得**最后完成的那台机器的时间（Makespan）最小**。
- **难点**：作业顺序可能任意，一旦某个机器被大作业“堵住”，整体完成时间就会被拉长。这是强 NP 难问题。

#### 🤖 2. 算法具体怎么跑？（Greedy List Scheduling）
```


1. 初始化：所有 m 台机器当前负载 L_1 = L_2 = ... = L_m = 0
2. For i = 1 to n:  // 按给定顺序逐个处理作业
3. 找到当前负载最小的机器 j（若有多个，任选其一）
4. 把作业 i 分配给机器 j
5. 更新 L_j ← L_j + p_i
6. Return max(L_1, ..., L_m)

```

**一句话动作**：来了新活儿，永远丢给当前“最闲”的那台机器。

#### 💡 3. 直觉：为什么它不会太离谱？
想象工厂流水线。贪心策略很公平：谁闲着就给谁。
- 如果最后整体完成时间 $T$ 特别长，一定是某台机器 $M_j$ 堆了太多活。
- 盯着 $M_j$ 上**最后一个被塞进来的作业 $p_k$**。在塞 $p_k$ 之前，$M_j$ 反而是**最闲的**（不然贪心不会选它）。
- 这说明：在塞 $p_k$ 之前，**所有机器的负载都已经不低了**。整体工作量本来就大，加上 $p_k$ 本身也是个硬骨头，所以总时间被“平均负载”和“最大单作业”双重卡住，怎么算都超不过最优解的 2 倍。

#### 📐 4. 近似比证明（逐行拆解，对应笔记 8.3.1）
我们要证：$T / T* ≤ 2$（$T$ 是算法解的完成时间，$T*$ 是最优完成时间）

**第①步：定义关键变量**
- 设算法运行后，负载最高的机器是 $M_j$，它的完成时间为 $T = L_j$。
- 设最后一个被分配到 $M_j$ 的作业是 $p_k$。
- 所有作业的总处理时间记为 $W = Σ p_i$。

**第②步：分析塞入 $p_k$ 前的瞬间状态**
- 在分配 $p_k$ **之前**，$M_j$ 的负载是 $T - p_k$。
- 根据贪心规则，算法之所以选 $M_j$，是因为它此刻**负载最小**。
- 这意味着：其他 $m-1$ 台机器当时的负载都 $≥ T - p_k$。
- 因此，当时的总工作量至少是：$W ≥ m × (T - p_k) + p_k$
  - （$m$ 台机器每台至少 $T-p_k$，再加上刚塞进去的 $p_k$）

**第③步：放缩出 $T$ 的上界**
- 从第②步的不等式整理：
  $m(T - p_k) ≤ W - p_k$
  $T - p_k ≤ (W - p_k) / m ≤ W / m$
- 所以：$T = (T - p_k) + p_k ≤ W/m + p_k$
- 📌 **关键点**：算法的完成时间 $T$，被拆成了两部分：“平均负载” + “最后那个大作业”。

**第④步：引入最优解 $T*$ 的两个下界**
- 下界1（平均原理）：$m$ 台机器平分 $W$ 的工作量，最少也得 $W/m$ 时间。所以 $T* ≥ W/m$。
- 下界2（瓶颈原理）：任何一台机器都必须完整执行分配给它的作业，所以最优解不可能比最大的单个作业还短。即 $T* ≥ max(p_i) ≥ p_k$。

**第⑤步：联立收网**
$$$
T ≤ W/m + p_k       （第③步结论）
  ≤ T* + T*         （代入两个下界：W/m ≤ T*,  p_k ≤ T*）
  = 2T*
→ T / T* ≤ 2
$$$
证毕。近似比严格 ≤ 2。

#### 🔍 5. 最容易卡壳的 3 个坑
| 坑点                                                 | 为什么错                 | 正确理解                                                     |
| :--------------------------------------------------- | :----------------------- | :----------------------------------------------------------- |
| “为什么非要盯着最后一个作业 $p_k$？”                 | 觉得随便挑个作业也能证。 | 只有 $p_k$ 满足**“塞它之前，该机器是最闲的”**这一关键性质。这是把 $T-p_k$ 和全局平均负载挂钩的唯一桥梁。 |
| “$T* ≥ W/m$ 和 $T* ≥ p_k$ 是显然的吗？”              | 容易忽略这两个基础下界。 | 这是调度问题的**铁律**。最优解再神，也逃不过“总工作量除以机器数”和“最大单任务不可拆分”的物理限制。证明全靠它们兜底。 |
| “如果作业按时间从大到小排序（LPT），近似比还是2吗？” | 混淆了基础贪心和改进版。 | 基础贪心（任意顺序）上界是2。如果先排序再贪心（LPT），上界会收紧到 $4/3$。但笔记要求证的是基础版的 $2$，用上述证明即可。 |

#### ✅ 总结记忆锚点
> **“贪心挑最闲，压垮看最后。塞前皆不低，总量被平均。最优卡双界，二倍自然成。”**

### 🟦 问题 5：最小集合覆盖（Minimum Set Cover）→ 近似比 $H(d) ≈ ln|X|+1$

#### 📖 1. 问题到底在干嘛？
- **输入**：全集 $X$，一堆子集构成的族 $F$，保证 $∪_{S∈F} S = X$。
- **目标**：挑出**最少数量**的子集 $C ⊆ F$，使得它们的并集覆盖 $X$。
- **难点**：子集之间严重重叠。贪心如果只挑“大的”，可能留下很多“难啃的零碎元素”；如果挑“小的”，又会浪费集合名额。

#### 🤖 2. 算法具体怎么跑？（Greedy-Set-Cover）
```

1. U ← X          // U 记录还没被覆盖的元素
2. C ← ∅
3. While U ≠ ∅ Do
4. 挑一个 S ∈ F，使得 |S ∩ U| 最大（覆盖最多新元素）
5. U ← U - S   // 把新覆盖的元素从 U 中剔除
6. C ← C ∪ {S}
7. Return C

```

**一句话动作**：永远挑“能盖住最多**未覆盖元素**”的集合，盖完清账，继续挑。

#### 💡 3. 直觉：为什么是调和级数？（会计游戏/发红包模型）
别直接去比“算法选了几个集合 vs 最优解选了几个集合”。证明用了一个绝招：**不数集合，算钱**。
- **规则**：算法每选一个集合，就掏出 $1$ 块钱。这 $1$ 块钱**平分给本次首次被覆盖的元素**。
- 如果一次盖住 5 个新元素，每人拿 $1/5$ 块；如果一次只盖住 1 个，那人独得 $1$ 块。
- 算法总代价 = 选的集合数 $|C|$ = 全场所有元素拿到钱的总和 $Σ c_x$。
- **核心问题**：随便从 $F$ 里抓一个集合 $S$（大小为 $d$），它里面的 $d$ 个元素，总共能拿到多少钱？
- 因为贪心总是挑“能盖住最多新元素”的集合，所以 $S$ 里的元素每次拿钱时，**分母都很大**（人多钱少）。随着未覆盖元素变少，分母从 $d$ 慢慢降到 $1$，累加起来正好是 $1 + 1/2 + ... + 1/d = H(d)$。
- 最优解 $C*$ 用 $|C*|$ 个集合覆盖全场，每个集合最多贡献 $H(d)$ 块钱。全场总钱数自然 $≤ |C*|·H(d)$。证毕。

#### 📐 4. 近似比证明（逐行拆解，对应笔记 8.3.2）
我们要证：$|C| / |C*| ≤ H(d)$（$d = max{|S| : S∈F}$）

**第①步：定义元素代价 $c_x$**
- 设算法第 $i$ 步选了集合 $S_i$，此时新覆盖了 $k_i = |S_i ∩ U_{i-1}|$ 个元素。
- 给这 $k_i$ 个元素每人发 $c_x = 1 / k_i$ 块钱。
- 显然，算法选的集合总数 $|C| = Σ_{x∈X} c_x$。（因为每步 $1$ 块钱全分完了，$k_i × (1/k_i) = 1$）

**第②步：核心引理（证明任意集合 $S$ 的总代价 $≤ H(|S|)$）**
- 随便从 $F$ 里抓一个集合 $S$，设它的大小为 $|S| = d$。
- 跟踪 $S$ 中元素被覆盖的过程。设第 $i-1$ 步后，$S$ 中还有 $u_{i-1}$ 个元素没被覆盖（初始 $u_0 = d$）。
- 第 $i$ 步，贪心选了 $S_i$。根据贪心性质：$S_i$ 新覆盖的元素数 $|S_i ∩ U_{i-1}| ≥ u_{i-1}$（因为 $S$ 本身就能覆盖这 $u_{i-1}$ 个，贪心挑的只会更多或相等）。
- 所以，$S$ 中在第 $i$ 步被覆盖的那些元素，每人拿到的钱 $c_x ≤ 1 / u_{i-1}$。
- 这一步 $S$ 中减少了 $u_{i-1} - u_i$ 个未覆盖元素。它们拿的总钱数 $≤ (u_{i-1} - u_i) / u_{i-1}$。
- **神奇放缩（关键！）**：$(u_{i-1} - u_i) / u_{i-1} ≤ Σ_{j=u_i+1}^{u_{i-1}} (1/j)$
  - *为什么？* 举例：$u_{i-1}=5$（剩5人），$u_i=2$（剩2人），这步盖了3人。平均每人拿 $≤ 1/5$，总和 $≤ 3/5$。
  - 但如果我们按“最后3个被覆盖”的顺序发钱，他们应拿 $1/3 + 1/4 + 1/5 ≈ 0.95$。显然 $3/5 ≤ 1/3+1/4+1/5$。
  - 数学上，把 $k$ 个 $1/m$ 替换成分母从 $m-k+1$ 到 $m$ 的倒数求和，分母变小值变大，不等式恒成立。
- 把所有步骤累加：$Σ_{x∈S} c_x ≤ Σ_{i} Σ_{j=u_i+1}^{u_{i-1}} (1/j) = 1/1 + 1/2 + ... + 1/d = H(d)$。
- 📌 **结论**：任何一个大小为 $d$ 的集合，它里面的元素拿到的总钱数 ≤ $H(d)$。

**第③步：联立最优解 $C*$ 收网**
- 最优解 $C*$ 覆盖了全场 $X$。
- 全场元素总钱数 $Σ_{x∈X} c_x$ 怎么算？按 $C*$ 里的集合分组算：
  $Σ_{x∈X} c_x ≤ Σ_{S∈C*} (Σ_{x∈S} c_x)$ 
  （⚠️注意：元素可能属于 $C*$ 中多个集合，右边会重复计算，所以是 $≤$，这反而帮我们放大了上界，安全！）
- 由第②步，每个 $Σ_{x∈S} c_x ≤ H(|S|) ≤ H(d)$。
- 所以：$Σ_{x∈X} c_x ≤ Σ_{S∈C*} H(d) = |C*| · H(d)$。
- 别忘了 $|C| = Σ_{x∈X} c_x$，代入得：$|C| ≤ |C*| · H(d)$。
- 证毕：$|C| / |C*| ≤ H(d) ≤ ln d + 1 ≤ ln|X| + 1$。

#### 🔍 5. 最容易卡壳的 3 个坑
| 坑点                                               | 为什么错                     | 正确理解                                                     |
| :------------------------------------------------- | :--------------------------- | :----------------------------------------------------------- |
| “为什么 $Σ c_x ≤ Σ_{S∈C*} Σ_{x∈S} c_x$ 是不等号？” | 以为元素只属于一个最优集合。 | 集合覆盖允许重叠。一个元素可能被 $C*$ 中多个集合包含。按 $C*$ 分组求和会把同一个元素的钱算多次，所以是 $≤$（放大），不影响上界证明。 |
| “$(u_{i-1}-u_i)/u_{i-1} ≤ Σ 1/j$ 怎么来的？”       | 觉得是强行凑公式。           | 本质是“平均值 ≤ 调和级数片段”。把 $k$ 个 $1/m$ 替换成 $1/(m-k+1)+...+1/m$，分母递减，值增大，是经典积分放缩技巧。 |
| “近似比是 $ln|X|$ 还是 $H(d)$？”                   | 混淆了两种写法。             | 严格证明是 $H(d)$（$d$ 是最大集合大小）。因为 $d ≤ |X|$ 且 $H(d) ≤ ln d + 1 ≤ ln|X| + 1$，所以笔记里写 $ln|X|+1$ 是宽松上界，考试写哪个都对，但逻辑源头是 $H(d)$。 |

#### ✅ 总结记忆锚点
> **“贪心每选一个，掏出一块钱；平分新元素，人多单价贱。最优集合里，按步算余钱；分母从大到小，调和级数现。全局总和账，不超最优乘 H(d)。”**

### 🟦 问题 6：次模函数最大化（基数约束）→ 近似比 $1/(1-1/e) ≈ 1.58$

#### 📖 1. 问题到底在干嘛？
- **输入**：全集 $U$，一个集合函数 $f: 2^U → ℝ$，约束最多只能选 $k$ 个元素。
- **目标**：找出子集 $A$（$|A| ≤ k$），使 $f(A)$ 最大。
- **核心性质（必须满足）**：$f$ 是**单调次模函数**。
  - **单调**：加元素不会让值变小。$A ⊆ B ⇒ f(A) ≤ f(B)$。
  - **次模（边际收益递减）**：$f(A∪{x}) - f(A) ≥ f(B∪{x}) - f(B)$（当 $A ⊆ B$ 时）。就像往快满的杯子里滴水，越满，新增的一滴水带来的增量越小。
- **难点**：次模函数是非线性的，不能直接用背包或匹配的套路。但“递减”特性反而让贪心有了理论保障。

#### 🤖 2. 算法具体怎么跑？（Greedy Submodular Maximization）
$$$text
1. A ← ∅
2. For i = 1 to k:
3.    x* ← argmax_{x∈U-A} { f(A∪{x}) - f(A) }  // 找边际收益最大的元素
4.    A ← A ∪ {x*}
5. Return A
$$$
**一句话动作**：循环 $k$ 次，每次挑一个能让当前收益**涨得最多**的元素，塞进去。

#### 💡 3. 直觉：为什么是 $(1-1/e)$？
- 最优解 $OPT$ 有 $k$ 个元素。贪心每一步都在追 $OPT$。
- **次模性保底线**：$OPT$ 里剩下的那些元素，它们**加起来**能带来的潜在总增量，一定 ≤ $k$ 倍的“单个元素最大边际增益”。
- 贪心挑了那个最大的，所以贪心这一步，**至少吃掉 $OPT$ 剩余差距的 $1/k$**。
- 每次吃掉剩余差距的 $1/k$，就像复利衰减。连吃 $k$ 次后，剩余差距 ≤ 初始差距的 $1/e$。吃掉的部分自然 ≥ $1 - 1/e$。
- 因为是**最大化问题**，近似比定义为 $ρ = OPT / ALG$，所以 $ρ ≤ 1/(1-1/e) ≈ 1.58$。

#### 📐 4. 近似比证明（逐行拆解，对应笔记 8.3.3）
我们要证：$f(A_k) ≥ (1 - 1/e) · f(OPT)$

**第①步：定义符号与单步增益下界**
- 设 $A_i$ 为算法第 $i$ 步结束后的集合（$A_0 = ∅$）。
- 第 $i$ 步的边际增益：$Δ_i = f(A_i) - f(A_{i-1})$。
- 根据贪心规则，$Δ_i$ 是全局所有未选元素中最大的。所以它一定 ≥ 只从最优解 $OPT$ 里挑元素能带来的最大增益：
  $Δ_i ≥ max_{x ∈ OPT} [f(A_{i-1} ∪ {x}) - f(A_{i-1})]$

**第②步：最大值 ≥ 平均值（关键放缩1）**
- $k$ 个数里最大的那个，肯定 ≥ 这 $k$ 个数的平均值。
  $Δ_i ≥ (1/k) Σ_{x ∈ OPT} [f(A_{i-1} ∪ {x}) - f(A_{i-1})]$

**第③步：次模性质收拢（关键放缩2）**
- 次模函数有一个核心不等式：把集合 $OPT$ 里的元素**一个一个**加到 $A_{i-1}$ 上，带来的边际增益之和，一定 ≥ 把 $OPT$ **整体**并上去带来的总增益。
  $Σ_{x ∈ OPT} [f(A_{i-1} ∪ {x}) - f(A_{i-1})] ≥ f(OPT ∪ A_{i-1}) - f(A_{i-1})$
- 代入第②步：
  $Δ_i ≥ (1/k) [f(OPT ∪ A_{i-1}) - f(A_{i-1})]$

**第④步：单调性托底**
- 因为 $f$ 单调，$OPT ∪ A_{i-1}$ 包含了 $OPT$，所以 $f(OPT ∪ A_{i-1}) ≥ f(OPT)$。
- 于是：
  $Δ_i ≥ (1/k) [f(OPT) - f(A_{i-1})]$
- 📌 **物理意义**：算法这一步的进步量，至少是“当前离最优解还差多少”的 $1/k$。

**第⑤步：建立递推不等式**
- 把 $Δ_i$ 换回 $f(A_i) - f(A_{i-1})$，移项整理：
  $f(A_i) - f(A_{i-1}) ≥ (1/k) f(OPT) - (1/k) f(A_{i-1})$
  $f(A_i) ≥ (1/k) f(OPT) + (1 - 1/k) f(A_{i-1})$
- 两边同减 $f(OPT)$，定义剩余差距 $Gap_i = f(OPT) - f(A_i)$：
  $Gap_i ≤ (1 - 1/k) Gap_{i-1}$

**第⑥步：迭代 $k$ 次收网**
- 从 $Gap_0$ 开始连乘：
  $Gap_k ≤ (1 - 1/k)^k · Gap_0$
- 初始 $A_0 = ∅$，通常假设 $f(∅)=0$，所以 $Gap_0 = f(OPT)$。
- 数学常识：$(1 - 1/k)^k$ 随 $k$ 增大单调递增逼近 $1/e$，且恒有 $(1 - 1/k)^k ≤ 1/e$。
- 所以：$Gap_k ≤ (1/e) · f(OPT)$
- 换回原式：$f(OPT) - f(A_k) ≤ f(OPT)/e$  →  $f(A_k) ≥ (1 - 1/e) f(OPT)$。
- 证毕。近似比 $ρ = f(OPT)/f(A_k) ≤ 1/(1-1/e) ≈ 1.58$。

#### 🔍 5. 最容易卡壳的 3 个坑
| 坑点                                           | 为什么错                         | 正确理解                                                     |
| :--------------------------------------------- | :------------------------------- | :----------------------------------------------------------- |
| “次模性到底用在哪一步？”                       | 以为只在开头或结尾用一次。       | 它用在**第③步**：把 $k$ 个独立边际增益的求和，压缩成一个整体增量 $f(OPT∪A)-f(A)$。没有它，求和项无法和 $f(OPT)$ 挂钩。 |
| “为什么 $(1-1/k)^k ≤ 1/e$？$e$ 怎么冒出来的？” | 觉得是强行套公式。               | 这是微积分极限 $lim_{k→∞}(1-1/k)^k = 1/e$ 的离散版本。在近似算法里，我们只需要知道它**严格 ≤ 1/e** 即可。考试直接写“由经典不等式可知”。 |
| “近似比到底是 $0.63$ 还是 $1.58$？”            | 混淆了“性能保证”和“近似比定义”。 | 对最大化问题，算法保证拿到至少 $63%$ 的最优值（$1-1/e$）。但近似比 $ρ$ 定义为 $最优/算法$，所以 $ρ ≤ 1/0.63 ≈ 1.58$。两者是一枚硬币的两面。 |

#### ✅ 总结记忆锚点
> **“贪心挑最大，必吃剩余 $1/k$。次模压求和，单调补差额。连乘 $(1-1/k)$，$k$ 步剩 $1/e$。吃掉 $1-1/e$，$1.58$ 锁死界。”**

### 🟦 问题 7：最大割（Max Cut）局部搜索 → 近似比 2

#### 📖 1. 问题到底在干嘛？
- **输入**：无向图 $G=(V,E)$，每条边有权重 $W(uv) ≥ 0$。
- **目标**：把顶点集 $V$ 切成两半 $S$ 和 $V-S$，使得**跨越分界线的边**的总权重 $W(S)$ 最大。
- **难点**：NP难。顶点越多，划分方式呈指数级爆炸 $2^n$。

#### 🤖 2. 算法具体怎么跑？（ApproxMaxCut 局部搜索）
```


1. S ← {任意一个顶点}      // 随便初始切一刀
2. Repeat
3. 检查每个顶点 v：计算把它换到对面集合后，割权的变化量 cost(v,S)
4. If 存在 v 使得 cost(v,S) > 0:
5. 把 v 换到对面集合（更新 S）
6. Until 没有任何顶点移动能增加割权
7. Return S

```


**一句话动作**：谁换边能“赚”就换，直到全员“安于现状”（局部最优）。

#### 💡 3. 直觉：为什么它不会太离谱？
想象你在分两队拔河，目标是让“跨队握手”的总价值最大。
- 算法停下来时，说明**每个人都“懒得换队”了**：他留在当前这边，连向对队的边权，已经 ≥ 换到对队后连向新对队的边权。
- 换句话说：**每个顶点连向“对面”的边权，至少占他所有连出边权的一半**。
- 把所有人的“跨队边权”加起来（每条跨队边被两头各算一次，正好等于 $2倍割权$），就能证明算法割权 ≥ 全图总边权的一半。而最优割绝对不可能超过全图总边权，所以比值自然卡在 2 以内。

#### 📐 4. 近似比证明（逐行拆解，对应笔记 8.4.1）
我们要证：$W(S*) / W(S) ≤ 2$（$S*$ 是最优割，$S$ 是算法停下来的局部最优割）

**第①步：翻译“局部最优”条件**
- 算法终止时，对任意顶点 $v$，移动它不会增加割权。即 $cost(v,S) ≤ 0$。
- 分情况看：
  - 若 $v ∈ S$，换到 $V-S$ 的代价变化 $≤ 0$ ⇒ $v 连向 S 内部边权 ≤ v 连向 V-S 的边权$
  - 若 $v ∈ V-S$，同理 ⇒ $v 连向 V-S 内部边权 ≤ v 连向 S 的边权$
- 无论 $v$ 在哪边，都满足一个铁律：
  $v 连向对侧的边权 ≥ v 连向同侧的边权$
- 设 $d_v$ 为 $v$ 连出的**总边权**。由上式可得：
  $v 连向对侧的边权 ≥ d_v / 2$

**第②步：全局求和（关键桥梁）**
- 对所有顶点 $v ∈ V$ 求和：
  $Σ_{v∈V} (v连向对侧的边权) ≥ Σ_{v∈V} (d_v / 2)$
- 左边怎么算？每条**跨越割的边**连接两个顶点，在求和时**被两头各算了一次**。所以左边 $= 2 · W(S)$。
- 右边怎么算？$Σ d_v$ 是把图中**所有边**的权值算了两头，所以 $Σ d_v = 2 · W(E)$（$W(E)$ 是全图总边权）。右边 $= (1/2) · 2 · W(E) = W(E)$。
- 代入不等式：$2 · W(S) ≥ W(E)$  →  $W(S) ≥ W(E) / 2$

**第③步：卡住最优解上界**
- 最优割 $S*$ 也是从图里挑一些边，它的总权重绝不可能超过图中所有边的总权重：
  $W(S*) ≤ W(E)$

**第④步：联立收网**
$$$
W(S*) ≤ W(E)          （最优割不超总边权）
      ≤ 2 · W(S)      （第②步结论）
→ W(S*) / W(S) ≤ 2
$$$
证毕。近似比严格 ≤ 2。

#### 🔍 5. 最容易卡壳的 3 个坑
| 坑点                                       | 为什么错                             | 正确理解                                                     |
| :----------------------------------------- | :----------------------------------- | :----------------------------------------------------------- |
| “$cost(v,S)$ 公式看着头晕，到底怎么来的？” | 被笔记里的双重求和符号吓住。         | 别背公式！直接想物理意义：$换边前的跨边权$ vs $换边后的跨边权$。差值 ≤ 0 就是“换了不赚”。用 $连向对侧 ≥ 连向同侧$ 代替公式，一目了然。 |
| “为什么求和左边是 $2W(S)$？”               | 漏算了“每条边被两个端点各贡献一次”。 | 画个图：割线穿过一条边，这条边的权值会被端点A算一次，被端点B算一次。所以全局累加一定是割权的2倍。 |
| “局部最优真的能逼近全局吗？”               | 觉得贪心+局部搜索容易陷在局部坑里。  | 对于 Max Cut，**局部最优的性质极强**。它直接锁死了“每点至少贡献一半边权”，这个结构性质足以推出全局 1/2 下界。这是特例，不是所有局部搜索都有常数近似比。 |

#### ✅ 总结记忆锚点
> **“局部不赚就不换，每点跨边占一半。全网加和乘二倍，总权一半保底盘。最优不超总边权，二倍比值自然翻。”**

### 🟦 问题 8：0-1背包问题（0-1 Knapsack）FPTAS → 近似比 $1/(1-ε)$

#### 📖 1. 问题与痛点
- **输入**：$n$ 个物品，重量 $w_i$，价值 $v_i$，背包容量 $C$。
- **目标**：选一个子集，总重 $≤ C$，总价值 **最大**。
- **痛点**：精确DP的时间复杂度是 $O(n · Σv_i)$。如果物品价值非常大（比如 $v_i$ 达到 $10^9$），DP状态空间爆炸，根本不是多项式时间。
- **破局思路**：既然价值太大导致DP慢，我们就把价值**按比例缩小**（抹掉零头），让总价值变小，跑完DP后再把解映射回原问题。只要控制“抹掉的价值”不超过总价值的 $ε$ 比例，就能保证近似比。

#### 🤖 2. 算法具体怎么跑？（ApproxPacking）
```

1. V_max ← max{v_i}                 // 找单个物品的最大价值
2. δ ← ε · V_max / n                // 设定“抹零头”的步长
3. For i = 1 to n Do:
       v'_i ← ⌊v_i / δ⌋             // 价值向下取整缩放
4. 用 (w_i, v'_i, C) 跑标准0-1背包DP，得到最优解集合 S
5. Return S
 
```

**一句话动作**：给所有价值除以一个很小的数 $δ$ 并向下取整，把“大价值”压缩成“小整数”，跑精确DP，返回原解。

#### 💡 3. 直觉：为什么误差可控？
- **抹零规则**：每个物品原价值 $v_i$，缩放后变成 $δ·v'_i$。因为向下取整，每个物品最多被抹掉 $< δ$ 的价值。
- **总量预算**：一个解最多选 $n$ 个物品，所以**任意解的总价值最多被抹掉 $< n·δ$**。
- **故意设 δ**：我们令 $δ = ε·V_max / n$，那么 $n·δ = ε·V_max$。也就是说，**全场最多损失 $ε·V_max$ 的价值**。
- **最优解兜底**：最优解 $OPT$ 至少能装下价值最大的那个物品（或组合），所以 $OPT ≥ V_max$。
- **结论**：损失的价值 $≤ ε·V_max ≤ ε·OPT$。还原后算法价值 $ALG ≥ OPT - ε·OPT = (1-ε)OPT$。近似比自然卡在 $1/(1-ε)$。

#### 📐 4. 近似比证明（逐行拆解，对应笔记 8.5.1）
我们要证：$ALG ≥ (1-ε) · OPT$（最大化问题，近似比 $ρ = OPT/ALG ≤ 1/(1-ε)$）

**第①步：建立缩放前后的不等式关系**
- 由取整定义：$v'_i = ⌊v_i / δ⌋$
- 显然：$δ·v'_i ≤ v_i < δ·v'_i + δ$
- 移项得核心不等式：$δ·v'_i > v_i - δ$  （①式）

**第②步：DP在缩放价值上是最优的**
- 设算法返回的解集合为 $S$，原问题最优解集合为 $OPT$。
- 在缩放后的价值体系里，$S$ 是DP找到的最优解，所以它的缩放总价值一定 $≥ OPT$ 的缩放总价值：
  $Σ_{i∈S} v'_i ≥ Σ_{i∈OPT} v'_i$
- 两边同乘 $δ$：$δ·Σ_{i∈S} v'_i ≥ δ·Σ_{i∈OPT} v'_i$  （②式）

**第③步：换算回原价值，代入误差上界**
- 算法实际价值 $ALG = Σ_{i∈S} v_i$。由①式左边 $δ·v'_i ≤ v_i$，得：
  $ALG ≥ δ·Σ_{i∈S} v'_i$
- 联立②式：$ALG ≥ δ·Σ_{i∈OPT} v'_i$
- 对右边再次使用①式 $δ·v'_i > v_i - δ$：
  $ALG > Σ_{i∈OPT} (v_i - δ) = Σ_{i∈OPT} v_i - Σ_{i∈OPT} δ$
- $OPT$ 最多包含 $n$ 个物品，所以 $Σ_{i∈OPT} δ ≤ n·δ$
- 代入 $δ = ε·V_max / n$，得：$ALG ≥ OPT - n·(ε·V_max / n) = OPT - ε·V_max$

**第④步：引入最优解下界收网**
- 背包问题有一个天然下界：最优解的价值至少等于单个物品的最大价值（否则只装最值钱的那个物品就比它好了）。即 $OPT ≥ V_max$。
- 所以：$ε·V_max ≤ ε·OPT$
- 代入上一步：$ALG ≥ OPT - ε·OPT = (1-ε) · OPT$
- 整理得近似比：$ρ = OPT / ALG ≤ 1 / (1-ε)$。证毕。

**附：为什么叫 FPTAS？**
- 缩放后总价值 $V' = Σ v'_i ≤ n·(V_max/δ) = n²/ε$
- DP时间复杂度 $O(n·V') = O(n³/ε)$
- 运行时间是 $n$ 和 $1/ε$ 的多项式 → **完全多项式时间近似模式 (FPTAS)**。

#### 🔍 5. 最容易卡壳的 3 个坑
| 坑点                                             | 为什么错                         | 正确理解                                                     |
| :----------------------------------------------- | :------------------------------- | :----------------------------------------------------------- |
| “向下取整 $⌊⌋$ 会不会让可行性判断出错？”         | 混淆了“价值”和“重量”。           | 缩放**只改价值 $v_i$，不改重量 $w_i$**。背包容量约束只看重量，所以缩放不影响可行性。DP只在缩小后的价值上找最优，解集完全合法。 |
| “$OPT ≥ V_max$ 一定成立吗？如果最值物品超重呢？” | 忽略标准假设或预处理。           | 近似算法理论中通常假设“单物品重量 $≤ C$”。若存在超重物品，预处理直接剔除即可，不影响 $OPT ≥ V_max$ 的成立。 |
| “近似比是 $1-ε$ 还是 $1/(1-ε)$？”                | 混淆了“性能保证”与“近似比定义”。 | 算法保证拿到至少 $(1-ε)$ 倍的最优值。但近似比 $ρ$ 定义为 $最优/算法$，所以 $ρ ≤ 1/(1-ε)$。当 $ε$ 很小时，$1/(1-ε) ≈ 1+ε$，两者本质一致。 |

#### ✅ 总结记忆锚点
> **“价值除δ向下整，单件抹零不超δ。n件总量卡预算，最优保底V_max。损失比例锁ε，多项式时间FPTAS。”**

### 🟦 问题 9：子集和问题（Subset Sum）FPTAS → 近似比 $1/(1-ε)$

#### 📖 1. 问题与痛点
- **输入**：正整数集合 $S=\{x_1, x_2, \dots, x_n\}$，目标上限 $t$。
- **目标**：找一个子集 $A \subseteq S$，使得 $\sum_{x \in A} x \le t$，且**总和最大**。
- **痛点**：精确DP需要维护一个列表 $L_i$，记录前 $i$ 个数能凑出的所有合法和。$L_i = L_{i-1} \cup (L_{i-1} + x_i)$。列表大小最坏是 $2^i$，指数级爆炸。
- **破局思路**：既然列表太长，我们就**定期修剪**。如果两个和非常接近（比如 $98$ 和 $100$），在目标 $t$ 很大时，保留 $100$ 就足够代表 $98$ 了。每步只丢一点点精度，$n$ 步累积起来误差依然被 $\epsilon$ 锁死。

#### 🤖 2. 算法具体怎么跑？（Approx-Subset-Sum）
```


1. L₀ ← <0>                     // 初始只能凑出和为0
2. δ ← ε / n                    // 设定每步允许的“相对损失率”
3. For i = 1 to n Do:
4. L_i ← Merge-List(L_{i-1}, L_{i-1} + x_i)  // 合并旧和 & 旧和+当前数
5. L_i ← Trim(L_i, δ)        // 修剪：相近值只留一个代表
6. 删除 L_i 中所有 > t 的元素
7. z ← L_n 中的最大值
8. Return z
 
```

**Trim 函数核心规则**（对应笔记）：
列表已排序。保留第一个数 $last$。遍历下一个数 $y$，如果 $y·(1-δ) > last$，说明 $y$ 比 $last$ 大了足够多，**保留 $y$ 并更新 $last=y$**；否则 $y$ 太接近 $last$，直接丢弃。
**一句话动作**：每生成一轮新和，就把“挤在一起”的数砍掉，只留间隔至少 $1/(1-δ)$ 倍的代表。

#### 💡 3. 直觉：为什么误差不会滚雪球？
- **修剪保底线**：Trim 丢弃 $y$ 时，一定存在一个保留的代表 $z$，满足 $z ≤ y < z/(1-δ)$。换句话说，**丢弃的值最多比保留值小一个 $(1-δ)$ 的比例**。即 $z ≥ y·(1-δ)$。
- **误差预算分配**：我们故意把总误差 $\epsilon$ 平均分给 $n$ 步，每步只允许相对损失 $\delta = \epsilon/n$。
- **连乘不爆雷**：第1步最多乘 $(1-δ)$，第2步再乘 $(1-δ)$……第 $n$ 步后，最多乘 $(1-δ)^n = (1-\epsilon/n)^n$。
- **数学常识**：$(1-\epsilon/n)^n ≥ 1-\epsilon$（伯努利不等式）。所以 $n$ 步累积下来，**最终结果至少是最优解的 $(1-\epsilon)$ 倍**。近似比自然卡在 $1/(1-\epsilon)$。

#### 📐 4. 近似比证明（逐行拆解，对应笔记 8.5.2）
我们要证：$z ≥ (1-ε) · OPT$（$z$ 是算法返回值，$OPT$ 是精确最大值）

**第①步：Trim 的核心保序性质**
- 设修剪前的完整列表为 $P_i$，修剪后的列表为 $L_i$。
- 由 Trim 规则：对任意 $y \in P_i$，必存在某个 $z \in L_i$，使得：
  $z ≤ y$ 且 $z ≥ y·(1-δ)$  （其中 $\delta = \epsilon/n$）
- 📌 **物理意义**：修剪只“向上取代表”，代表的值不会比原值小太多，相对差距被 $(1-δ)$ 兜底。

**第②步：数学归纳法建立递推关系**
- **命题**：$\forall i \in [0,n]$，对精确列表 $P_i$ 中的任意值 $y$，都存在修剪列表 $L_i$ 中的 $z'$，满足：
  $(1-δ)^i · y ≤ z' ≤ y$
- **基例** $i=0$：$P_0=L_0=\{0\}$，显然成立。
- **归纳假设**：假设对 $i-1$ 成立。即 $\forall y' \in P_{i-1}, \exists z' \in L_{i-1}$ 使 $(1-\delta)^{i-1}y' \le z' \le y'$。
- **归纳步骤** $i$：
  - $P_i$ 中的元素要么来自 $P_{i-1}$，要么来自 $P_{i-1}+x_i$。
  - 无论哪种，都对应一个 $y' \in P_{i-1}$ 使得 $y = y'$ 或 $y = y' + x_i$。
  - 由归纳假设，存在 $z' \in L_{i-1}$ 满足 $z' \ge (1-\delta)^{i-1}y'$。
  - 那么 $z' + x_i$ 一定在合并后的 $P_i$ 中（修剪前）。
  - 由第①步 Trim 性质，修剪后的 $L_i$ 中必存在代表值 $z$，满足：
    $z ≥ (z' + x_i)·(1-δ) ≥ z'·(1-δ) ≥ [(1-δ)^{i-1}y'+ x_i]·(1-δ) ≥[(1-δ)^{i-1}y'+ (1-δ)^{i-1}x_i]·(1-δ) ≥ (1-δ)^i y$
  - 命题成立。

**第③步：代入 $i=n$ 收网**

- 精确算法的最优解 $\text{OPT} \in P_n$。
- 由归纳命题，存在 $z \in L_n$ 使得：
  $z ≥ (1-δ)^n · OPT = (1 - ε/n)^n · OPT$
- 使用不等式 $(1 - x/n)^n ≥ 1 - x$（当 $0<x<1, n≥1$ 时成立）：
  $z ≥ (1 - ε) · OPT$
- 证毕。近似比 $\rho = \text{OPT}/z \le 1/(1-\epsilon)$。

**附：为什么叫 FPTAS？**
- 修剪后列表 $L_i$ 中相邻元素比值 $> 1/(1-\delta)$，所以列表长度最多为 $\log_{1/(1-\delta)} t \approx n \ln t / \epsilon$。
- 每步合并+修剪耗时 $O(|L_i|)$，总耗时 $O(n^2 \ln t / \epsilon)$。
- 运行时间是 $n$ 和 $1/\epsilon$ 的多项式 → **完全多项式时间近似模式 (FPTAS)**。

#### 🔍 5. 最容易卡壳的 3 个坑
| 坑点                                      | 为什么错             | 正确理解                                                     |
| :---------------------------------------- | :------------------- | :----------------------------------------------------------- |
| “为什么 $\delta$ 非要设成 $\epsilon/n$？” | 随便设个常数不行吗？ | 误差是**连乘累积**的。如果每步损失固定比例 $\delta$，$n$ 步后损失 $(1-\delta)^n$，当 $n$ 很大时会指数级衰减到0。除以 $n$ 能把总误差“摊平”，保证最终只亏 $\epsilon$。 |
| “$(1-ε/n)^n ≥ 1-ε$ 怎么来的？”            | 觉得是硬套公式。     | 这是**伯努利不等式**的特例。展开看：$(1-\epsilon/n)^n = 1 - \epsilon + \binom{n}{2}(\epsilon/n)^2 - \dots$。因为交错级数且首项后为正，截断到一次项必然 $\ge 1-\epsilon$。考试直接写“由经典不等式可知”。 |
| “Trim 会不会把最优解真值删没？”           | 担心砍掉关键数字。   | Trim 删的是“冗余近似值”，不是“精确路径”。归纳法证明了：**被删的值一定有一个保留的代表，且代表的值不会比它小太多**。算法虽然没存精确值，但存了“足够接近且合法”的值，最终最大值依然能逼近 OPT。 |

#### ✅ 总结记忆锚点
> **“精确列表太庞大，相近数值砍掉它。每步只亏 $(1-δ)$，$δ$ 均分 $ε/n$。连乘 $n$ 步不衰减，伯努利不等锁误差。FPTAS 稳落地，$(1-ε)$ 保底盘。”**

### 🟦 问题 11：集合覆盖（Set Cover）Primal-Dual → 近似比 $f$

#### 📖 1. 问题与痛点
- **输入**：全集 $X$，子集族 $F$，每个集合有代价 $c(S)$。每个元素 $e$ 最多出现在 $f$ 个集合中（$f$ 称为**最大频率**）。
- **目标**：选总代价最小的集合族覆盖 $X$。
- **痛点**：贪心算法保证 $ln|X|$，但当 $f$ 很小（比如区间覆盖、平面图覆盖，$f$ 是常数）时，贪心界太松。Primal-Dual 利用线性规划对偶，能给出更紧的 $f$ 倍界，且**不需要精确求解 LP**。

#### 🤖 2. 算法具体怎么跑？（Primal-Dual Set Cover）
```


1. 原始变量 x ← 0 （初始不选任何集合）
2. 对偶变量 y ← 0 （元素初始预算为0）
3. U ← ∅ （已覆盖元素集）
4. While U ≠ X Do:
       挑一个未覆盖元素 e ∈ X - U
       慢慢增加 y_e （让 e 的预算“涨水”）
       一旦存在集合 S 满足 Σ_{e∈S} y_e = c(S) （S 变“紧”）：
           令 x_S = 1 （买下 S）
           把 S 覆盖的所有元素加入 U
5. Return 所有 x_S=1 的集合
 
```

**一句话动作**：未覆盖元素轮流“涨预算”，哪个集合的预算池攒够了它的价格，就立刻买下它，覆盖一批新元素，直到全覆盖。

#### 💡 3. 直觉：为什么近似比是 $f$？
- **对偶变量 = 元素付的钱**：算法里 $y_e$ 就是元素 $e$ 出的预算。集合 $S$ 变紧时，说明它覆盖的元素们**刚好凑够了买它的钱**：$Σ_{e∈S} y_e = c(S)$。
- **总花费 = 元素总付款的加权和**：算法总代价 $Σ c(S)·x_S$ 其实等于每个元素付的钱 $y_e$ 乘以“它被多少个选中的集合覆盖”。
- **关键限制 $f$**：一个元素 $e$ 最多只属于 $f$ 个集合。算法可能让**多个同时包含 $e$ 的集合**在同一轮变紧并被买下。此时 $e$ 付的这笔钱 $y_e$，会被最多 $f$ 个集合**重复报销**。
- 所以：$算法总代价 ≤ f × 所有元素的总预算$。而“所有元素的总预算”正好是对偶问题的目标值，根据弱对偶定理，它**绝对不超过最优解**。因此近似比卡在 $f$。

#### 📐 4. 近似比证明（逐行拆解，对应笔记 8.6.3）
我们要证：$cost(C) ≤ f · OPT$

**第①步：原始 LP 与对偶 LP 定义**
- 原始（松弛）：$min Σ c(S)x_S$, s.t. $Σ_{S∋e} x_S ≥ 1 (∀e)$, $x_S ≥ 0$
- 对偶：$max Σ y_e$, s.t. $Σ_{e∈S} y_e ≤ c(S) (∀S)$, $y_e ≥ 0$
- **弱对偶定理**：任意对偶可行解 $y$ 满足 $Σ y_e ≤ OPT_f ≤ OPT$。

**第②步：算法构造的解满足“松弛互补条件”**
笔记命题1指出：若 $x,y$ 可行，且满足：
- $α-松弛原始条件$：$x_S≠0 ⇒ c(S)/α ≤ Σ_{e∈S} y_e ≤ c(S)$
- $β-松弛对偶条件$：$y_e≠0 ⇒ 1 ≤ Σ_{S∋e} x_S ≤ β$
则：$Σ c(S)x_S ≤ α·β·Σ y_e$

看我们的算法：
- **α=1**：集合 $S$ 只有当 $Σ y_e = c(S)$ 时才被买下（$x_S=1$）。刚好凑够，不多付。满足 $α=1$。
- **β=f**：元素 $e$ 一旦付钱（$y_e>0$），它最多只出现在 $f$ 个集合里。算法最多把这 $f$ 个集合全买（$x_S=1$）。所以 $Σ_{S∋e} x_S ≤ f$。满足 $β=f$。

**第③步：联立收网**
$$$
cost(C) = Σ c(S)x_S 
        ≤ α·β·Σ y_e          （命题1，代入 α=1, β=f）
        = f · Σ y_e
        ≤ f · OPT            （弱对偶定理：对偶值 ≤ 最优解）
→ cost(C) / OPT ≤ f
$$$
证毕。近似比严格 ≤ $f$。

#### 🔍 5. 最容易卡壳的 3 个坑
| 坑点                                       | 为什么错                  | 正确理解                                                     |
| :----------------------------------------- | :------------------------ | :----------------------------------------------------------- |
| “$f$ 是集合最大大小吗？”                   | 混淆了 $|S|$ 和元素频率。 | $f$ 是**元素出现的最大次数**（max frequency）。一个元素如果在 3 个集合里，$f≥3$。近似比只跟“元素重叠度”有关，跟集合多大无关。 |
| “$y_e$ 不是只付一次吗？怎么会被重复报销？” | 忽略了“同时变紧”的情况。  | 算法第6步是“增加 $y_e$ 直到**存在**集合变紧”。可能一个 $y_e$ 的上涨，同时让多个包含 $e$ 的集合达到价格阈值。这些集合全被买下，$y_e$ 就被算了多次，最多 $f$ 次。 |
| “Primal-Dual 一定比贪心好？”               | 以为理论界小就一定优。    | 看数据特征。若 $f$ 是常数（如区间覆盖 $f=2$），Primal-Dual 给 $2$ 倍，远优于贪心 $ln n$。但若 $f$ 接近 $n$，Primal-Dual 界就崩了，此时贪心更稳。 |

#### ✅ 总结记忆锚点
> “元素攒钱付预算，集合凑够立刻买。α=1 不多付，β=f 不超频。对偶总值卡最优，$f$ 倍近似稳落地。”
